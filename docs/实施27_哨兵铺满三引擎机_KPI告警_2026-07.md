  survive 登出、可读机器级 `EVENT_INGEST_KEY`）；
- 首跑实证：.176 / .198 各 "巡检通过 · Boundless 任务全绿"，SYSTEM 上下文 LastTaskResult=0。

## 四、相对方案的二次优化

1. **scp 绕开 LFS 障碍**：两引擎机 git pull 被 LFS smudge 挡住（老板视频资产），与其修 LFS
   环境，不如 scp 直传运维脚本（不涉版本库依赖、字节精确、带 BOM）——快且不碰老板的资产管线；
2. **哨兵统一 SYSTEM 账户**：三机一致，survive 登出（幻声机初次误注册到 user 账户，已改 SYSTEM）；
3. **KPI 包装器复用同款幂等骨架**：与 ledger-backup-cron 同结构（down-flag/恢复/静默），
   运维心智一致、易维护。

## 五、坑与处置

- **两引擎机 git pull 被 LFS 阻断**（`git-lfs filter-process failed`）：本轮用 scp 规避；
  根因是老板加了 LFS 跟踪的媒体资产而引擎机无 LFS 环境——**遗留项**：给引擎机装 git-lfs 或
  对引擎机的 pull 配 `GIT_LFS_SKIP_SMUDGE=1`（下轮，属引擎机 git 卫生，不阻塞告警）。
