## 二、深度思考：三视角分析

### 视角 A · 开发工程师（可维护性 / 管线 / 集成）

**A1. 原状态的工程问题（分析）**

1. **资产没有单一事实源**：品牌图片散在两份 `website/public/brand`（telegram-mtproto-ai 与 ai-p0-integration 各一份，已知双份分叉风险，见实施 05§五），desktop/extension 目录甚至没有任何图标资源。哪张是母版、哪张是派生品，没有任何记录。
2. **旧抠图管线有实质缺陷**：`build-boundless-marks.ps1` 的算法是「像素接近白色 → 透明」，不区分"外部背景的白"和"图形内部的白"。后果实测可见：面具眼睛、∞ 环内屏、星芒高光全部被抠穿成洞，深色头像上图标"漏底"。这就是为什么 TG 头像 `boundless-avatar.png` 里主标中心是个透明窗。
3. **命名/尺寸无规范**：只有 256 和 512 两档，没有 favicon 多尺寸、没有单色版（水印/印刷必需）、没有浅色场景版本。
4. **字体授权隐患**：此前任何带文字的图若用系统微软雅黑（`msyh.ttc`）制作，**微软雅黑不可再分发、不可嵌入对外交付物**（版权属北大方正），商用有真实法律风险。

**A2. 本轮工程优化（已落地）**

1. **单一事实源 + 可再生**：`00_master/src` 只放 9 张白底母版（唯一人工维护物），106 个产物全部由 `build_brand_assets.py` 派生。改口号/加产品/换字体 = 改脚本常量重跑，不存在"改一处漏五处"。`MANIFEST.md` 自动记录每个文件的尺寸与用途。
2. **抠图算法重写**（这是本轮最核心的技术修复）：
   - 洪水填充从画布四角出发，锁定"外部可达的白" → 透明；
   - 内部封闭白色区域按**连通域面积**分类：>0.4% 画面的是结构性镂空（∞ 内屏 6.8%、智聊气泡 10.8%、幻影屏幕 16.7%、靶环间隙 6.2%）→ 照旧抠透（维持既有品牌在深色场景的外观）；小面积的是高光星芒 → **保留**（旧管线会抠穿）；
   - 输出前做去白边（defringe），深色底上无 1px 白色包边。
3. **命名 = 代码键名**：所有文件名用 `brand.ts` 的 `ProductKey`（reachx/chatx/…），运营文件名与代码 key 一一对应，接 Bot API（`setProfilePhoto`）或 CI 时可直接编程索引。
4. **字体合规**：改用 Noto Sans CJK SC + Montserrat（均 OFL 授权，可商用可随库分发），字体文件进库，任何机器重跑结果一致，无授权风险。
5. **官网集成闭环**：实施 07 的"待补专属图标"回填完成，`PRODUCT_IMG` 摘除临时复用；旧 ps1 管线保留为站内备用但注明缺陷与新管线位置。

**A3. 工程侧遗留建议（未做）**

- ∞ 主标目前只有位图母版（AI 生成 1536×1024）。极端放大（展会喷绘）需要**矢量重绘**（Illustrator 手工描或委托设计），位图管线覆盖 ≤2560px 场景足够。
- 两份 website 的 `brand.ts` 仍是 5 产品（主干）vs 7 产品（部署源）分叉——本轮只同步了图片这种无害新增，代码合并按实施 05 的建议由老板定（统一以 ai-p0-integration 为源）。

### 视角 B · 市场经理（品牌架构 / 渠道 / 转化）

**B1. 原状态的营销问题（分析）**

1. **账号视觉无体系**：客服号、资源号、产品号全用同一张 TG 头像（或干脆没有统一头像），用户分不清"哪个是官方、哪个是客服、哪个是素材频道"。私域灰产环境里**仿冒客服诈骗是真实威胁**——没有可识别的官方视觉记号，用户被假客服骗了算在品牌头上。
2. **场景资产缺口**：只有 TG bot 头像一张场景化产物。X/FB/YouTube/公众号引流位、竖屏故事、销售讲产品矩阵的"一张图"，全部缺失，每次运营要用都得临时拼。
3. **英文排版无规则**：BOUNDLESS 在官网里是代码常量，但视觉上从未定义过它与"无界科技"的字号比例、对齐方式，运营自己排版必然五花八门。
4. **产品系没有视觉表达**：实施 07 定了"智连/幻境/通达"三系，但三系只存在于文案里，没有任何颜色/图形上的区分。

**B2. 本轮营销优化（已落地）**

1. **三级账号识别体系**（这是本轮最核心的营销设计）：
   - **官方主号** = 纯主标深空底（最干净、最大识别度）；
   - **资源号/频道** = 主标 + **全彩渐变光环**（光环 = "官方资源"记号，成本极低但辨识极强，40px 缩略图也一眼可辨）；
   - **客服号** = 主标 + 底部**大字胶囊徽标**「客服 / SUPPORT」（用户有付费问题时在会话列表里秒找到入口；中英双版对应中文盘/英文盘）。
   - 配套话术建议：在频道置顶写明"官方客服头像带『客服』徽标，无徽标皆假冒"——把视觉体系变成**防伪协议**。
2. **产品系色环**：产品资源号头像的光环用所属系的颜色（智连=蓝、幻境=紫、通达=橙）。用户关注多个产品号时，颜色即分类；三系概念从文案落到视觉。
3. **场景全覆盖**：TG 贴文/竖屏故事/X/FB/公众号/YouTube/桌面共 9 张背景，均按各平台官方尺寸与安全区（YouTube 文字置于 1546×423 安全区内，FB 顶部避开头像遮挡区）。
4. **矩阵海报 = 销售一张图**：1920×1080 一页讲清"母品牌 → 三系 → 七产品"，客服发给客户、销售投屏、频道置顶三用。这张图同时是品牌架构的**对内共识物**——新员工/代理商看一张图就懂产品线。
5. **中英排版定调**（详见 §三）：中文主、英文辅，比例与对齐规则固化进脚本，运营拿到的每张图排版天然一致。

**B3. 营销侧遗留建议（未做）**

- 光环/徽标体系需要**运营执行纪律**：所有客服号统一换头像并在频道公示防伪规则，否则体系不成立。
- 东南亚多语市场如后续做泰/越语盘，SUPPORT 徽标可加 ฝ่ายบริการ/CSKH 版本（脚本改一行参数即可出图）。
- 头像可做 A/B：光环版 vs 纯标版的关注转化率，用两个新频道跑一周即知。

### 视角 C · 用户（识别 / 信任 / 可读性）

**C1. 用户真实使用环境（分析）**

1. **TG 头像 = 圆形 + 极小**：会话列表约 40–56px、圆形裁切。原 `boundless-avatar.png` 主标只占画面 64%，圆裁后四角浪费，40px 下主标细节糊成一团；任何贴边元素会被裁掉。
2. **深色环境为主**：TG/Discord 用户大多开深色模式，深空底头像融入环境、彩色主标跳出——这个方向原头像是对的，保留并强化。
3. **信任判断在 3 秒内完成**：用户判断"这是不是官方号"靠头像+名称的一致性。七个产品号如果头像风格散乱，用户无法建立"这一族都是无界"的认知。
4. **中文用户读中文**：目标客群是东南亚华人（实施 05 拍板），tagline、徽标必须中文优先；英文是"科技感背书"而不是信息主体。

**C2. 本轮用户体验优化（已落地）**

1. **小尺寸可读性**：头像版主标占比从 64% 提到 68–74%（圆裁安全区内最大化）；客服徽标做成**大字胶囊**（字高 ≈ 画面 9%）而非小角标，40px 下"客服"二字仍可辨；所有头像出 128px 预览版，运营上传前可自查缩略效果。
2. **一族一脸**：7 产品头像同底色、同构图、同图标风格，用户在"添加的会话"列表里扫一眼就知道哪些是无界系账号；系色光环进一步给出"这是哪一类产品"的免读分类。
3. **双模式覆盖**：深色版为主（匹配 TG 环境），浅色版全套备份（文档、白底官网、印刷），用户在任何环境看到的品牌形象一致。
4. **矩阵海报的用户价值**：新客户问"你们都有什么"时，一张图代替 300 字介绍——降低理解成本就是降低流失。

## 三、LOGO 文字排版与英文排版规范（本轮定稿并固化进脚本）

**核心结论：中文是主标语言，英文是辅助层；两者用「等宽两端对齐」锁定关系，比例恒定，任何人重排都不会走样。**

1. **字重与字体**
   - 中文：Noto Sans CJK SC **Black**（公司名/产品名）、Medium（口号/说明）。选 Black 是因为主标本身浑圆粗壮，中文太细会"压不住"图形。
   - 英文：Montserrat **ExtraBold 800**（公司名）、Bold 700（产品名）、Medium 500（口号）。Montserrat 的几何圆形骨架与 ∞ 主标的圆润管状造型同源。
2. **中英锁定规则（lockup）**
   - `BOUNDLESS` 通过**加字距（letterspacing）拉到与「无界科技」完全等宽**，形成上下两端对齐的稳定块面——这是 CJK+Latin 组合标的经典做法，比"英文居中放下面"稳定得多，且字距拉开的全大写自带高端科技感。
   - 英文字高 ≈ 中文字高 × 0.35–0.4（辅助层级，不与中文抢主导）。
   - 产品组合标同规则：`ReachX` 等宽对齐「智拓」。`…X` 后缀统一驼峰（ReachX 不是 REACHX），保持产品名的"型号感"、与公司名 BOUNDLESS 全大写形成层级差。
3. **口号排法**
   - 竖排版：让沟通，无界（Medium）在上，`Communication, Boundless.`（Montserrat 500，字号 × 0.75）在下。
   - 横排版（横幅类）：`让沟通，无界 · Communication, Boundless.` 单行，中点分隔。
4. **间距系统**：标与文字间距 = 主标高 × 0.13–0.15；中文与英文行距 = 中文字号 × 0.13；所有比例参数化在脚本里，不靠目测。
5. **禁用**：不拉伸变形、不给主标加投影描边、深色文字不上深色底（备白字版）、英文不用衬线体。
