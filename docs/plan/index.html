02 / STRATEGY

产品分层策略 · 三梯队

按"用户上手成本"而非"技术炫酷度"排序：能云端即用的先上，需用户自建 GPU/真机的暂缓，重资产转为我们自己的增长引擎。

第一梯队 · 立即上线

轻量安装 / 云端即用

SaaS 订阅，注册即用，零本地部署。8 周内全部上线。
通译 LingoX聊天翻译，网页/插件轻客户端① 首发
智聊 ChatX 云版走官方渠道 API，授权即用② 次发
幻声 VoiceX 云 API我们托管 GPU，用户 API 调用③ 三发
第二梯队 · 暂缓产品化

重本地部署 / 实时 GPU

用户自建成本高、实时算力贵。8 周内不做轻量化，走定制/项目制承接。
幻颜 FaceX图片/视频换脸，GPU 重暂缓
幻影 LiveX实时直播换脸换声，延迟敏感暂缓
通传 VoxX会议同传，实时语音算力暂缓
第三梯队 · 转自运营

不卖散户 · 自己用来获客

用现有账号+设备驱动获客承接，为第一梯队免费导流、造血。
智拓 ReachX手机集群真机批量获客自用
智聊 ChatX 个号版个人号 AI 承接转化自用
重资产 GPU自托管，支撑云 API 与内容自用
为什么 VoiceX 归第一梯队？ 它虽需 GPU，但可做成"我们托管、用户按量调用云 API"的形态，用户端零部署——符合"轻量即用"。真正暂缓的是实时低延迟类（直播换脸/同传），它们算力成本与运维复杂度在 8 周窗口内不划算。

产能测算（30 台）

  • 每台/天有效触达 25–60（防封配额内）
  • 满负荷月触达 2.2万–5.4万
  • 折算月新增付费 4–170+（三档）

风控与防封

  • 配额管控 + 人类行为模拟（已内建）
  • 代理 IP 池轮换 + 出口健康检查
  • 养号节奏 + Cloudflare 抗封（已有基础）

运营 SOP

  • 分组养号 → 分批触达 → 话术 A/B 选优
  • 坐席工作台接管高意向对话
  • 每日漏斗数据复盘，迭代话术/人群
闭环价值：自运营既是"免费获客渠道"，又是"产品真实用户与数据来源"，还顺带验证了 ReachX/ChatX 的实战效果——三重收益，且几乎不吃投放预算。
