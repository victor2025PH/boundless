## 四、实施中发现并当场修的坑（二次优化）

1. **install_tasks.ps1 cmd 链陷阱（严重，已修+实测）**：`if not exist X mkdir X & cmd` 在 X
   已存在时，cmd 的单行 if 会把 `& cmd` 一并当 if 体跳过——任务 Last Result 0 但壳脚本
   **从未执行**（静默哑火）。修复：if 体括号隔离 `(if not exist X mkdir X) & cmd`，
   两种路径实测均执行；
2. **SYSTEM 账户无 python PATH**：任务首跑退出码 2 并给出明确指引（壳脚本防呆生效），
   重装带 `-PythonExe` 全路径解决——README 矩阵已把此列为 -Execute 前核对项；
3. **ssh stdin 传密钥不可靠**：Windows 管道到远端 PowerShell ReadLine/ReadToEnd 会出现
   字符级丢失/增添（65/63 字符两种坏形态），改 scp 临时文件 + 远端长度校验 + SHA256 回验 +
   即删；顺带发现三台机默认 shell 不一（.176=PowerShell5 / .198=cmd / .117=pwsh），
   远程命令要按机适配；
4. **VPS 写密钥的引号陷阱**：PowerShell 会先展开 `\$(...)`，第一次写入了空值——已删行重写并
   以值长度=64 验证；教训：跨 shell 传命令一律单引号包裹或落文件；
5. **旧试点数据根出仓**：`deploy/instances/tongyi/data` 改名备份后移动到
   `D:\chengjie-instances\`——改名后不再命中 `data/` 的 gitignore 规则，留在仓内会污染 git。
