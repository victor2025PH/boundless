## 一、人设总线（P5 第一步）——矩阵护城河动工

**模型**：一个 persona = face（形象）/ voice（声纹）/ prompt（语言人格）/ knowledge（知识）四个可选槽位；注册表只存元数据+sha256 指纹+授权关系，**资产本体永不出引擎**。

| 交付 | 位置 |
|---|---|
| 契约文档 | `platform/identity/PERSONA_BUS.md`（模型、grants 语义、全域清除协议、各引擎槽位映射、合规红线） |
| 账本 schema v3 | `personas` / `persona_grants` / `persona_purges` 三表（lib/ledger.ts 迁移 v3 + ledger-lib.mjs 逐字同步） |
| 数据层 | `website/lib/personas.ts`（幂等 upsert、purged 不复活、grant/revoke、requestPurge 按 grants 推导目标引擎、ackPurge 全回执置 purged，全程 audit） |
| 控制台 | `/console/personas` 列表（四槽位点亮图示/状态徽章/搜索筛选）+ 详情（授权矩阵 7 产品、归属客户、危险区"全域清除"带影响范围二次确认、purge 进度表）；客户 360 增"名下人设"分区 |
| 机器通道 | `GET/POST /api/sync/personas/purges`（Bearer EVENT_INGEST_KEY，最小权限：只拉清除指令与回执，不含客户数据） |
| 导入链 | `tools/persona_bus/export_avatarhub_personas.py`（只读导出，含防本体泄漏校验器）→ `scripts/ledger-import-personas.mjs` |

**全域清除协议**（写进契约）：console 发起 → status=purge_pending + 按 source_system ∪ grants 推导引擎各生成指令 → 引擎轮询拉取 → 本地删资产 → 回执 → 全部 ack 置 purged → 此后同键导入永不复活（skippedPurged）。冒烟 42 断言 + 独立复验（import→grant→purge→pull→ack→purged）全过。
