## 一、侦察中发现的关键问题：定位精神分裂（必须先解决）

代码实况暴露出**两套互相矛盾的定价与合规姿态并存**：

| 维度 | 官网 `website/lib/pricing.ts`（对外） | 后端 `billing.py`（系统内） |
|---|---|---|
| 结算货币 | **USDT** | **USD** |
| 计价模型 | 私有部署一次性 / 账号月费 | 按消息量 + 席位（SaaS 计量） |
| 头条产品 | **换脸 980 / 2580 USDT**（最高风险） | 通用套餐 community/basic/pro/flagship |
| 姿态标签 | "无审查、数据不出网、USDT 结算" | consent/授权/计量（合规 SaaS） |
| 通译（翻译） | **无定价**（不是可售 SKU） | 能力齐全但未单独计价 |
| 定价档位 | 换脸 2 档 + 智聊 2 档（198/598） | basic $49 / pro $149 / flagship $499 |

**结论**：
1. "USDT + 无审查 + 换脸头条"是典型的**规避监管信号**，会把整个公司拖进灰色地带，且与 `billing.py` 的合规 SaaS 完全对不上。
2. **最该先卖、风险最低、能力最成熟的"通译"，官网连价格都没有**——等于把现金流引擎藏起来，把定时炸弹摆头条。
3. 这正是三视角报告"卖点分散、宣称>实际"的根因。**不先把定位理顺，做再多功能都是在错误的货架上摆货。**

## 二、正式产品拆分：从「五产品线大杂烩」到「3 可售 SKU + 1 定制 + 1 隔离」

沿用团队已有品牌名（`brand.ts`），但**重新定义可售性分层与合规隔离**：

### SKU-A｜通译 LingoX — 跨境实时翻译 SCRM　【立即主推】

- **一句话**：让不会外语的团队，在 WhatsApp/Telegram/LINE 上跟全球客户即时双向对话，并自动沉淀客户资产。
- **底层能力（均已在代码，非画饼）**：
  - 多引擎+故障转移 `translation_engines.py`（AI/DeepL/Google/Hunyuan 本地）
  - 术语强制+品牌词保护 `translation_glossary.py`（mask/restore，引擎无关）
  - 翻译记忆 `translation_memory.py`、置信度徽标 `translation_confidence.py`、引擎健康 `translation_engine_stats.py`
  - 多模态：图片 OCR 翻译、语音翻译、内容级缓存（P58）
  - 术语库控制台 + CSV 运营（P59-60）、合规触达闭环（P61）
  - 统一收件箱 + 多坐席 + 客户 journey（对比云译"孤立翻译外挂"是降维优势）
  - 官网已有 `/interpreting` 落地页 + `TranslateDemo` + `RoiCalculator` 组件
- **合规等级**：🟢 低（纯沟通效率工具）
- **目标客户**：外贸个人/小团队、跨境电商客服、出海营销团队
- **缺口**：① 官网无定价（本阶段补）；② 前端翻译"即时感"体验待打磨；③ 未做轻量交付包

### SKU-B｜智聊 ChatX — 个人号 AI 获客中台　【护城河，需工程化】

- **一句话**：用个人号矩阵把"加到人→翻译沟通→AI 自动跟进→转人工→沉淀客户"跑成自动流水线。
- **底层能力**：`mobile-auto0423`（FB/Messenger 全漏斗 RPA）+ `telegram-mtproto-ai`（统一收件箱/AI 承接/handoff）；双键 BI 归因；`billing.py` 4 档 + `gate.py` 授权 + `branding_routes` 白标。
- **合规等级**：🟠 中高（个人号 RPA 违反平台 ToS，封号是模式固有代价）
- **目标客户**：中大型跨境私域团队、代运营公司
- **缺口**：真实 KPI 全空、多租户 E2=0%、真号 N4/N5 未联调（详见上游报告 L2）

### SKU-C｜幻声 VoiceX — 语音克隆　【低风险第二现金流】

- **一句话**：克隆任意音色做配音/朗读/客服语音/虚拟主播。
- **底层能力**：`index-tts`（IndexTTS2）+ `fish_speech` LAN + `tts_pipeline.py`（多后端）+ 情感层 `voice_emotion.py`。能力是真的（已跑在 7899）。
- **合规等级**：🟢 低（内容生产工具；需在授权音色/知情同意前提下使用）
- **目标客户**：有声内容、虚拟主播、IP 运营、客服语音
- **缺口**：未产品化封装、未独立定价、未做 API 化交付

### 定制项｜幻颜 FaceX + 幻影 LiveX — 换脸/直播换脸　【降级为定制，移出标品货架】

- **实况**：聊天主链**未接入**，仅 `faceswap_client.py` 薄 HTTP 客户端 + 报价书 + 营销页；需 GPU 主机。**是 PPT 阶段，不是标品**。
- **动作**：从 `pricing.ts` 头条**撤下**，官网标注为"定制交付项目（私有化 GPU 部署，需评估）"。980/2580 USDT 的一次性私有部署报价保留为**定制项目报价**，不与合规 SaaS 混列。
- **理由**：2026 年深度伪造立法与平台监管持续收紧；把它当旗舰会在尽调时引爆信任与合规风险。

### 隔离项｜情感陪伴 companion（深度人设/自拍/亲密度）

- **实况**：能力深（R 线做了大量陪聊质量/安全工作），但"AI 虚拟人设+情感引流+默认日本中年女性人设+批量私信"客观上靠近灰色地带。
- **动作**：**品牌 + 公司主体 + 客户准入三层物理隔离**，不与合规工具同品牌出现；作为 SKU-B 的高级能力仅对通过准入审核的客户开放。

| 待决项 | 建议 | 理由 |

## 四、SKU-A「通译」定价建议（对标云译，结合已有档位）

外部竞品锚点：云译/易翻译字符包 **$30/150 万字符**、端口订阅 **$48–1300/月**、包月无限 **$88/月**。

建议通译定价（**待老板拍板**，USD）：

| 档 | 价格 | 含量 | 对标/差异化卖点 |
|---|---|---|---|
| 免费试用 | $0 | 7 天 / 每日 200 次翻译 | 对标竞品免费版，降低试用门槛 |
| 入门（字符包） | **$39 一次性 / 150 万字符** | 单机翻译+术语库 | 略高于云译 $30，但带术语强制+翻译记忆 |
| 团队（订阅） | **$59/月起**（含 5 端口，阶梯加价） | +多坐席+客户 journey+漏斗计数器 | 对标云译端口订阅，但多"客户资产沉淀" |
| 专业（订阅） | **$99/月**（不限字符+多模态翻译+置信度+引擎健康） | 全功能 | 差异化：术语版本化、多模态同享术语强制 |

> 差异化定价逻辑：**不打价格战，打"翻译+SCRM 客户资产"**——竞品只给翻译，我们给"翻译 + 术语强制一致性 + 翻译记忆省钱 + 客户漏斗可见 ROI"，这些是续费理由。
