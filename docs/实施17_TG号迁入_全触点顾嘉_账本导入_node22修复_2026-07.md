## 四、VPS 部署链断裂与修复（本轮最大意外，已根治）

- 现象：deploy.sh 在 `npm ci` 阶段失败自动回滚——better-sqlite3 12.11.1 在 node 20.20.2
  （ABI 115）**无 linux-x64 预编译产物**，回落源码编译又踩 node-gyp 依赖目录 bug
  （`.deps/**.d.raw ENOENT`，预建目录也治不了）；之前两次部署成功属侥幸。
- **根治**：VPS node 20 → **22.23.1（ABI 127）**——better-sqlite3 v12 官方发布了 v127 的
  linux-x64 prebuilt，`npm ci` 直接下载安装，彻底绕开源码编译；重跑 deploy.sh 全绿
  （health OK / collect·grants·console·home 全 200 / better-sqlite3 读账本正常 personas=32）。
- 教训入档：**生产 node 版本要跟 native 依赖的 prebuilt 矩阵对齐**，deploy.sh 的回滚机制
  这次真正救了场（两次失败均自动恢复原站，线上零中断）。

4. **自启提前**：基于"挂号后可用性需求"推翻 runbook 时序，风险由幂等脚本+巡检覆盖。
