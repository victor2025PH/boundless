## 1. 启用 pre-commit + gitleaks（会改 Python 环境，需你确认）

> 目的：从源头拦截密钥/大文件再次入库。两个仓库各做一次。

```powershell
pip install pre-commit          # 环境变更点：装到当前 Python

foreach ($r in @("D:\workspace\mobile-auto0423","D:\workspace\telegram-mtproto-ai")) {
  Push-Location $r
  pre-commit install
  pre-commit install --hook-type pre-push
  pre-commit autoupdate         # 把 gitleaks rev 更新到最新稳定
```

## 4. 依赖锁定的落地注意事项

AI 已生成两个 `requirements.lock`（目标 Python 3.13，`uv pip compile` 只解析未安装）。落地前须知：

### 4.1 telegram 的 torch 是 GPU 坑

- 运行环境实际是 `torch==2.6.0+cu124`（CUDA，来自 pytorch 专用索引）；但 lock 解析出的是 `torch==2.13.0`（CPU，来自 PyPI）。
- **直接按 lock 装会得到 CPU torch**，GPU 加速失效。GPU 部署须单独：
  ```powershell
  pip install torch==2.6.0+cu124 --extra-index-url https://download.pytorch.org/whl/cu124
  ```
- 建议：在 lock 顶部注明该约束，或用 `requirements-gpu.txt` 单列 torch。

### 4.2 openai-whisper 体积决策（产品决定）

- `openai-whisper` 是 `whisper_local` 提供商（UI 可选），会拽入整个 torch（GB 级）。默认后端是更轻的 `faster-whisper`（AI 本轮已在 requirements 补上声明）。
- **若部署不需要 whisper_local**：从 `requirements.txt` 移除 `openai-whisper` 一行，可大幅减小基础镜像/环境体积；`faster_whisper` + 线上 `openai` 兼容 ASR 仍可用。移除后重跑 `uv pip compile` 更新 lock。

### 4.3 干净复现验证（可选，建独立 venv，不污染 mur）

```powershell
uv venv D:\_runtime_assets\verify-tg --python 3.13
uv pip install --python D:\_runtime_assets\verify-tg -r D:\workspace\telegram-mtproto-ai\requirements.lock
# 验证关键包可导入后删除该 venv
```
