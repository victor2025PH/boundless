## 零、为什么这件事必须先做

侦察 `website/lib/content.ts`、`pricing.ts`、`brand.ts`、`app/layout.tsx` 后，确认官网存在**三层互相打架的产品定义**，且**最该卖的通译在货架上是错位的、最危险的"无审查 AI"摆在明处**。这不是润色文案，是**在错误且高风险的货架上卖货**——不修，后面所有获客/尽调/支付都踩在雷上。

## 一、侦察实况：官网到底哪里乱（带证据）

### 问题 1｜通译/智聊套餐张冠李戴（`content.ts` `solutions`）

`translate`（通译 LingoX）条目自相矛盾：

```21:1:ai-p0-integration/website/lib/content.ts
// content.ts solutions[] 中 id: "translate"
title: "克隆音实时同传 + 双语字幕",          // ← 描述是"同传/字幕"
pricing: [
  { plan: "入门", price: "58 / 月", detail: "3 账号，双向翻译，1 平台" },
  { plan: "团队", price: "198 / 月", detail: "10 账号，全平台，AI 自动回复" },   // ← 套餐是"AI 自动回复"（智聊）
  { plan: "旗舰", price: "598 / 月", detail: "50 账号，人工接管，数据看板" },   // ← 智聊
]
```

`58/198/598 + 账号 + AI 自动回复 + 人工接管` 是**智聊 ChatX 的定价模型**（对照 `pricing.ts::autochatOffers` 的 198/598），却挂在通译名下。**真正的翻译 SCRM 产品（字符包/端口订阅）在官网无处可买。**

### 问题 2｜最高危产品线摆在明处（`content.ts` `solutions[id=private-ai]`）

```453:463:ai-p0-integration/website/lib/content.ts
{
  id: "private-ai",
  tag: "无界底座",
  title: "无审查 · 无禁区 AI 私有部署",
  desc: "私有化部署无内容审查、无话题禁区的大模型…",
  features: ["无内容审查", "无话题禁区", "私有部署不出网", "模型微调定制"],
  pricing: [
    { plan: "云端 API", price: "≈1.2x token", detail: "充值制，无审查中转" },
    ...
  ],
}
```

"无内容审查/无话题禁区/无审查中转"是**支付通道、应用商店、B 端尽调、广告投放的通杀红灯词**。它还不在 `brand.ts` 的 5 产品线里——是个游离的第六项。

### 问题 4｜三处产品结构对不齐

- `brand.ts`：5 产品线（facex/voicex/livex/lingox/chatx）。
- `content.ts::solutions`：6 条（voice/faceswap/translate/private-ai/digital-human/video-dubbing）——多出 `private-ai`，且 livex 被拆成 digital-human + video-dubbing。
- `pricing.ts`：只有 realtimeOffers（换脸）+ autochatOffers（智聊）两组 schema offer，**通译/语音无 schema 定价**。

## 三、具体改造 diff 方案（按文件）

> 定价数字用《实施01》第四章建议值（**待老板拍板**）。货币先用占位 `USD`，最终由老板定（第五章决策）。

### 3.1 `pricing.ts` — 权威 SKU offer 重构

**现状**：只有 `realtimeOffers`（换脸 980/2580 USDT）+ `autochatOffers`（智聊 198/598 USDT）。

**改造**：
1. 新增 `translateOffers`（通译，主推）：
   ```ts
   export const translateOffers: PriceOffer[] = [
     { id: "translate-charpack", name: "字符包", price: "39", currency: "USD",
       unit: "one-time", description: "One-time; 1.5M chars, term-locked glossary + translation memory." },
     { id: "translate-team", name: "Team", price: "59", currency: "USD",
       unit: "month", description: "Per month; multi-seat inbox, customer journey, funnel counter." },
     { id: "translate-pro", name: "Pro", price: "99", currency: "USD",
       unit: "month", description: "Per month; unlimited chars, multimodal translate, confidence + engine health." },
   ];
   ```
2. 新增 `voiceOffers`（幻声，低风险第二现金流）。
3. `realtimeOffers`（换脸）：`currency` 改 `USD`，并在 `description` 前缀标注 `Custom project (private GPU deployment)`；**从 schema 主 offer 撤下**（见 3.4）。
4. `currency` 字段类型从 `"USDT"` 收窄为 `"USD"`（换脸定制项目若保留 USDT，另开 `customCurrency` 字段，不混入标品）。

### 3.2 `content.ts` — 产品货架重排 + 套餐归位

1. **通译归位**（最重要）：`solutions[id=translate]` 的 `title/desc/pricing` 全部换成**真正的翻译 SCRM**：
   - title：`跨境实时翻译 SCRM`
   - desc：多引擎+术语强制+翻译记忆+多模态+客户资产沉淀（对比云译差异化）
   - pricing：改用 `translateOffers`（39 字符包 / 59 团队 / 99 专业）
2. **智聊独立**：把原 translate 里的"账号/AI 自动回复/人工接管"套餐移到 `chatx` 对应条目（或新建 `id: "autochat"`），与 `pricing.ts::autochatOffers` 对齐。
3. **删除 `private-ai` 条目**（无审查 AI）：整条移除；若确有私有 LLM 部署业务，改为中性表述"私有化部署（数据主权/本地推理）"，去掉"无审查/无禁区/无审查中转"。
4. **换脸/直播降级**：`faceswap`/`digital-human`/`video-dubbing` 合并标注为"定制交付项目"，移出主推区。
5. **hero/badge/trustline/pricingSection 话术**（L391/396/397/491）：
   - 删"USDT 结算""数据不出网"作为**卖点前缀**（数据主权可留，但不用"不出网"这种规避语气）。
   - hero 主线改为"多语种拟人翻译 + AI 自动跟进成交"，换脸从主句移除。
   - pricingSection `unit/priceCol` 的 "USDT" 改 "USD"（或老板定的币种）。

### 3.3 `brand.ts` — 产品分层元数据

1. 给每个产品加**可售性分级**字段（新增，供 UI 与销售统一消费）：
   ```ts
   tier: "flagship" | "core" | "assist" | "custom" | "isolated"
   // lingox→flagship(主推), chatx→core, voicex→assist,
   // facex/livex→custom(定制), companion 相关→isolated(不在公开货架)
   ```
2. `PRODUCT_ORDER` 调整：把 `lingox` 提到**第一位**（当前是第 4 位），换脸系后置。
3. 移除/改写 `discountPrefix` 等无影响，但检查是否有"无审查"关联文案。

### 3.4 `app/layout.tsx` — schema.org 结构化数据 + meta（SEO 层，易被忽略但很关键）

1. `description`（L17/45/52/63）：删除中英文 "USDT"、"uncensored private-deployment base"、"data stays off the public net"；改为"multilingual translation SCRM + AI-assisted cross-border sales; privately deployable with data sovereignty."
2. keywords（L29 "USDT"）：移除。
3. `PRODUCT_OFFERS`（L70-72）：`livex: realtimeOffers` 撤下或标 custom；**新增 `lingox: translateOffers`**（让主推产品有结构化定价被 Google 收录）。
4. Service `description`（L89）："settled in USDT" → 中性。

### 3.5 其余文件（成套替换"无审查/USDT"）

`ProductMatrix.tsx`(L17,27)、`site.ts`、`bot-knowledge.ts`、`telegram-bot.ts`、`tg-broadcast.ts`、`catalog-posts.ts`、`legal-content.ts`、`opengraph-image.tsx`、`manifest.ts`、`public/proposal/index.html`：按同一口径清理"无审查/无禁区/数据不出网卖点/USDT 结算"，通译提为主线。**建议用一次全局串替换 + 人工复核**（约 15 处文件）。
