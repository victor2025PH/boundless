## 三、P2：控制台实名账号 + RBAC（废除共享口令登录）

- 账本 schema v2：`users`（scrypt 密码散列）+ `sessions`（库存 token 的 sha256，cookie 存原值，12h 绝对生命期+滑动 last_seen）。
- 引导：users 空 → 持 CONSOLE_KEY 走 bootstrap 建首个 master；非空后 bootstrap 与口令登录关闭。
- RBAC：viewer（只读）/ admin（客户与归属写操作）/ master（用户管理）；`x-console-key` 头通道保留给脚本巡检（视为 master）；不能禁用/降级最后一个 master；禁用/重置密码即时撤销会话；全链路 audit（actor=`console:<username>`）。
- UI：登录卡改账号密码；/console/users 用户管理页；页头用户名+角色徽章；viewer 隐藏写控件。
- 29/29 冒烟断言通过（bootstrap/登录/403 矩阵/会话撤销/唯一 master 保护）。

## 四、P4：事件从 spool 接到集团

- **独立事件库** `group-events.db`（决策：事件量≫账本量，分库让钱权账本备份保持轻小）：events 表 + 三索引，event_id 主键幂等。
- **收集器** `POST /api/collect`：Bearer `EVENT_INGEST_KEY`，≤500 条/批，信封校验与 emitter 同口径，INSERT OR IGNORE 幂等。
- **补传器** `platform/observability/uploader.py`：断点续传（state 原子写+偏移推进）、5xx 指数退避、4xx 不重试、脏行不堵队列；selftest 28 项全过。
- **KPI 页** `/console/kpi`：总量、近 7/30 天按产品堆叠、Top 事件、最近事件表；空库给接入引导。
- 部署：VPS 配 `EVENT_INGEST_KEY`；各机 cron 每 5 分钟跑 uploader 指向 `https://bd2026.cc/api/collect`。
