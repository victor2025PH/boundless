## 1. 三层依赖模型（目标 vs 现状）

```
目标：  website / products / apps  ──►  engines(avatarhub/chengjie/huoke)  ──►  platform(共享底座)
现状：  website 独立可编译                三引擎自带全部横切能力(各一份)          platform 空壳(仅契约)
        products 空壳                    ▲ 横切能力(授权/合规/可观测)重复 3 份 ─┘
```

- **正确的部分**：产品线之间**横向零 import**（各引擎独立）、website 只经数据层(`lib/brand.ts`)驱动、每引擎自带 `.gitignore` 与 requirements。
- **缺口**：`platform` 该收敛的横切能力（`provenance/license/metrics/brand`）目前在 avatarhub/chengjie 各有一份，是"3 个仓放进一个文件夹"，还不是"共享底座的一个系统"。

## 2. 逐模块架构（每个模块的职责 / 关键文件 / 结构 / 依赖 / 完善度）

### 2.1 `platform/`（共享底座）— 🔴 未落地
- 职责：身份·资产总线 / 授权计量 / 合规出口 / 可观测 / 品牌令牌（横切五契约）。
- 结构：`identity/ licensing/ compliance/ observability/ brand/`，当前仅 `.gitkeep` + `README.md`（契约与迁移路径已定义）。
- 依赖：被所有层依赖，绝不反向依赖。
- 完善度：**契约已立、实现未迁**。当前实现散落：`engines/avatarhub/provenance.py|watermark.py|license*.py|metrics.py|telemetry*.py`、`engines/chengjie/src/licensing|monitoring`、`website/lib/brand.ts`。

### 2.2 `engines/avatarhub/`（数字分身引擎，原模仿音色）— 🟢 可运行 / 🟠 含冗余
- 职责：换脸·克隆·数字人·同传的技术底座（幻颜/幻声/幻影/通传 + 通译语音侧）。
- 关键入口：`avatar_hub.py`(9000 中枢编排) · `app_config.py`(服务清单/路径单一真相) · ~22 个 `*_server.py`/`*_api.py`（fish_speech/stt/lipsync/vcam/faceswap/live_interpreter…）· `doctor.py`(自检) · `provision.*`。
- 结构：510 py；`static/`(控制台前端) `requirements/`(各 conda env 基线) `docker/` `installer/` `tools/`(claims_lint/gate)。
- 依赖：内部微服务经 `app_config.SERVICES` 编排；LLM 接 Ollama。
- 完善度：运行面完整；**冗余**：`demo_record/`(292 文件·Playwright 缓存) `clothes/`(152) `ui_snapshots/`(31) `refs/`(8) 不该入库（见 §3）。

### 2.3 `engines/chengjie/`（承接引擎，原 telegram-mtproto-ai）— 🟢 体量最大 / 🟠 含重复站
- 职责：多平台统一收件箱 + AI 承接/翻译栈/陪伴（智聊 ChatX + 通译 LingoX 文字侧）。
- 关键入口：`main.py`(启动编排) · `requirements.txt` · `src/`(ai/inbox/contacts/integrations/licensing/monitoring/voicecall/web…)。
- 结构：1380 py + 159 tsx；`config/`(预设 overlay) `domains/`(行业 KB) `services/` `plugins/` `tests/`。
- 完善度：功能最全；**重复**：内嵌 `website/`(232 文件) 与全域 `website/` 并存——需确认是否同一站，若冗余应移除（见 §3）。

### 2.4 `engines/huoke/`（获客引擎，原 mobile-auto0423）— 🟢 可运行
- 职责：真机 RPA 获客（智拓 ReachX；FB/Messenger/TikTok）。
- 关键入口：`server.py` · `requirements.txt` · `migrations/`(DB 版本) · `src/`(behavior/device_control/host/app_automation…)。
- 结构：590 py；`config/`(大量 yaml 策略) `plugins/` `tests/` `vendor/`。
- 完善度：结构清晰、带 migrations；`config/*.yaml` 已脱敏（本轮）。

### 2.5 `products/`（产品线薄封装）— 🟠 仅骨架
- 职责：7 条产品线的产品化封装（SKU/定价/overlay/落地页联结/交付包/产品文档）。
- 结构：`zhituo zhiliao tongyi tongchuan huansheng huanying huanyan`，各仅 `README.md` + `.gitkeep`。
- 完善度：**只有壳**。缺：每产品的 SKU 定义、裁剪 overlay（如 tongyi 的 `lingox.overlay.yaml`）、落地页/引擎映射、交付脚本。

### 2.6 `website/`（官网 usdt2026 · Next.js）— 🟢 良好
- 职责：统一销售门面 + 控制面（/admin /api /app 小程序 + 多语 /en /ja /ko）。
- 关键文件：`lib/brand.ts`(三系七产品单一真相) · `lib/content.ts`(文案 zh/en) · `lib/pricing.ts` · `components/ProductMatrix.tsx`(三系陈列) · `components/Navbar.tsx`(三系下拉) · `app/layout.tsx`(SEO/schema)。
