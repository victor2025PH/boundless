## 二、当日拦截的事故(机制存在的理由)

1. **单体词条丢失更新**(09:51):两条工作流并行编辑 `web_i18n.py`,后写覆盖先写——中英齐平门禁当场抓住;催生 i18n packs 机制。
2. **半成品 API 被生产热重载捕获**(14:1x):开发树上未提交的 `collect_packs` 三元组改造被运行实例热重载,fail-safe 兜住(UI 无损)但词条冻结;`git merge` 又因实例持锁失败——催生生产独立 checkout 与切换门禁。
3. **切换会回退未合并修复**(15:0x):`main..feat` 差集里躺着 drafts 422 修复——切换脚本 P2 门由此而来;脚本上线数分钟即拦下一个新出现的同类提交。
4. **422 疑云排除**(14:4x):怀疑生产 drafts 全量 422,探测证实为 401(修复已随同事重启上线)——先取证后行动,避免了一次不必要的紧急干预。
