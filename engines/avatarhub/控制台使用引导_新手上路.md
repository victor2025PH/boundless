# 无界运营后台 · 新手上路（照着做，10 分钟会用）

> 面向：客服 / 运营 / 老板。这是"跟着点"的上手引导；完整功能说明见 `控制台使用文档.md`。
> 后台地址：**https://usdt2026.cc/console**（任意电脑/手机浏览器打开即可，不用装任何软件）。

---

## 第 0 步 · 它是什么

一个网页后台，管这些事：**客户名单、装机的机器、发授权（两种方式）、兑换码、订单、销售额、到期**。
所有客服打开同一个网址、各自账号登录，看到的是**同一份实时数据**——甲改了乙刷新就看到，天生互通。

> 记住一句话：**发"授权"要靠公司那台"签发机"在后台默默盖章。** 签发机没开，授权会一直卡在"签发中"。见文末。

---

## 第 1 步 · 登录并改密码

1. 浏览器打开 **https://usdt2026.cc/console**。
2. 输入管理员账号登录（初始 `admin` + 给你的初始密码）。
3. 左边点「**客服账号**」→ 找到 admin 那行 →「改密」→ 设一个你自己的强密码。**务必先改，初始密码只是临时的。**

## 第 2 步 · 给每个客服建一个账号

「客服账号」→「+ 新建账号」→ 填用户名 + 初始密码 + 选角色：

| 角色 | 能做什么 |
|---|---|
| **管理员 admin** | 全部，包括管理客服账号 |
| **客服 agent** | 管客户 / 发授权 / 发兑换码 / 处理订单（日常客服用这个） |
| **只读 viewer** | 只能看，不能改（适合查数据的人） |

建好后把网址 + 各自账号发给客服，他们自己登录即可。

## 第 3 步 · 认识主界面

左边一列菜单，从上到下：

- **概览**：一眼看客户数、在线机器、有效授权、待签发、近 14 天签发趋势。
- **客户**：建客户档案（谁买的、联系方式、公司、备注）。
- **机器**：装了产品并联网的机器**自动出现**在这里（指纹 / 显卡 / 版本 / 在线状态）。
- **授权**：路线A——按机器指纹签发正式授权，出 `license.key` / 授权码。
- **兑换码**：路线B——批量生成兑换码发给客户，客户自己在软件里输码激活。
- **订单**：官网下单的订单，标记已付 / 一键开通。
- **销售**：营收、按套餐、30 天内到期的授权（催续费用）。
- **审计日志**：谁在什么时候改了什么。

---

## 第 4 步 · 给客户开授权（两条路，任选）

### 路线A — 直接给授权码 / key 文件（最常用）

适合："客户把机器指纹发给你，你签一份发回去"。

1. 让客户在软件里点「🔑 授权」→ 复制「机器指纹」发给你（形如 `184F-76D5-1262-1586`）。
2. 后台「**授权**」→「+ 签发授权 (路线A)」→ 填：
   - 机器指纹：粘客户发来的（填 `*` = 站点授权，不绑机，慎用）；
   - 档位：试用 / 标准 / 旗舰 / 企业；
   - 有效天数：如 365；填 0 或负数 = 永久；
   - 被授权方：写客户名（可选，只用于展示）。
3. 点「签发」。这条授权会先显示「**签发中**」。
4. 等公司签发机盖好章（通常几分钟内），状态从「签发中」变「**有效**」，右边出现「**下载key**」按钮。
5. 点「下载key」下载 `license.key` 发给客户。（这个文件里的内容就是"授权码"；客户若想要文本形式，用记事本打开它复制内容即可。）
6. 客户：把 `license.key` 放到软件目录、或用软件里「导入 license.key」按钮、或把文件内容粘进「粘贴授权码」框点激活。三选一，效果一样。

### 路线B — 发兑换码，客户自己激活（适合批量 / 自助）

1. 后台「**兑换码**」→「+ 生成兑换码 (路线B)」→ 选档位、天数、**座位数**（这码能激活几台机器）、数量（可一次生成多个）。
2. 点「生成」，码自动复制（形如 `AVH-XXXX-XXXX-XXXX`），发给客户。
3. 客户在软件「🔑 授权」→「订单号/兑换码」框输码 → 点「在线激活」。
4. 第一次点会提示"正在开通，请稍后重试"（签发机在盖章），**过一两分钟再点一次「在线激活」**就激活成功了。

> 座位数 = 这一个码最多能在几台不同机器上激活。1 座 = 一码一机。

---

## 第 5 步 · 机器名单是怎么来的

客户装了产品并联网后，会**自动**把机器信息（指纹 / 显卡 / 版本 / 档位）报到「机器」页，你无需操作。
可以把机器「归属」到某个客户，方便按客户管理。

> 如果机器该出现却没出现，见 `控制台使用文档.md` 的「机器不显示怎么办」。

---

## 第 6 步 · 吊销 / 到期 / 续费

- **吊销**（退款/违约）：「授权」→ 找到那条 →「吊销」→ 填原因。签发机会重签"吊销名单"下发，客户机联网后失效。
- **看谁快到期**：「销售」页底部「30 天内到期授权」列表 → 主动联系续费。
- **续费**：给客户新签一份（路线A）或发新码（路线B）即可。

---

## ⚠️ 唯一要记住的坑：签发机必须开着

发授权（路线A/B）的"盖章"这一步，是在**公司那台保管密钥的电脑**上完成的（私钥绝不放服务器，这是安全底线）。
那台机器要一直运行签发程序：

```
python sign_worker.py --watch 20
```

- 开着 → 授权几分钟内从「签发中」变「有效」。
- 没开 → 授权一直卡「签发中」，客户激活也一直"正在开通"。

**发现授权老是"签发中"，第一件事就是去看那台签发机的程序还在不在跑。**

---

## 常见问题速查

| 现象 | 怎么办 |
|---|---|
| 授权一直"签发中" | 签发机 `sign_worker.py` 没开，去开上（见上）。 |
| 客户输码提示"正在开通，请稍后重试" | 正常，签发机盖章中，过 1-2 分钟让客户再点一次。 |
| 客户说"机器不匹配" | 授权绑的指纹和客户当前机器不一致——让客户重发指纹，重新签发。 |
| 机器没出现在列表 | 该机没跑会上报的新版程序，或不是装的中枢；见完整文档。 |
| 忘记密码 | 让管理员在「客服账号」里给你「改密」。 |
