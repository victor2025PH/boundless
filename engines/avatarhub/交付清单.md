## 1. 环境就位（部署指南 §1–§3）

- [ ] Windows 10/11 + NVIDIA GPU（显存 ≥8GB；HD/活体推荐 4090/5090）
- [ ] GPU 驱动 / CUDA 与显卡匹配（RTX 5090=sm_120 → CUDA 12.8 / torch≥2.7）
- [ ] Miniconda/Anaconda 已装
- [ ] `provision.bat` 体检 6 个 conda 环境就位；缺失用 `provision.bat --create`
- [ ] 模型已就位（MuseTalk / LivePortrait / GFPGAN / TTS 等，见 `MANIFEST.json`）
- [ ] LLM 就绪（本机 Ollama 或云端 key 写入 `secrets.bat`）
- [ ] `app_config.py` 自检路径解析正确（`<env>\python.exe app_config.py`）

## 2. 启动与就绪（部署指南 §4–§5）

- [ ] `launcher.bat` 或 `start_all_services.bat` 启动核心链路
- [ ] `python ready.py` 退出码 0（核心链路就绪）
- [ ] 控制台可访问 `http://127.0.0.1:9000/ui`；手机端 `:9000/phone`
- [ ] 至少 1 个角色已创建并绑定声音，激活后"数字人准备中"提示能正常显示并转就绪

## 3. 商用安全加固（暴露到局域网/外网时必做，README §4.1）

- [ ] 管理面令牌 `AVATARHUB_API_TOKEN` 已设（跨机写操作需令牌；看板可设）
- [ ] GPU 子服务令牌/白名单 `AVATARHUB_SERVICE_TOKEN` / `AVATARHUB_SERVICE_ALLOW_IPS` 已设
- [ ] CORS 白名单 `AVATARHUB_CORS_ORIGINS` 按访问来源收敛
- [ ] 厂商私钥 `secrets\license_vendor_sk.pem` **未随交付包外泄**（`.gitignore` 已忽略）

## 4. 授权激活（对外卖授权时，README §4.1）

- [ ] 已用客户机器指纹签发 `license.key`：`license_admin.py issue --machine <指纹> --edition <档> --days <N>`
- [ ] `python license_admin.py status` 或 `GET /api/license/status` 显示 valid + 正确档位/有效期
- [ ] 如需强制：`set AVATARHUB_LICENSE_ENFORCE=1`（默认 0=只评估不拦截；强制下无效授权只降级不崩）

## 5. 容量与并发（部署指南 §7，定容量/定价依据）

- [ ] `deploy.env.bat` 已设 `CONV_MAX_CONCURRENT=auto` + `CONV_MAX_QUEUE=20`（单卡 K=1 优雅排队）
- [ ] `GET /api/capacity` 显示 `enabled:true, auto:true, max=健康卡数`
- [ ] HD 直播机已设 `LIPSYNC_HD_PREWARM=1`（首条 HD 句即满速）
- [ ] 容量预期已与客户对齐：**单卡 = 1 路实时数字人 + 观众排队串行应答**；真并发需加卡（`SVC_LIPSYNC` 多副本）

**单卡实测基线（RTX 5090）**：单路首帧 STD ~1.3s / HD ~4.5s；N≥2 路独立并发不可行（须加卡）。

- [ ] 报告留档 `logs/deliver_report.json`
