# 开播页优化 · 依赖本地化 · 全站容错统一 —— 改动说明 v1

> 本轮目标：以「开发 / 市场 / 用户」三视角重构 **开播页** 体验，并把成熟范式（前置体检 / 健康灯 / 简易模式 / 失败指引）外溢到其余创作与客户端页面；同时完成依赖本地化（离线可交付）。
> 全部改动为前端自包含、lint 通过、已对运行中的服务（`:9000`）实测验证、可随时回退。

## 一、改动清单（按主题）

### A. 开播页（`static/ui.html` · 开播 Tab）

| # | 改动 | 为什么 | 风险/回退 |
|---|------|--------|-----------|
| A1 | **简易 / 专家 双模式**（顶部工具条，`localStorage` 记忆） | 简易模式只露关键操作，降低非技术主播的认知过载 | 纯显隐，删工具条即回退 |
| A2 | **开播前置体检清单**（角色 / 视频源 / 麦克风 / 虚拟声卡 四徽章 + 悬停指引） | 设备名不匹配时此前直接失败丢英文报错；改为开播前预防 | 删 `preflight()` 调用即回退 |
| A3 | **就绪门控**：未激活角色时禁用「一键开播」+ 橙色提示 | `one_click_start` 必须有 profile，否则必然失败 | 去掉 `!streamReady()` 即回退 |
| A4 | **设备实时检测提示**：音频输入/输出下拉显示「已检测到 N 个 / 未检测到」 | 让用户知道刷新设备是否成功 | 文案行，删除即回退 |
| A5 | **效果调节 简易/专家分层**：简易只给预设，参数收进专家 | 减少首屏旋钮数量 | `x-show` 包裹，移除即回退 |
| A6 | **换脸四参数标「🧪 实验·暂未生效」并禁用** | 后端引擎当前不应用这些参数，避免误导用户空调 | 去掉 `disabled` 与标记即回退 |
| A7 | **实时质检健康灯**（流畅/一般/卡顿/未开播）+「⚡一键降档」常驻；原始 ms/失败率折叠进「技术详情」 | 把技术指标翻成一眼可懂状态；客户演示更干净 | `streamHealth()` + `showPerfDetail` |
| A8 | **内嵌「换脸前 / 换脸后」对比预览**（16:9 + object-contain，约 0.16s 刷新心跳） | 解决「必须弹窗、面板内变形」痛点；天然营销素材 | `previewTick` 心跳随 `mjpegOn` 启停 |
| A9 | **主 CTA 视觉强化**：渐变实心主按钮 + 描边次级停止按钮 | 拉开主次层级 | 改 class 即回退 |
| A10 | **「🔴 直播中」状态卡** + 演示模式联动（自动收起技术指标） | 开播成功仪式感 + 对客户干净 | `perf.streaming` 驱动 |
| A11 | **修复真实 bug：效果预设此前未下发后端** | 后端 `one_click_start` 早已支持 `video_width/fps/...`，但前端漏传，预设一直不生效 | 见 `oneClickStart()` payload |
| A12 | **失败指引映射表 `streamStepHint()`** | 把后端原始报错（ADB/scrcpy/VB-Cable/模型/端口/超时…）翻成「原因+下一步」 | 删调用即回退 |
| A13 | **统一摄像头选择器**：合并 `cameraOverride → selectedCamera` | 此前两套状态、易不一致 | 旧变量保留占位 |

### B. 依赖本地化（离线可交付）

- **新增 `static/vendor/`**：`tailwindcss-3.4.js` / `alpine-3.14.1.min.js` / `jszip-3.10.1.min.js`，`ui.html` 改为本地引用，**移除全部外网 CDN**。
- Alpine 由浮动 `3.x.x` **钉死到 3.14.1**（可复现）；文件名带版本号（升级即天然 cache-bust）。
- 经核实本机 **无 `node`**，故未走 Node+Tailwind 构建路线，vendor 方案零工具链、低风险，先拿 80% 收益。

### C. 全站容错统一

- **新增 `static/_errutil.js`**：全站唯一错误映射 `window.friendlyErr`，覆盖 取消 / 麦克风(被拒绝·未检测到·被占用·参数·HTTPS) / 网络 / 超时 / HTTP 4xx·5xx。
- `phone.html` / `converse.html` / `ui.html` 删除各自的本地副本，统一引用共享 util（避免多副本漂移）。
- **修复真实 bug**：`phone.html` 的「按住说话」`getUserMedia` 此前**无错误处理**，拒绝授权会静默失败——现给友好 toast。
- `converse.html` 录音/免提/对话失败的原始英文报错 → 人话指引。
- `ui.html` 克隆录音的麦克风错误 → `micErrText()` 委托共享 util（带本地兜底）。

### E. 组件化（无构建，无需 Node）

- **抽离 `hub()` Alpine 逻辑到 `static/hub.js`**：将 `ui.html` 内联的约 2860 行 / 128KB JS（`function hub(){…}` + `enc` 全局工具）整体外移为独立文件，`ui.html` 仅保留 `<script src="/static/hub.js"></script>`。
- 语义不变：经典脚本（在 body 内、Alpine 为 `defer`）按解析顺序先于 Alpine 执行，`hub()`/`enc` 仍在全局作用域，Alpine 表达式照常解析。
- 收益：标记与逻辑分离，`ui.html` 体积 33 万→20 万字符，可维护性显著提升；**零工具链、可回退**（把 hub.js 内容贴回内联 `<script>` 即还原）。
- 已实测：`/static/hub.js` 200；`/ui` 引用 hub.js ✓、内联 `function hub()` 已消失 ✓。

### D. 创作组一致性（`ui.html`）

- **语音 Tab**：合成前置体检（角色/语音服务/文本）+ TTS 离线警告 + 空文本按钮门控。
- **批量 Tab**：补 TTS 离线警告（唯一缺口；无角色/文本门控原已有）。
- **唱歌 Tab**：补空歌词按钮门控（服务离线降级提示原已有）。
- 同传 Tab：原已有在线状态 + 启动指引，**本轮零改动**（避免冗余）。

## 二、改造前后（开播页）
