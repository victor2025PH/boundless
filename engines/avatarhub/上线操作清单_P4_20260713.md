## 上线顺序（灰度渐进，每步可停）
1. **备份私钥**：把 `secrets/release_sign_ed25519_sk.pem` 复制到离线保险位置（1Password/加密 U 盘）。这是整个签名体系的根，丢了所有存量客户端都收不到未来的签名更新。
2. **填 `release.config.json`**（从 example 复制）：`base_url` 指 live 版本根、`sign_release: true`、`rollout_percent: 10`（首灰度 10%）、`telemetry_url/telemetry_token` 填 VPS 端点与令牌、`sign`（exe 代码签名证书，若有 EV/OV）。

   > **无代码签名证书也能上线**（当前无证书即走此路）：`release.config.json` 里 `sign.enabled=false` + `sign_release=true`。区别务必分清：
   > - **代码签名证书（sign，付费，可选）**：只决定 Windows 首次运行是否弹「未知发布者」。没有它 → 用户首次运行点「更多信息 → 仍要运行」即可，程序功能不受影响。下载页/install_notes 已加入该引导 + SHA-256 自校验。
   > - **更新清单 Ed25519 签名（sign_release，自签免费，已内置，必开）**：保证 OTA 更新不被篡改，与代码证书无关。
   > **架构红利**：只有"首次下载安装包"会碰 SmartScreen；之后所有版本更新走程序内置 app 增量通道（Ed25519 验签），**永不再触发 SmartScreen**。所以无证书的负面影响仅限"新用户第一次安装"，且随下载量累积 Windows 会自动积累文件信誉。日后买了 EV/OV 证书，把 `sign.enabled=true`+证书填上重发一版即可，无需改任何代码。
3. **出可发布树**：`python publish_release.py`（会自动：构建 exe→签 exe→make_release→**给 app 打 10% 灰度戳+签 manifest+验签自检**→签安装包→本地自检）。任一门禁失败即停、不产出树。
4. **上传**：按 `dist/publish/<ver>/UPLOAD.md` 传到 live 站（上传脚本已改"临时名→原子改名"，不会让客户端拉到半成品）。
4.5. **上传前 go/no-go 预检**（强烈建议）：`python tools/golive_preflight.py --version <ver>` —— 一键红绿灯核对：manifest 验签、无私钥泄漏进发布树、app 灰度戳未误停、包 sha 一致、遥测端点可达、证书/SHA256SUMS 状态。**NO-GO(exit 2) 就别上传**，按阻断项修完重跑到 GO。（已内置 FAIL 拦截：篡改签名/私钥泄漏/误停放均实测能拦。）
5. **上传后冒烟**：`python publish_release.py --verify-remote`，或 `python tools/golive_preflight.py --version <ver> --remote` —— 验远端 manifest 可达+**验签通过**+控制通道未误停当前版+各组件体积/SHA 一致。不过不要继续。
6. **灰度观察**：开 `/t/dash?token=...` 看板，盯"版本×错误率"交叉视图。10% 跑 6–24h，崩溃占比无异常 → `python tools/rollout_ctl.py set 50 dist/publish/<ver>/manifest.json`（重签）→ 重传 manifest → 再观察 → `set 100`。
7. **开生产自动刹车**（可选但推荐）：在 VPS 的 `avatarhub-ingest.service` 环境加 `AH_INGEST_MANIFEST=/var/www/dl/releases/<ver>/manifest.json`（指向可写的 live manifest），重启服务。此后某版本崩溃占比超阈会自动 halt+重签（坏版本自动刹车）。
8. **转强制验签**：live 全量、且看板确认再无"无签名放行"日志（约两版后），把 `release_sign.py` 的 `REQUIRE_SIGNATURE=True`，随下一个 app 组件发出——此后客户端拒绝任何无签名 manifest（最强防篡改）。

## 回退
- 某版本有问题：`rollout_ctl.py halt <manifest>` 重传 → 停止继续扩散；已升级机器由客户端 probation/quarantine 自愈或用户「回滚上次程序更新」。
- 要全网撤版：把 manifest 的 app 组件 sha/version 回退到旧包并重签重传（客户端下次检查即回到旧版）。
- exe 出问题：live 站换回上一版 exe + release_manifest。
