# 历史页（历史 tab）三视角优化美化方案
> 2026-07-07 · 语音页 P0~P7 / 对话页 C1~C2 模式复用第三站
> 页面：`static/ui.html` 历史 tab + `static/hub.js`（后端 `avatar_hub.py` speak_history 一族端点）

## 一、三视角快评（浓缩）

**开发工程师**：
- D1 **播放器实锤坏**：`<audio :src="…/api/history/{id}/audio">` 指向的是 JSON+base64 端点，浏览器解不出声——历史页"播放"从来播不响；字节端点 `/audio.wav`（Song-P1 加的）一直闲置。
- D2 **隐形全列预取**：展开区用 `x-show` 常驻 DOM，列表渲染即给每一行建 `<audio src>`——一页 50 行就是 50 个音频 JSON 请求（每个含全量 base64），带宽与内存双输。
- D3 **深链装载缺口**：`$watch('tab')` 只在切换时触发，F5/书签直达 `#history` 时 `loadHistory()` 无人调用，列表恒空（同款坑 sing/stream 都修过，history/settings 漏了）。
- D4 **分组日期用 UTC**：`toISOString()` 切「今天/昨天」，UTC+8 的 0~8 点昨晚记录标成今天、今天的按日期平铺。

**市场经理**：
- M1 历史页是"资产库"，但单条音频拿不走（只有整包 ZIP 导出）——客户想要"这一条"时没有动线。
- M2 全链路都讲 C2PA 溯源（语音页嵌凭证、对话页可验真、验真工具独立页），资产库反而验不了——交付前逐条自证来源做不到。
- M3 唱歌记录显示原始值 `sing`，行话直接暴露给客户。

**用户**：
- U1 点开历史想听，播放器永远转不出声（D1 的直接感受）。
- U2 唱歌记录点「重新合成」，回填到语音页的是歌名/文件名，念出来是灾难。
- U3 凌晨用系统，「今天」分组里混着昨晚的记录（D4 的直接感受）。

## 二、H1 实施结果（2026-07-07 完成，门禁全绿）

| 项 | 结果 |
|---|---|
| H1-1 播放器修复（D1+D2） | 展开区 `x-show` → `template x-if` 懒挂载：点开才建 `<audio>`；src 换到字节端点 `/api/history/{id}/audio.wav`（`preload="metadata"`）。两个实锤问题一次修掉：能出声了 + 不再整列预取；JSON 端点保留给 fetch 场景（结果卡找回/验真取回） |
| H1-2 单条下载（M1） | 展开区加「⬇ 下载 wav」`<a download>` 直链字节端点——服务端 `Content-Disposition` 带 UTF-8 文件名（文本前 40 字），下载即得名 |
| H1-3 历史验真（M2） | 展开区加「🏷 验真」：`verifyHistory()` 经 JSON 端点取回 base64 → `POST /api/provenance/verify`，五档结论就地展示（✅ 本系统生成·签名有效 / ✅ 凭证有效 / ⚠ 疑似被篡改 / ⚠ 仅检出水印 / — 未检出凭证·注明"早期记录或未开水印"）。与对话页/验真工具同一口径，C2PA 链路补上资产库一环 |
| H1-4 深链装载补齐（D3） | init 末尾补 `if(tab==='history') loadHistory()` 与 `if(tab==='settings') loadMetrics()+kbRefresh()+loadHwGuide()`——与既有 sing/stream 的直达修法同一模式；顺带治了「直达设置页运行指标/决策①面板全 0」 |
| H1-5 本地日期分组（D4+U3） | `historyGrouped` 的今天/昨天改本地日期拼串（`_loc()`），截图验证：凌晨 4 点昨晚 22 点的记录正确归「昨天」（修前标「今天」+日期平铺三组） |
| H1-6 sing 人话化（M3+U2） | `emotionCN/emotionEmoji` 补 `sing:'唱歌'/🎵`（历史页徽标显示「🎵 唱歌」）；唱歌记录**不给重合成入口**（text 是歌名/文件名，回填语音页无意义）——与语音页 P2「唱歌记录不进最近合成」同一治理逻辑 |

**实施中发现并追加的方案外优化（再次优化点）：**

1. **摸底即发现两个实锤 bug（D1/D2）**——本轮原计划是"模式迁移微增强"，实际最大价值是修复"播放器从来没响过"和"隐形全列预取"；印证了"每页动手前先摸底"的工序价值。
2. **深链修复顺带覆盖设置页**——排查 D3 时发现 settings 同样漏了直达装载（运行指标/决策①面板直达时全 0），一并补齐，决策①的读数入口不再依赖"先去别的页再切回来"。
3. **验真结论五档措辞对齐对话页**——避免同一凭证在两页给出不同说法；未检出凭证的文案主动解释"早期记录或合成时未开水印"，不让用户误以为系统坏了。

**验证**：`test_ui_optimization.py` 新增 UH1 断言 7 条（终态 299/299 全过，含并行会话同窗新增的 S16/S17 组）；`gate.py` 三层全绿；无头 Edge 截图两轮对照：修复前（今天分组混昨晚记录、无下载/验真入口）→ 修复后（分组正确、🎵 唱歌徽标、展开区三动线）。纯前端改动，刷新即生效。
**同窗竞态记录**：验证期间 Tier U 两次假红均为并行授权会话（S17）写 `license_server.py`/`license.py` 中途被断言撞上，字符串落齐后自愈；一次 PowerShell 宿主 OutOfMemory 与门禁无关。

## 四、H2 巡检结果（2026-07-07 完成，门禁全绿）

按门禁清单「模式落点索引」对语音/批量/对话/历史四页 × 四类模式逐格核对。**巡检定位：只修偏差不加新功能**——三处偏差全部是"同一模式在别页有、这页缺"的对齐型修复。

### 修正的偏差（3 处）

| 页 × 模式 | 偏差 | 修法 |
|---|---|---|
| 批量 × 能力诚实 | 情感引擎离线时，语音页胶囊会灰化+给原因，批量页情感下拉却照常可选——选了带情感（或 auto 检出非普通）要逐行跑到失败才知道 | 下拉非 neutral 选项 `:disabled="!emotionSvcUp()"` + title 说明；已选非 neutral（含 localStorage 恢复的旧选择）时出橙色警示条：说清"哪些行会失败"+「去启动」跳自检页。TTS 全离线的既有警示条保留，两条互补不重叠 |
| 语音 × 结果卡动线 | 对话页/历史页的音频都能就地「🏷 验真」，**产出源头**的语音页结果卡反而没有验真入口——C2PA 链路在生产端断了一环 | 复用 `verifyHistory()`（加 `hid` 可选参，历史页原调用不动）：结果动作行加「🏷 验真」，凭 `speakHistId` 走同一 JSON 端点+`/api/provenance/verify`，五档结论行措辞与历史页逐字一致；未落库（无 histId）明说"这条还没落历史"而不是装死 |
| 对话 × 能力诚实 | 状态点黄灯 title 里写了"语音识别离线"，但麦克风/免提按钮照常可点——用户录完一整段话才收到失败，最贵的失败形态 | `pollHealth` 里新增 `_applySttGate()`：STT 全离线时两按钮 `disabled`+半透明+title 给原因与去处（"文字输入仍可"兜底动线）；录音/免提进行中不抢按钮（下轮探测再收）；`/health` 无 services 数据时不做假告警（与语音页 `emotionSvcUp()` 同一防误报口径） |

### 核对无偏差 / 有意豁免（记录在案，防止后续"补齐执念"）

- **反馈闭环 × 批量页**：豁免。批量是产线操作，行级 ▶ 试听即质检；单行不满意走「✎ 送润色」回语音页，那里有完整反馈闭环——不在批量页重复建 👍👎。
- **结果卡 × 对话页缺"下载"**：豁免。回复气泡用浏览器原生 `<audio controls>`，自带下载菜单项，不重复做入口。
- **能力诚实 × 历史页**：无偏差。历史页只读不依赖合成服务；验真按钮在溯源模块未加载时就地报 503 说明（H1 已做）。
- **持久化语义**：四页无偏差（批量 localStorage 1h / 语音·对话 sessionStorage / 历史服务端 SQLite，各配语义注释与门禁断言）。

## 五、H3 批量验真（2026-07-07 完成，条件触发项落地）

R1/UK1/UK2 收口后用户持续要求推进，触发 H2 展望里挂着的两个条件项。本轮把「历史页批量验真」做实、把「克隆历史列表」核对后记豁免。

### 1. 批量验真（做实）

**场景**：交付前挑出整批里"被改过/没凭证"的条目——单条验真（H1-3）一条条点在几十条选中时不现实。

| 层 | 落点 |
|---|---|
| 逻辑 | `bulkVerify()`：选中集过滤 `has_audio` → 2 路并发队列逐条 `verifyHistory(it)`（复用五档口径唯一实现，不另写判定）→ 汇总 `{ok,warn,none,skipped}` |
| 工具栏 | 多选工具栏新增「🏷 批量验真」，忙时按钮原地显示进度 `n/m` |
| 摘要条 | 列表上方横幅：⚠ 存疑 n 条（黄底，"先别交付"）/ 全绿则绿底；跳过无音频条数灰字注明 |
| 行角标 | 每行日期旁落 `✅ 已验 / ⚠ 存疑 / — 无凭证` 角标（title 带完整结论），不展开也能一眼挑出问题条 |
| 复位 | 退出多选清摘要；角标随 `_vrf` 留存至列表刷新（临时字段不落库） |

**并发取 2 的理由**：每条验真 = 音频 base64 取回 + 服务端 LSB 水印解码（纯 CPU），全并发会把 Hub 打卡死；串行 50 条又太慢。2 路是"不冒烟、可接受等待"的折中。

### 2. 克隆历史列表（记豁免，不重造）

摸底发现并行会话的**资产面板**（`vaOpen`，角色库页「🗂️ 资产」）已完整承接该需求：`/api/voice_assets` 列出全部克隆音（含未绑定孤儿）、试听、被谁引用、绑定找回、改名、回收站还原。再造一个"克隆历史列表"= 同一数据两套 UI，必然漂移。

**处理**：克隆页标题右侧加「🗂️ 看我克隆过的声音」桥接按钮（`vaOpen('voice')` 直开声音页签），把"我以前克隆过什么"的疑问就地引到已有面板；豁免理由记入门禁清单模式索引。
