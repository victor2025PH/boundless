## 1. 这个页面是什么、用在哪（使用场景）

**页面身份**：`/phone` 是面向**最终观众/客户**的实时 AI 数字人对话前台——与 `/ui` 控制台（自己人的作业台）相对，这是产品的"成品出口/见面页"。控制台侧栏里它是「上线」组的跳出型页签（`hub.js tabs: {id:'phone', group:'上线', href:'/phone'}`）。

**五个真实使用场景**：

| 场景 | 页面内的支撑功能 |
| --- | --- |
| ① 音色克隆交付验收 | 客户打开链接选自己的克隆角色，看相似度/自然度评分、「听我说一句」试听、「听听像不像」盲听 AB，验收后「分享」生成海报裂变 |
| ② 直播助播 / 数字人客服 | 免按键 VAD 自动断句 + 亚秒级打断（barge-in），「直播」推画面到对外直播/录制（📡 直播中角标），数字人常驻接话 |
| ③ 展会 / 门店大屏演示获客 | 待机角标「可直接说话」，观众上前即说即答；「本地运行·数据不出本机」「AI 生成·可验真伪」信任条是合规卖点 |
| ④ 跨语言口播 | 「翻译朗读」模式：说中文 → AI 用角色克隆音色以目标语种朗读 |
| ⑤ 手机 / 平板触屏体验 | 「按住说话」大按钮、竖屏布局、回声自检浮球「对方能听到吗」 |

**版块地图**（分析的事实基础）：
顶栏（状态点/品牌/分享/工作室/⋯菜单）→ 信任条 → 授权提示条（条件显示）→ 主区两栏：**左舞台**（待机头像/金标徽章/试听按钮组/示例胶囊/视频层/波形/三段状态 overlay）+ **右控制栏**（角色卡 roster→历史搜索→对话历史→底部输入堆：模式行/更多抽屉/提示/示例/语种条/文字行/麦克风/按钮行）→ 弹层（分享海报/音色调节抽屉）。

## 2. 四视角深度分析

### 2.1 开发工程师（可维护性 / 一致性）

**亮点**：全页已吃令牌体系（elev/r-md/acc-rgb）；`prefers-reduced-motion` 降级、uivr 确定性截图口径、无本地记忆时跟随 Hub active 的防呆注释都很专业。

**痛点**：
- **D1 最后一块巨石**：ui.html 已按页签拆进 `ui_src/`（P2-1/P3-1），phone.html 3655 行仍是单文件；CSS 靠 P11/P12 层 `!important` 叠罗汉，样式债在涨。
- **D2 双对话页并存**：converse.html（桌面对话页）与 phone.html 功能重叠约八成，且《对话页方案》§六已实锤 phone 功能面**超**converse（验真/择优重试/降级横幅/导出俱全）。同一模式两套形态（如 STT 门控），修 bug 要修两处。
- **D3 角色选择三个真相**：phone 记 `localStorage(ah_profile)`、converse 记 `converse_profile`、Hub 又有 active——同一台机器三处各记各的，切页角色漂移。
- **D4 术语无单源**：质量徽章文案（相似度/自然/待检/体检句数）散落在字符串模板里，与工作室/看板口径靠人工对齐，白标与改口径时是漏改点。

### 2.2 市场经理（转化 / 品牌 / 演示安全）

**亮点**：分享海报+体验链接闭环、金标音色、信任条（把合规做成卖点）、👍👎 反馈进数据面板，都已在位。

**痛点**：
- **M1 转化断头路**：这是分享链接的落地页、验收页——全站离"客户掏钱"最近的页面，但体验完**没有任何商业出口**（无「我也要克隆自己的声音」→ /order 或联系入口）。pro_upsell 漏斗（P2-2）只铺了控制台，恰好漏掉转化率最高的页。
- **M2 诊断信息对客户裸奔**：63% 红分、⚠ 质量告警、🩺 待检、📋 体检句数是**运营语言**，对最终观众/验收客户可见等于当面自曝短板；演示与分享落地场景需要"观众净化"。
- **M3 观众被引去后台**：顶栏「工作室」「音质看板」入口对观众是噪音甚至风险（点进去满屏运维信息）；分享出去的链接同样带着这些门。
- **M4 最有说服力的数字埋没**：「相似度 92%」是成交话术的核心论据，目前只在角色卡 12px 小徽章上；选中角色后舞台主视觉没有复述这个数字。

### 2.3 用户（动线 / 认知负担）

**亮点**：三步引导（选角色→说话→听回答）、免提四态胶囊、打断标记、会话恢复、语种状态条一键复位——历史批次成果扎实。

**痛点**：
- **U1 "怎么开始"要扫过 6 层控件**：底部输入堆自上而下 模式行→提示→示例→语种条→文字行→麦克风→按钮行；主动作「按住说话」压在最底部，首次打开视线要穿过全部次要控件才落到它。
- **U2 角色多了靠翻**：角色卡区无搜索/过滤/排序（对话历史反而有搜索）；角色一多，"找到我的角色"成第一道坎。质量分排序、"有声音优先"都没有。
- **U3 触屏端 tooltip 体系失效**：相似度/自然/待检的解释全挂在 `title=` 上，手机/平板无 hover——恰恰这个页面叫"对话端"，触屏是一等场景。
- **U4 高频"闭嘴"不顺手**：直播/演示中打断回复是刚需，⏹ 停止藏在底部按钮行且默认隐藏；说话中的大目标停止钮（phase overlay 已有位置）会更符合肌肉记忆。

### 2.4 高级美术设计师（视觉系统 / 细节）

**亮点**：头像质量环（相似度→环色）是好设计；字阶 P12 层已把信息文字抬到 12px 下限；待机 halo/呼吸动画克制。

**痛点**：
- **A1 功能位 emoji 的最后漏网页**：🎤 按住说话 / 🖐 免按键 / 📺 直播 / ⏹ 停止 / 🔊 对方能听到吗 / 🌐 语种条——launcher、ui.html、换脸面板已在 P3~P5 清完，phone 是全站最后一页（`<option>` 内 emoji 按公约豁免）。
- **A2 徽章色谱过载**：右栏同屏可出现 金/绿/黄/红/橙 五色徽章 + 红 ⚠ 角标，视觉噪音高于信息量；诊断色应收敛为「达标金 / 中性灰 / 异常红」三档以内。
- **A3 大屏待机的留白没干活**：左舞台空闲态重心偏上，hero 按钮组到左下浮球之间大片空白；展会立牌场景这里是二维码/slogan 的天然位。
- **A4 信任条常驻吃一行**：`bd-trustbar` 永远占一整行高度，小屏挤压舞台；可首展后收纳为顶栏两枚小徽章。

### 不做清单（明确刹车）

- 不给 /phone 加运营深度（设置/日志/体检永远在 /ui，前台保持纯粹）；
- 不动 `/profiles` 接口与质量分算法（本轮只动展示层）；
- `<option>/<optgroup>` 内 emoji 保留（HTML 无法渲染 SVG，公约豁免位）；
- 不做 phone 页的桌面宽屏三栏化（现两栏在 1280~1440 表现已达标，改版收益<回归成本）。

## 5. P0 实施记录（2026-07-16 夜 · 全部完成）

> 动手前先做了"此前未开发过"核对：全仓 grep 无 `guest/kiosk/观众模式` 页面级开关、phone.html 无任何排序/角色搜索代码、无 `/order` 链接与 `phone_cta` 埋点；落地页 `/s` 的「我也做一个」只把访客送到 /phone 就断头——三项均为真实缺口，未与既有功能重叠（唯一相邻能力=控制台 demoMode，本轮把它接为联动档而非重造）。

### P0-1 观众模式净化 ✅（实施中升级为"双档"）

- **比原方案更优的一步——单档拆双档**：原方案只有 `?guest=1`。实施时发现"演示现场（主播机上给客户看）"与"真访客（自己手机上打开分享链）"的净化需求不同——演示时主播仍需要「对方能听到吗」随身自检与观众提问面板。落地为两档：`bd-guest`（=guest 参数 **或** 控制台演示模式 `hub_demo` 同源联动，藏诊断+后台入口+露转化出口）、`bd-guest-url`（仅真访客，追加隐藏主播随身工具并**跳过其轮询初始化**，访客设备省流量）。
- 净化清单：顶栏「工作室」+ ⋯菜单「平台首页/对外直播/音质看板」（`ops-entry` 类统一门控）；角色卡 ⚠告警角标（fetch 直接短路）、🩺待检、📋体检句数、黄/红低分徽章与黄/红质量环（达标 ok/gold 才示人）；装机向导、调音横幅、readyTip 未体检警告；降级横幅**保留**（诚实告知"为什么没声音"）但去掉 /ops 后台门；直播角标不再是看板跳板。
- 角色列表过滤（分层兜底防空列表）：达标克隆音 → 有声音 → 全量；**防呆：分享链接指名的 `?profile=` 角色必须幸存**（访客就是冲 TA 来的，低分也得能对话）。
- 访客身份传递：落地页 `/s` 的 `goMake()` 跳 /phone 时自动带 `&guest=1`，海报扫码→落地页→对话端全链路净化闭环。

### P0-2 角色排序 + 搜索 ✅

- 排序权重 `_profScore`：有声音优先 → 相似度降序 → 自然度次键。**实测抓到一处初版 bug**：权重系数没拉开量级（自然度 ×10 能翻过相似度 ×100 的档位），Playwright 断言当场报"77% 排在 79% 前"——改为字典序量级（1e7/1000/1），复测全对。副收益：无本地记忆时的 `profiles[0]` 兜底从"随机首个"变成"最好的那个"。
- 搜索框：角色 >8 才显示（≤8 不占一行）；就地过滤卡片 display（不重建 DOM，保住 active 高亮与事件），命中范围=名字+简介。
- **验证方式对齐典型体量**：本机角色库 32 个，正好是"角色多了靠翻"的真实规模，搜索/排序断言都在真库上实测。

### P0-3 转化出口 + 埋点 ✅

- hero 区新增 `hero-cta`「我也要克隆自己的声音 →」直达 `/order`（新窗），仅 `bd-guest` 态显示——主播日常自用不被打扰，观众/演示态才见（转化目标人群=看的人，不是开的人）。图标走 `brand-icons.svg #i-mic`（功能位无 emoji，守公约）。
- **埋点选型的深想**：不用 `/api/share/track`（action 白名单后端写死、加新值要重启 Hub，且 `cta` 语义已被落地页占用会污染归因），改走 `/api/ui/event {ev:'phone_cta'}`——免重启、/ops 入口热度直接聚合、与 pro_upsell 同表同会话口径（sid 复用 `bd_sid`）；id 记 `guest/demo` 区分"真访客点的"还是"演示现场点的"；不记角色名（该日志公约=不含 PII）。实测点击后 `logs/ui_events.jsonl` 落盘正确。

### 实施中深挖根修的一个存量问题（本批最大意外收获）

- **现象**：像素回归 4 张 phone 图超阈，但差异图显示的是"角色区停在『加载中…』"而非新排序——且跨次双稳态（同代码 0.00%↔4.34%）。
- **定位**：对照计时实锤（HEAD 版与当前版各连拍 3 次）：`/profiles` 渲染耗时两版都在 **0.1s~6.4s 之间随机漂移**——是存量问题不是本批回归。根因=页面初始化把 `/profiles` 与七八个诊断/运维/同传代理轮询**同批**发出，HTTP/1.1 同源 6 连接位被慢端点（`/interp/*` 代理探测）挤占，第一要素反而排队。以前基线恰好都截到快相位，问题一直潜伏；本批改动扰动了请求时序才把它翻出来。
- **根修两刀**：① uivr 回归模式下所有诊断轮询（直播角标/装机向导/降级横幅/LLM 冷启动/观众提问/随身自检）全部短路——像素基线只该盯 UI 本身，这些本就是"随运行态翻转"的漂移源，还占连接池；② `loadProfiles` 改为**先占连接位**（`fetch('/profiles')` 提到 `await loadQualityAlerts()` 之前）——真实用户同样受益：角色区首屏不再被诊断请求堵到数秒。修后连拍 3 次渲染稳定 0.15s；像素回归连续两轮 12/12 全绿（第二轮最大差异 0.25%，phone 系全部 0.00%）。

## 9. 四视角复盘：「一页三面」的可理解性问题（2026-07-17 · 模式系统显性化方案）

> 触发：P0/P1 上线后，产品负责人连问两次「这个是演示用的吗 / 很难理解这个使用方法」——**当最懂产品的人都说不清这页处于哪种用法时，问题不在功能而在可理解性**。本节四视角只聚焦这一个问题，不重复 §2 的功能分析。

### 9.1 根因（先说透）

P0 给这页做出了三种"面孔"，但三个面孔**全部靠隐形开关驱动，页面上没有任何标识**：

| 面孔 | 谁在看 | 怎么进入 | 页面有提示吗 |
| --- | --- | --- | --- |
| 工作台（默认） | 主播/运营自己 | 直接打开 /phone | ❌ |
| 演示间 | 你 + 客户当面看 | **去另一个页面**（/ui 顶栏）开演示模式，回来刷新 | ❌ |
| 店面（访客态） | 分享/扫码进来的陌生人 | 链接自带 `?guest=1`，访客无感 | ❌（对访客这是对的） |

三个设计缺陷叠加：**① 开关不在现场**——演示模式的开关在控制台顶栏，对话端只能被动读取，主播开播前想切个演示得跨页面；**② 状态不可见**——切了也没有任何徽章/提示说"现在是演示态"，唯一的区别是一些东西悄悄消失了，看起来像坏了而不是像切了模式；**③ 概念没名字**——文档里叫 bd-guest/bd-guest-url/hub_demo，这是实现名不是用户语言，没人能用它向别人解释这页怎么用。

### 9.2 四视角

**开发工程师**：模式判定散落成四个布尔（`_UIVR`/`_GUEST`/`_GUEST_URL`/`hub_demo`），每处消费点各自 if——没有单一的"视图模式"模型。后果不只是难懂：客服排障时"客户说找不到工作室按钮"要靠猜 URL 参数；下批再加 kiosk 模式，散点 if 会翻倍。**该收敛为一个显式状态**：`VIEW = work | present | guest`（uivr 是测试口径不算模式），显隐规则进一张配置表，消费点只查表。

**市场经理**：「工作台/演示间/店面」的区分本来是销售资产（演示不自曝、分享有门面），但现在**销售没法操作它**——见客户前要教一遍"先去控制台点演示再回来"，等于资产锁在仓库里。要的是：现场一键切换 + 明确的"演示中"标识（销售敢确认状态才敢投屏）；访客端则相反——不要任何模式标识，店面就该看起来是唯一的样子。

**用户**：三种用法没有一句话说明。首访引导教了"选角色→说话→听回答"（怎么对话），但没教"这页可以自己用、可以演示、可以分享出去"（怎么用这页）。加一张「这页怎么用」三场景卡，比任何文档都省事——用户的疑问就是最好的需求书。

**高级美术设计师**：模式标识已有现成视觉语言可复用——/ui 顶栏演示按钮（绿点 + 「演示中」胶囊，`i-demo` 图标）是全站既定范式，对话端照搬即可，零新设计负担；注意两条克制：访客态零徽章（店面不挂"店面"牌子）、工作态也零徽章（默认态不需要解释自己），**只有演示态需要标识**——因为它是"我主动切换出来的临时状态"，需要提醒退出。
