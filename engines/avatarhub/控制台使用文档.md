# 无界运营后台 · 使用文档（完整参考）

> 地址：**https://bd2026.cc/console** · 上手引导见 `控制台使用引导_新手上路.md` · 部署运维见 `官网后台_CONSOLE_OPS.md`

## 1. 总览

后台是官网 bd2026.cc 上的一个网页应用，只跑在一台服务器上，数据也集中在服务器。特点：
- **多客服**：一人一账号一角色，谁改了什么都留审计。
- **零安装**：任意电脑/手机浏览器打开网址登录即用；所有人共享同一份实时数据。
- **签发分离**：后台只做"下单/管理"，真正给授权"盖章签名"由公司本机的**签发机**完成，私钥绝不上服务器。

角色权限：

| 角色 | 客户 | 机器 | 授权/兑换码 | 订单 | 客服账号 | 说明 |
|---|:--:|:--:|:--:|:--:|:--:|---|
| 管理员 admin | ✅ | ✅ | ✅ | ✅ | ✅ | 全权 |
| 客服 agent | ✅ | ✅ | ✅ | ✅ | ❌ | 日常客服 |
| 只读 viewer | 只看 | 只看 | 只看 | 只看 | ❌ | 查数据 |

安全：口令 scrypt 加密存储；会话是 HttpOnly 签名 Cookie（12 小时过期，改角色即失效）；登录单 IP 10 分钟 8 次失败即锁；`/console` 不被搜索引擎收录。

## 2. 各页详解

### 2.1 概览
只读看板：客户总数、注册机器、24h 在线、有效授权、7 天内到期、已吊销、未用兑换码、待签发；下方是近 14 天签发趋势柱状图 + 有效授权档位分布。

### 2.2 客户
- 字段：名称（必填）、联系方式、公司、标签、备注、归属客服（仅管理员可分配）。
- 操作：新建 / 编辑 / 删除（删除只解绑其机器与授权，不物理删授权，留痕）。
- 客服角色新建客户时，自动归属到自己名下；`?mine=1` 只看自己的客户。

### 2.3 机器（联网自动建档）
- 来源：装了产品并联网的机器自动上报（指纹 / 主机名 / 系统 / 显卡 / 显存 / 版本 / 档位 / 匿名ID / IP）。
- 在线判定：24 小时内有上报=绿点在线；顶部可筛"24h 在线 / 7 天活跃"。
- 操作：详情（看规格 + 该机的授权）、归属（挂到某客户）、直接对该机「签发授权」。
- 这是**授权/资产管理通道**，只收机器面信息，不含任何内容数据（脸/声音/视频/聊天一律不收）。

### 2.4 授权（路线A + 全生命周期）
- 列表列：序列号、机器指纹、档位、来源（离线签发/兑换码/一键发码/试用/订单）、签发→到期、状态。
- **签发（路线A）**：填机器指纹（或 `*`）、档位、天数（≤0 永久）、被授权方 → 提交后状态「签发中」→ 签发机盖章后「有效」→ 点「下载key」得 `license.key`（文件内容即"授权码"，客户导入或粘贴激活）。
- **状态**：签发中（等签发机）/ 有效 / 已吊销 / 已过期。
- **吊销 / 恢复**：吊销填原因，签发机重签"吊销名单"（CRL）经 `/api/revocations` 下发，客户机联网即失效；可恢复。

### 2.5 兑换码（路线B）
- 生成：档位、天数、**座位数**（可激活的机器数）、数量（批量）、被授权方、备注。生成后自动复制。
- 码形如 `AVH-XXXX-XXXX-XXXX`；客户在软件「在线激活」输码，经签发机换回绑本机的授权（异步，几分钟）。
- 容错：客户小写、漏连字符、带空格也能识别。
- 停用 / 启用：停用后该码不能再激活。
- 用量显示 `已用/座位`。

### 2.6 订单
- 来源：官网 `/order` 下单写入的订单（与官网、Telegram 通知同一套订单库）。
- 状态：待付款 → 已付款 → 已开通 / 已取消。
- 操作：标记已付（会私信已绑 TG 的客户）、**一键开通**（按订单指纹自动走路线A签发）、标记开通、编辑。

### 2.7 销售
- 累计营收 / 近 30 天营收（来自已付款+已开通订单）、按套餐拆分、近 30 天营收柱状图。
- **30 天内到期授权**列表：催续费的跟进名单（≤7 天高亮）。

### 2.8 客服账号（仅管理员）
- 新建（用户名+初始密码≥6 位+角色）、改密、停用/启用、改角色/备注。停用后立即无法登录。

### 2.9 审计日志
- 谁在何时、从哪个 IP、做了什么（登录、建客户、签发、吊销、发码、改订单、建/停用账号……）。

## 3. 两条发码路对比

| | 路线A（授权/key） | 路线B（兑换码） |
|---|---|---|
| 客户端要联网 | 激活时不强制（可离线导入 key） | 需要（输码在线激活） |
| 交付物 | `license.key` 文件或授权码文本 | 短码 `AVH-…` |
| 适合 | 单台、内网、精确绑机 | 批量、客户自助 |
| 后台入口 | 授权 → 签发授权 | 兑换码 → 生成 |
| 都要 | **签发机在线**才能真正出授权 | 同左 |

## 4. 签发链路（务必理解）

```
客服在后台点"签发/发码/吊销"  →  官网把任务写入"签发队列"（授权先显示"签发中"）
        ↓  （公司本机签发机每 20 秒轮询一次）
本机 sign_worker.py 用本地私钥盖章  →  回填官网（状态转"有效"，写入 license.key 内容）
        ↓
路线A：客服下载 key / 复制授权码发客户    路线B：客户输码"在线激活"，几分钟后成功
```

- 签发机命令（公司保管私钥那台电脑，需常驻）：`python sign_worker.py --watch 20`
- 私钥文件 `secrets/license_vendor_sk.pem` 只在这台机器；服务器被攻破也伪造不了授权。
- 出的授权用产品内置公钥可验签，绑机器指纹、带有效期、可吊销。

## 5. 机器不显示怎么办（排查顺序）

机器出现在「机器」页，前提是该机运行的产品会向 `https://bd2026.cc/api/register` 上报。按序排查：

1. **该机是否装了带"联网注册"的新版程序？** 注册模块 `admin_client.py` 是新增的，旧版本没有 → 需要更新该机产品代码。
2. **该机跑的是"中枢(Hub)"还是"子服务(分机)"？** 注册钩子只挂在中枢 `avatar_hub.py`。GPU 分机（换脸/STT/口型等子服务）默认不上报。
3. **临时手动登记任意一台**（最快）：把 `admin_client.py` 拷到该机产品目录，运行 `python admin_client.py` 即立刻登记该机。
4. **该机能否联网到 bd2026.cc？** 内网机若无外网，注册发不出去（可设 `AVATARHUB_ADMIN_URL` 指向可达地址）。
5. **是否被关了？** 环境变量 `AVATARHUB_ADMIN_REGISTER=0` 会关闭上报；限频为每机 6 小时一次（`python admin_client.py` 是强制立即上报，不受限频）。

> 想让 GPU 分机也自动出现：需在各子服务启动处也挂一行 `admin_client.install(...)`（可让我加），再更新到分机并重启。

## 6. 数据与备份

后台数据都在服务器 `~/hualing-leads/` 下的 JSON 文件：`console-users.json`（客服账号）、`console-customers.json`、`console-machines.json`、`console-licenses.json`、`console-codes.json`、`console-signq.json`（签发队列）、`console-revocations.json`（吊销名单）、`console-audit.jsonl`（审计）。沿用官网现有留资备份脚本即可一并备份。

## 7. 术语

- **机器指纹**：由机器稳定标识派生的一串 `XXXX-XXXX-XXXX-XXXX`，授权绑它，换机需重签。
- **座位数**：一个兑换码可激活的不同机器数量。
- **签发中**：授权已下单、等签发机盖章的中间态。
- **CRL / 吊销名单**：签名的已吊销授权清单，下发到客户机使其失效。
- **档位**：试用 / 标准 / 旗舰 / 企业，决定开放的能力与并发。
