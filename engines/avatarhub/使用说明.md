# AvatarHub · 实时数字人系统 — 使用说明

> 语音克隆 + 活体口型 + 换脸 + WebRTC 直播的本地实时数字人平台。
> 核心链路：**说话 → STT → LLM → 克隆音 TTS → 流式口型(活体) → WebRTC / OBS 虚拟摄像头**。
>
> 本文档面向「日常使用 / 运维」，开发与架构细节见 `README.md` 与 `升级开发路线图_v3.md`。

---

## 1. 一分钟上手

| 想做什么 | 怎么做 |
|---|---|
| 打开控制台 | 浏览器访问 **http://127.0.0.1:9000/ui** |
| 手机端对话 | 手机连同一局域网，访问 **http://<本机IP>:9000/phone** |
| 启动全部服务 | 双击 `launcher.bat`（图形启动器，推荐）或运行 `start_all_services.bat` |
| 只重启控制中心 | 见 §3.3（最轻量、不动 GPU 模型） |
| 一键体检 | `<facefusion>\python.exe doctor.py` 或启动器里的「一键体检」 |

> 本机 IP 示例：当前部署 Hub = `192.168.1.49`。手机端就用 `http://192.168.1.49:9000/phone`。

## 2. 系统构成（服务清单）

中央编排器 `avatar_hub.py`（端口 9000）聚合下列微服务，各跑独立 conda 环境以隔离依赖。
✅ = 实时对话核心链路（默认启动）；⬜ = 可选扩展（`set START_EXTRAS=1` 后启动）。

| 服务 | 端口 | 核心 | 说明 |
|---|---|:--:|---|
| AvatarHub 中枢 | 9000 | ✅ | 角色管理 / 编排 / API / 控制台 |
| 克隆音 TTS (Fish-Speech) | 7855 | ✅ | 零样本声音克隆（对话主链路） |
| 语音转文字 (Whisper) | 7854 | ✅ | STT |
| 口型同步 (MuseTalk+活体) | 8090 | ✅ | LipSync |
| 广播中枢 (OBS+WebRTC) | 7870 | ✅ | 虚拟摄像头 / 直播推流 |
| 换脸 | 8000 | ⬜ | inswapper + GFPGAN/CodeFormer |
| 情感 TTS (CosyVoice) | 7852 | ⬜ | 情感语音 |
| 高清口型 (LatentSync) | 8091 | ⬜ | 512 离线扩散 |
| 人脸增强 (GFPGAN) | 8092 | ⬜ | 人脸复原 |
| 唱歌 (GPT-SoVITS) | 7853 | ⬜ | 歌声合成 |
| 发型 (HairFastGAN) | 8001 | ⬜ | 换发型 |
| 可商用克隆 TTS (VoxCPM2) | 7856 | ⬜ | Apache-2.0 / 48kHz |
| 流式 STT (Nemotron3.5) | 7857 | ⬜ | 可商用 / WS 增量 |

> LLM 默认接本机 **Ollama**（如 `qwen2.5:14b`，端口 11434），可经环境变量覆盖。
> **多机分担**：部分算力服务可放到别的机器。当前拓扑（单一真相在 `cluster_map.json`）：
> 换脸 `192.168.0.104`(4070)、情感 TTS/Qwen3 `192.168.0.117`(3060)、STT/Nemo `192.168.0.140`(4070)、口型冷备 `192.168.0.198`(4070)。
> 分机搬迁 / 新机配置：打开 `/ops` →「分机部署 · 拓扑向导」，env 片段一键生成 + 拓扑 lint。
> 服务清单的「单一真相」在 `app_config.py` 的 `SERVICES`，中枢 / 自检 / 启动器共用。

## 3. 启动 / 停止 / 重启

### 3.1 图形启动器（推荐）
双击 `launcher.bat`，界面里可「启动核心链路 / 启动全部 / 停止 / 重启选中」，并实时显示每个服务的就绪状态（● 就绪 / 加载中 / 停止），可一键打开控制台、一键体检。

### 3.2 命令行启动
```bat
:: 仅核心链路（实时对话最小集），就绪后自动放行
start_all_services.bat

:: 需要换脸 / 唱歌 / 高清口型 / 情感 TTS 等扩展：
set START_EXTRAS=1
start_all_services.bat
```
脚本会先杀掉旧的同名窗口进程，再依次拉起服务，最后用 `ready.py` 探针等就绪、跑一次 `doctor.py` 自检。

### 3.3 只重启「控制中心 Hub」（最常用、最轻量）
当只改了前端页面 / Hub 配置、GPU 服务无需重载模型时，**只重启 Hub** 即可（秒级，不动显存）：
```powershell
# 1) 释放 9000 端口（停掉旧 Hub）
$p = Get-NetTCPConnection -LocalPort 9000 -State Listen -ErrorAction SilentlyContinue
if ($p) { Stop-Process -Id $p.OwningProcess -Force }

# 2) 以独立进程重新拉起（自带 env_config + 日志轮转）
Start-Process cmd -ArgumentList '/c','_launch_hub_detached.bat' -WorkingDirectory 'C:\模仿音色' -WindowStyle Minimized

# 3) 等就绪
Invoke-WebRequest http://127.0.0.1:9000/health -UseBasicParsing
```

### 3.4 停止 / 守护
```bat
python service_manager.py status     :: 查看各服务真实存活/健康（含外部拉起的）
python service_manager.py stop       :: 停止全部
python service_manager.py daemon     :: 守护模式：崩溃自动重启核心 + 管理 API :9999
```
> 注意：`service_manager.py start/restart` 适合在 **daemon 常驻**下使用；一次性 CLI 执行会在脚本退出时回收子进程。日常「拉起后即退出」的启动请用 `start_all_services.bat`（窗口进程独立存活）。

## 5. 日志与监控

所有日志在 `logs\` 目录。重点关注：

| 文件 | 内容 |
|---|---|
| `logs\hub_console.log` | Hub 控制台输出（uvicorn 访问日志 + 启动信息），重启后看这个 |
| `logs\avatar_hub.log` | Hub 业务日志（带请求 ID，含对各子服务的探针结果） |
| `logs\lipsync.log` / `vcam.log` / `fish_tts.log` / `stt.log` | 各核心子服务日志 |
| `logs\mem_watchdog.log` | 内存 / 显存看门狗（每 ~20s 一行占用快照） |
| `logs\alerts.jsonl` / `alerts_state.json` | 告警事件流 / 当前告警状态 |
| `logs\service_manager.log` | 编排器日志（daemon 模式下的崩溃重启记录） |

实时跟踪（PowerShell）：
```powershell
Get-Content logs\hub_console.log -Wait -Tail 30     # 跟随 Hub 日志
Get-Content logs\mem_watchdog.log -Wait -Tail 5     # 跟随内存/显存
```
快速排错：
```powershell
Get-Content logs\hub_console.log -Tail 400 | Select-String 'ERROR|Traceback|Exception'
```

> 提示：`/realtime/raw.jpg`、`/realtime/swapped.jpg` 返回 **503** 属正常——表示当前没有开启摄像头 / 直播会话，并非故障。

## 6. 健康自检与验收

### 6.1 就绪探针 / 体检 / 验收
```bat
<facefusion>\python.exe ready.py            :: 仅核心链路是否就绪（退出码 0=全就绪）
<facefusion>\python.exe ready.py --all      :: 全部服务
<facefusion>\python.exe doctor.py           :: 一键体检（全绿即可用）
<facefusion>\python.exe acceptance.py        :: 上线前一键体检 / 验收（已含「全链路健康」预检项 pipeline）
```
（`<facefusion>` = `C:\Users\user\Miniconda3\envs\facefusion\python.exe`）

### 6.2 全链路自检（健康 + 关键延迟 + 红旗）
一次性扫描全部服务 `/health`，测 NMT/STT/TTS 关键阶段延迟，高亮「红旗」——尤其**换脸掉 CPU（实时不可用）/ 核心服务离线 / 阶段偏慢**：
```bat
<facefusion>\python.exe selfcheck_pipeline.py              :: 人类可读全表 + 末尾机器可解析摘要行
<facefusion>\python.exe selfcheck_pipeline.py --no-latency :: 只做健康扫描（更快）
<facefusion>\python.exe selfcheck_pipeline.py --json       :: 机器可读 JSON
```

### 6.3 定时巡检 + 自动告警（7×24 无人值守）
红旗经 `alerts.py` 自动 raise/clear（去抖 + 恢复通知），落 `logs\alerts.jsonl`，可推手机（见 §6.4）。三种触发：
```bat
:: 方式一：前台常态巡检（看得见，Ctrl+C 停）
selfcheck_watch.bat            :: 默认每 120s 一轮 + 自动告警
selfcheck_watch.bat 60         :: 改 60s
selfcheck_watch.bat 60 noalert :: 干跑（不联动告警，用于演示）

:: 方式二：Windows 计划任务（后台最稳：崩溃自动重跑、可开机自启、单实例防叠加）
powershell -ExecutionPolicy Bypass -File register_selfcheck_task.ps1                   :: 注册（默认每 5 分钟）
powershell -ExecutionPolicy Bypass -File register_selfcheck_task.ps1 -IntervalMinutes 10
powershell -ExecutionPolicy Bypass -File register_selfcheck_task.ps1 -Action status    :: 看状态/上次结果
powershell -ExecutionPolicy Bypass -File register_selfcheck_task.ps1 -Action remove     :: 卸载
```
（计划任务每轮跑 `selfcheck_once.bat`，最近一次明细见 `logs\selfcheck_last.log`）

### 6.4 配置手机告警（钉钉 / 企业微信）
不配 webhook 时，告警只落文件 + 本地托盘弹窗。配一个群机器人即可手机收警：
```bat
<facefusion>\python.exe setup_alert_webhook.py             :: 交互式：看现状→粘贴URL→写入→自动发测试
<facefusion>\python.exe setup_alert_webhook.py --url <URL> :: 直接添加并发测试
<facefusion>\python.exe setup_alert_webhook.py --list      :: 看当前通道（掩码，不泄露 token）
<facefusion>\python.exe setup_alert_webhook.py --test      :: 只发测试告警验证连通
```
- **企业微信**：群 → 添加群机器人 → 复制 Webhook（`https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=XXXX`）
- **钉钉**：群设置 → 智能群助手 → 自定义机器人 → 复制 Webhook（`https://oapi.dingtalk.com/robot/send?access_token=XXXX`；安全设置加关键词如「告警」）
- 类型按 URL 自动识别；也可直接编辑 `secrets\alert_webhooks.txt`（每行一个 URL），或用环境变量 `AVATARHUB_ALERT_WEBHOOK`。

### 6.5 专项压测 / 诊断工具
```bat
<facefusion>\python.exe bench_tts.py          :: TTS 首包(TTFA)/总延迟压测（fish/qwen3/cosyvoice，服务未起自动跳过）
<facefusion>\python.exe obs_selfcheck.py      :: OBS 虚拟摄像头是否可用 + 排障指引
<facefusion>\python.exe build_faceswap_trt.py :: 预构建换脸 TensorRT FP16 引擎缓存（装了 TRT 运行库才有提速）
```

## 7. 常用操作流程（控制台内）

1. **克隆一个声音**：控制台 → 角色/声音 → 上传一段参考音频（数秒到十几秒清晰人声）→ 克隆 → 试听。
2. **创建角色（数字人）**：上传形象图 / 视频 → 绑定克隆声音 → 选口型引擎（标准/高清）。
3. **实时对话**：选好角色 → 进对话页（`/converse` 或手机 `/phone`）→ 说话或打字，数字人用克隆音 + 口型实时回应（支持打断 barge-in）。
4. **开播 / 推流**：广播中枢（7870）输出到 OBS 虚拟摄像头或 WebRTC；竖屏直播默认 720×1280。

## 8. 故障排查速查

| 现象 | 排查 |
|---|---|
| 控制台打不开 | `python service_manager.py status` 看 hub 是否运行；按 §3.3 重启 Hub |
| 某服务红/未就绪 | 看对应 `logs\<服务>.log`；多为显存不足或模型未加载完，等待或重启该服务 |
| 端口被占用 | `start_all_services.bat` 会先 taskkill 旧窗口；或手动 `Stop-Process` 占用 PID |
| 显存爆 (OOM) | 关掉扩展服务（只跑核心）；看 `mem_watchdog.log` 占用；高清口型最吃显存 |
| 远端服务掉线 | 拓扑见 `cluster_map.json`（.104 换脸 / .117 情感 / .140 STT / .198 口型冷备）；`/ops` 拓扑向导有逐机 lint；`harden_remote.ps1 -Mode verify` 复验 |
| 改了前端不生效 | 浏览器硬刷新（Ctrl+F5）清缓存；必要时按 §3.3 重启 Hub |

## 8.5 硬件分档速查表（销售/部署选型用）

> 口径：全部来自本仓库实测（2026-07 标定），写方案 / 报价 / 官网时**引用区间，勿编单点数**。
> 详细对外口径见 `官网对齐方案_BOUNDLESS.md` §3。
> **运行时版本（2026-07-06 Phase 12-E）**：控制台「设置 → 🧮 硬件档位」（`GET /api/hardware/guide`）
> 按**本机 GPU 实测**给「档位 + 每功能 推荐🟢/降档🟡/不建议🔴」结论，与本表、《销售一页纸_BOUNDLESS.md》
> 三处同源；售前报预期以部署机的 `/api/hardware/guide` 结论为准，杜绝「入门卡跑高清数字人」式过度承诺。

| 档位 | 显卡 | 能开什么 | 预期指标（实测） | 不建议 |
|---|---|---|---|---|
| **入门** | 4060Ti 16G / 4070 | 图片/视频换脸（离线成片）、声音克隆、语音客服 | 换脸单帧 hd+GFPGAN ~230–260ms；克隆零样本秒级 | 实时高清数字人、同传全家桶 |
| **直播** | 4070（分机）/ 4080 | 实时换脸直播 720p 高清脸(默认档)、1080p 超清档、虚拟背景、双人换脸(2 槽) | 3 路并发 GFPGAN 8.8fps；1080p 并发均值 ~323ms；背景抠像 ~5ms/帧(CPU) | 与 LLM/TTS 同卡混跑 |
| **旗舰** | 4090 24G | 实时数字人（标清/降档）、口播极致档(CodeFormer ~5fps) | 口型 ~17–25ms/帧 | 25fps 高清活体满档 |
| **满血** | **5090 32G** | 25fps 高清活体+口型+克隆音+14B LLM 同驻；同传全链 | 首帧 ~0.9s；对话首音(缓存) ~150ms | — |
| **集群** | 5090 + 分机(4070×N) | 上述全部 + 换脸/STT/情感 TTS 分机、口型热备 | 拓扑向导一键生成 env；显存预算 lint | — |

**开播前 3 秒自检**：控制台开播页「🎛 设备体检」（麦 SNR + 摄像头脸占比 + 虚拟声卡 → 0-100 分），红灯项会在点「一键开播」时弹出「先修再播」。

## 9. 商用部署（暴露到局域网/外网时）

本机自用可全部留空（零摩擦）。一旦把 Hub 暴露到局域网 / 手机 / 外网，按 `env_config.bat` 的「COMMERCIAL DEPLOYMENT」段开启：
- **管理面令牌** `AVATARHUB_API_TOKEN`：挡住同网段任意机器改配置 / CRUD 角色 / 克隆声音。
- **GPU 子服务令牌 / IP 白名单** `AVATARHUB_SERVICE_TOKEN` / `AVATARHUB_SERVICE_ALLOW_IPS`：保护 8090/7855 等算力口。
- **离线授权** `AVATARHUB_LICENSE_ENFORCE=1`：按机器指纹 + 有效期 + 档位控能力（只降级，绝不崩）。
  ```bat
  python license_admin.py fingerprint                          :: 客户机取指纹
  python license_admin.py issue --machine <指纹> --edition pro --days 365
  python license_admin.py status                               :: 查看授权状态
  ```
