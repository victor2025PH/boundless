# AvatarHub 部署指南

> 本指南覆盖**新机部署 → 验证 → 商用上线**全流程。深入的配置/打包细节见 `README.md`（§4 配置、§4.1 安全与授权、§5 依赖、§7 打包）。

## 1. 环境要求

- **OS**：Windows 10/11
- **GPU**：NVIDIA，显存 ≥8GB（RTX 3060 起步可用；实时高清/活体推荐 RTX 4090/5090）
  - RTX 5090（sm_120）需 **CUDA 12.8 / torch ≥ 2.7**；旧卡按显卡架构选对应 CUDA。
- **运行时**：Miniconda / Anaconda（各 AI 微服务跑在隔离的 conda 环境里）
- **LLM**：本机 Ollama（如 `qwen2.5:14b`，端口 11434），或经 `secrets.bat` 接云端

## 2. 一键准备环境（推荐）

```bat
:: 体检：列出 6 个 conda 环境与关键模型目录是否就位（只读，不改动）
provision.bat
:: 自动按 requirements\ 基线创建缺失环境并装依赖（幂等，可重复跑）
provision.bat --create
```

> 路径 / conda 根 / 各环境 python **全自动探测**（`app_config.py`），换机/换盘符/换用户名无需改代码。
> 跨机差异（模型放别的盘、非标准 conda 根）：复制 `config.example.json` 为 `config.json` 填写覆盖。
> 自检配置解析：`<某环境>\python.exe app_config.py`

## 3. 下载模型

模型与 conda 环境**不随安装包分发**，需在目标机单独就位（体积大）。参见 `MANIFEST.json`，从源机复制：
`MuseTalk/models/`、`LivePortrait/`（活体权重+idle 模板）、`CosyVoice/pretrained_models/`、
`facefusion/.assets/models/`、`GFPGANv1.4.pth`、`Retrieval-based-Voice-Conversion-WebUI/assets/weights/` 等。

### 3.1 对话大脑（LLM）交付：二选一（内部伙伴部署必读）

对话/同传功能需要 LLM。**打包体系（manifest/packs）不含 LLM**，按伙伴机网络条件二选一：

- **云端 key（默认，有外网的伙伴机）**：产品默认 LLM 已是 DeepSeek 云端（`llm_backends.json` 的 `default: deepseek`）。
  把 `secrets.example.bat` 复制为 `secrets.bat`，填 `CONV_DEEPSEEK_API_KEY=<给该伙伴发的 key>` 即完成；
  本地 Ollama 仅作断网容灾兜底，可不装。
- **本地 Ollama（离线/内网伙伴机）**：介质须附带 ① `OllamaSetup.exe`（安装程序，官网下载一次即可复用）
  ② 预导出的模型目录。**不要整拷母机 `%USERPROFILE%\.ollama\models`**（母机实测 110GB、混有 8 个实验模型），
  用导出工具按名精确导出产品所需最小集（约 9.5GB）：

  ```bat
  :: 母机：导出 产品最小集 = 对话/同传主力 qwen2.5:14b + 语义检索嵌入 bge-m3
  ::（可选加 hy-mt2-7b-official ≈4.3GB：低显存机的同传应急兜底）
  python tools\export_ollama_models.py --models qwen2.5:14b bge-m3 --out <介质>\ollama_models
  :: 伙伴机：装 OllamaSetup.exe 后，把 ollama_models 下的 blobs/ + manifests/ 合并拷入
  ::   %USERPROFILE%\.ollama\models\ （或 setx OLLAMA_MODELS 指向介质目录）
  :: 验证：ollama list 出现上述模型名；产品「设置 → 对话大脑」切到本地引擎
  ```

  显存预算：`qwen2.5:14b` 常驻约 9–10G（`OLLAMA_CONTEXT_LENGTH=4096` 已在 `deploy.env.bat` 说明），
  16G 卡与实时链路同驻会挤占，建议 24G+ 或走云端 key。

## 4. 启动

```bat
:: 方式 A（推荐）：桌面启动器，图形化一键启停 + 实时就绪状态 + 一键体检
launcher.bat
:: 方式 B：命令行一键启动核心链路（就绪后自动放行）
start_all_services.bat
:: 需换脸/唱歌/离线高清等扩展服务：
set START_EXTRAS=1 & start_all_services.bat
```

访问统一控制台 **http://127.0.0.1:9000/ui**；手机端 **http://<本机局域网IP>:9000/phone**。

## 5. 验证（交付前必跑）

```bat
python ready.py                 :: 核心链路是否就绪（退出码 0=全就绪）
python doctor.py                :: 一键联机体检（服务/角色/容量/流式就绪）
python deliver_check.py         :: 一键交付自检（预检→探活→体检→回归门禁→收益）
```

`deliver_check.py` 退出码：**0=可交付 / 1=有警告可上线 / 2=不可交付**；报告落 `logs/deliver_report.json`。

## 6. 商用上线加固（暴露到局域网/手机/外网时必做）

详见 `README.md` §4.1。要点（在 `env_config.bat` 的「COMMERCIAL DEPLOYMENT」段开启）：

- **管理面令牌** `AVATARHUB_API_TOKEN`：挡跨机改配置/CRUD角色/克隆声音。
- **GPU 子服务令牌/白名单** `AVATARHUB_SERVICE_TOKEN` / `AVATARHUB_SERVICE_ALLOW_IPS`：保护算力口。
- **离线授权激活**（对外卖授权时）：
  ```bat
  python license_admin.py fingerprint                          :: 客户取机器指纹
  python license_admin.py issue --machine <指纹> --edition pro --days 365 --licensee "客户名"
  python license_admin.py status                               :: 或 GET /api/license/status
  set AVATARHUB_LICENSE_ENFORCE=1                              :: 开启强制（默认只评估不拦截）
  ```

## 7. 容量与并发（定容量/定价依据）

实测单张 5090（A3 压测）：

- **单路对话**：首帧 STD ~1.3s / HD ~4.5s，体验优秀。
- **单卡无法真并发**：TTS/口型共用一个 GPU，N≥2 路独立并发会互相拖垮（首帧 p95 可达数十秒）。
- **正确形态**：`deploy.env.bat` 设 `CONV_MAX_CONCURRENT=auto`（单卡 K=1）+ `CONV_MAX_QUEUE=20`
  → **1 路实时数字人 + 观众问题排队串行应答**，超额路优雅排队并推送 ETA，而非一起劣化。
- **要真并发**（多路同时独立对话）：加第二张卡 / 第二台机，`SVC_LIPSYNC=url1,url2`（分发+故障切换已验证）。
- HD 直播机建议 `LIPSYNC_HD_PREWARM=1`：启动时把 GFPGAN autotune 一次性付清，首条 HD 句即满速。

## 8. 打包为安装包（可选，交付给非技术客户）

```bat
build_launcher.bat                       :: 生成 dist\AvatarHub.exe（品牌控制台）
installer\build_installer.bat            :: 生成 dist\AvatarHub-Setup-1.0.0.exe（每用户安装）
```

> 安装包只含控制台+编排脚本+前端+依赖基线+文档；**绝不打包** conda 环境 / 模型 / 机密（`secrets.bat`/`license.key`/私钥已排除）。模型与环境仍按 §2–§3 在目标机单独 provision。

## 9. Linux GPU 推理面容器化（云部署 / 无 N 卡客户）

四个纯 HTTP GPU 微服务（克隆音 TTS / STT / 口型 / 换脸）可用 Docker 跑在任意
Linux + NVIDIA 机器（AutoDL / RunPod / 客户机房），Windows 中枢通过 `SVC_*` 指向远端：

```bash
python docker/gen_linux_requirements.py                    # 从 Windows 基线生成 Linux 依赖清单
docker compose -f docker/docker-compose.yml build
AVATARHUB_SERVICE_TOKEN=令牌 docker compose -f docker/docker-compose.yml up -d
```

详见 `docker/README.md`（架构对齐、服务清单、Windows 中枢接入、已知边界）。
