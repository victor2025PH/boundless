## 常见问题

### Q1: 提示 "Invalid sample rate"
**原因**: 输入输出设备采样率不一致

**解决**:
- 确保都用 **MME** 驱动 (不是WASAPI)
- MME默认44100Hz，WASAPI可能48000Hz
- 在RVC配置中统一为44100Hz

### Q2: 有回声或延迟
**原因**: 音频缓冲区设置不当

**解决**:
- 调整RVC的 `block_time` 参数
  - 默认值 `0.25` → 延迟小但可能卡顿
  - 改为 `0.5` → 延迟大但更稳定
- 检查WiFi信号强度

### Q3: 音频断断续续
**原因**: WiFi不稳定或CPU占用高

**解决**:
- 改用USB连接 (DroidCam支持USB模式)
- 降低DroidCam视频分辨率，优先保障音频
- 关闭不必要的后台程序

### Q4: 找不到DroidCam设备
**排查步骤**:
1. 确认DroidCam Client已连接手机
2. 检查Windows声音设置中能看到 "DroidCam Audio"
3. 重启DroidCam Client
4. 重启电脑 (首次安装后需要)

## 技术实现

### 核心文件
- `phone_control.py` - 手机控制+音频桥接
- `phone_audio_bridge.py` - 音频设备检测模块
- `static/ui.html` - UI控制面板

### 关键API
```python
# 后端: RVC配置
POST /rvc/config
{
  "sg_input_device": "DroidCam Audio (MME)",
  "sg_output_device": "CABLE Input (VB-Audio Virtual Cable) (MME)",
  "samplerate": 44100,
  "f0method": "pm"
}

# 后端: 检测设备
GET /rvc/devices
{
  "input_devices": [...],
  "output_devices": [...]
}
```

### 前端调用
```javascript
// 一键配置手机音频
rvcSetupPhoneAudio() {
  const droidInputs = this.rvcInputDevices.filter(d => 
    d.toLowerCase().includes('droid')
  );
  // 自动选择MME模式的DroidCam设备
  let bestInput = droidInputs.find(d => d.includes('MME'));
  this.rvc.inputDevice = bestInput;
}
```

## 性能优化建议

### 延迟最小化
```
手机 --[WiFi]--> DroidCam --[MME]--> RVC --[VB-Cable]--> OBS

优化点:
1. WiFi使用5GHz频段 (2.4GHz干扰多)
2. 手机和PC距离路由器 < 3米
3. RVC block_time = 0.25 (最低延迟)
4. 关闭DroidCam视频预览 (仅音频)
```

### 音质最优化
```
推荐设置:
- DroidCam: 音频质量 "High"
- RVC: index_rate = 0.3 (保护原声特点)
- RVC: pitch = 0 (或根据角色调整±6)
- RVC: f0method = "fcpe" (音准更好，但CPU占用高)
```
