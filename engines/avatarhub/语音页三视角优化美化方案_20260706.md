## 〇、一句话结论

**这一页是全站功能最富的创作页（1 个主面板 + 5 张折叠工具卡），但呈现方式是"工程师的工具抽屉"：能力很强、动线很断、黑话很多。当前正踩着三个坑：**

1. **流式按钮的提示在"说谎"（说反了）。** 按钮悬停提示写着"不支持 RVC、口型渲染"（ui.html 2468），但后端 `/tts/stream_sse` 自 P13 起就**逐句自动变声**（avatar_hub.py 16217 文档串 + `stream_sse_rvc_applied` 指标）。口型确实不支持，但 RVC 那半句是过时信息——这条提示正在劝退用户使用本页最强的低延迟能力。
2. **换情感 = 静默换引擎，音色会变，UI 零提示。** 只要选了非"普通"的情感或填了指令，TTS 就从 Fish 克隆链路切到 emotion_tts 链路（avatar_hub.py 7173~7186）。用户体感是"一换情感就不像本人了"，且没有任何界面解释——这是和上一轮换脸页 CodeFormer 空转同类的"信任硬伤"。
3. **合成结果是死胡同。** 普通合成没有下载按钮（唱歌页反而有 `downloadSing`）；流式合成结果只存在内存队列，**不进历史、不可回放、不可导出**（`_write_history` 三个调用点均不含 stream_sse）；三张 A/B 对比卡给结论不给动作（赢家不能一键固定）；错误消息 4 秒自动消失（hub.js 1520）。

| 统计 | 数量 |
|---|---|
| 首屏常驻交互控件（13 情感胶囊+语言+口型+指令框+3 就绪 chips+2 CTA） | 约 21 个 |
| 折叠工具卡 / 其中音质优化卡内子工具 | 5 张 / 6 个（⓪~⑤） |
| UI 声明与后端事实不符 | 2 处（流式 RVC 提示、情感换引擎无告知） |
| 结果闭环缺口（无下载 / 流式不入历史 / A/B 无动作 / 错误自动消失） | 4 处 |
| 主 CTA 未走 brand.css 设计系统（裸用 bg-blue-700 / bg-purple-700） | 2 处 |

## 二、三视角分析

### 2.1 开发工程师视角

**痛点：**
- **UI 与后端两套事实**：流式按钮提示停留在 P13 之前的能力状态。换脸页刚为同类问题（CodeFormer 空转）付过学费，说明缺"UI 能力声明由后端驱动"的机制，而不是每次靠人肉同步文案；
- **设计系统旁路**：brand.css v2 已提供 `bd-btn-primary / bd-strip / bd-seg / bd-progress / bd-panel`，本页 CTA 用裸 Tailwind 色、五张卡自绘折叠头、字号 text-[12px]/[13px]/xs 三档混用——同一站点两套视觉语言；
- **结果无对象化**：合成产物是一次性 data URL，覆盖即丢；流式结果连整段音频都拼不出来。下载、对比、入历史、复用全部无从谈起；
- **状态多头**：speakMsg/streamInfo/speakPhase/speakOk/audioSrc 多处写入，清理靠 3 个 setTimeout 竞速，错误态被 4s 定时器无差别抹掉；
- **A/B 工具的最后一公里没接**：赢家引擎固定角色的 PATCH 接口和函数都现成（applyCloneEngineRec），只差一颗按钮。

**主张：** 能力提示后端化（探测/features 驱动，灰化+原因）；合成结果统一为会话内结果对象列表；CTA/面板/提示条全量走 bd-* 组件；错误常驻+重试；A/B 卡补动作按钮。

### 2.2 市场经理视角

**痛点：**
- **首次价值时刻（TTFW）无护送**：新用户面对空文本框 + 13 个情感 + 两个分不清的按钮。唱歌页都有"示例歌词快填"，创作主力页反而没有；
- **王牌能力全部匿名**：AI 情感自动分析非默认、首句亚秒级响应只在完成 toast 一闪而过、C2PA 内容凭证埋在代码里、双引擎自动打分折叠在第四张卡——演示/截图/口碑传播全无抓手；
- **黑话劝退**：RVC、TTS、seed、cosine、holdout、best-of-N、vcam、RTMP……首屏和折叠卡标题都在讲管线，不讲结果；
- **场景断裂**：合成成功后没有"下载 / 存素材 / 去开播 / 生成口型视频"的下一步，创作→直播的核心商业闭环在这一页断开。

**主张：** 示例场景一键填（带货开场/知识口播/晚安电台/英文自我介绍）；完成态给四个下一步动作；折叠卡按结果重命名（见术语对照表）；把"首句 <1s""AI 听懂情绪""双引擎盲测"做成可见徽章；音质优化包装成"让声音更像本人"的一键仪式。

### 2.3 用户视角

**痛点：**
- "**两个按钮我点哪个？**"——合成 vs 流式(多句) 是管线命名，选错的代价（没口型/等更久）事前不可见；
- "**换了情感就不像我了**"——引擎静默切换，没人告诉我为什么；
- "**合成完怎么保存？上一条去哪了？**"——播放器被覆盖，找不到下载，流式的更是听完就没；
- "**报错一闪就没了**"——4 秒后连错误原因都看不到，只能重来；
- "**打完字按 Ctrl+Enter 没反应**"——快捷键在文本框里失效，而我焦点就在文本框里；
- "**音质优化那六步看不懂**"——我要的是"一键更像我"，不是 seed 和 holdout。

**主张：** 一个主按钮 + 一个模式开关；会话内结果卡可回放/下载/收藏；错误常驻+一键重试；情感默认自动、常用前置、其余折叠；音质优化改成"贴合度分数 + 一键提升"漏斗。
