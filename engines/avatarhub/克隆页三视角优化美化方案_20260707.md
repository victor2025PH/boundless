# 克隆页三视角优化美化方案（R 线）

> 2026-07-07 立项。语音（P0-P7）→ 对话（C1-C2）→ 历史（H1）→ 四页巡检（H2）收口后，按"新用户第一入口"优先级开 R 线。
> 工序沿用既定打法：**先摸底找实锤 bug → 修偏差/对齐模式 → 门禁固化**。
> 摸底范围原定"角色库+克隆"两页；实查发现角色库页由并行会话持续维护（Hero/三步向导/资产巡检/全员体检等 P0-P4 标记齐全，且新建表单已有重名 confirm 保护），**R1 收敛到克隆页 + 一处全站导航一致性**，不与并行线重叠。

## 一、摸底：三视角问题清单

**开发工程师**：
- D1 **试听区从未出现过（实锤）**：成功卡「🎧 试听克隆效果」绑定 `cloneResult.audio_b64`，但 `/api/voice_clone`（voice_clone.py）返回键是 `voice_b64`——`audio_b64` 压根不存在，`x-show` 恒 false，这块 UI 上线以来一次都没渲染过。
- D2 **克隆自动建角色静默覆盖同名角色（实锤，数据损失级）**：`POST /profiles` 对同名是**整体覆盖**（`_profiles[p.name] = {...}` 无条件写入，face/引擎/RVC 设置全部换成新值=基本清空）。「新建角色」表单已有重名 confirm 保护（并行会话加的），但克隆页勾选「自动创建角色」走的 `doClone()` 直接 POST，**无任何确认**——用户拿旧角色名克隆一次，精心配好的形象和引擎设置就没了，不可撤销。
- D3 **`tab=` 直赋值绕过 `goTab()`（全站 8 处）**：`goTab` 里做三件事——写 `location.hash`、记 `hub_tab`/`visitedTabs`、**离开批量页停掉行内试听/连播**（UI-P7-2 保护）。克隆成功卡「查看角色库」、`cloneUseInVoice()`、`reSynthesize()`、创建中心跳克隆、同传两处、语音页「前往角色库」、`?open=tune` 深链共 8 处直接 `tab='x'` 赋值，全部绕过：F5 回到旧页、命令面板"最近"失真、批量连播可能背景续播、懒挂载内容可能不渲染（visitedTabs 漏记）。

**市场经理**：
- M1 克隆是转化链第一跳，D2 的覆盖事故一旦发生在演示/交付角色上，是"最贵的一次点击"——信任损失远超功能价值。
- M2 成功卡承诺"试听克隆效果"却永远不出现（D1），新用户第一次成功就遇到承诺落空。
- M3 引擎推荐（Fish/Qwen3 对比）失败时静默消失：「正在对比…」转完就没了，不解释、不给替代动线——语音服务离线时每个新用户都会遇到。

**用户**：
- U1 克隆完成想马上听一下"保存了什么"，界面上没有任何可听的东西（D1 的直接感受）。
- U2 拿熟悉的名字再克隆一次（很自然的行为），旧角色被清掉且无提示（D2 的直接感受）。
- U3 点「查看角色库」后刷新，回到了别的页（D3 的直接感受）。

**核对无偏差项（记录在案）**：克隆链路（质检/声纹/水印）全部本地计算，不依赖 TTS 等外部服务——**无需**加"能力离线"警示条（与四页巡检的 readiness 模式不冲突，诚实原则=没有依赖就不告警）；录音上限 30s 对齐质检门槛、电平表/静音提醒、拖拽/转码链路均已完备（前几轮并行会话产物）。

## 二、R1 实施结果（2026-07-07 完成，门禁全绿）

| 项 | 结果 |
|---|---|
| R1-1 重名覆盖保护（D2/M1/U2） | `cloneNameTaken` getter（实时比对 profiles）+ Step 2 内联警示条：明说"形象、引擎选择、变声设置都会被清掉且不可撤销"+ 建议改名；勾选「我确认要覆盖这个角色」（`cloneOverwriteOk`）才解锁「开始克隆」，改名即重置勾选。`doClone()` 内同条件兜底 return。选内联条不选 confirm 弹窗：与同页削幅放行（`recOverride`）同款交互，向导流里弹窗打断感更差 |
| R1-2 goTab 全站统一（D3/U3） | 8 处 `tab=` 直赋值全部改 `goTab()`：克隆成功卡、`cloneUseInVoice`、`reSynthesize`、创建中心→克隆、同传两处、语音页无角色提示、`?open=tune` 深链（这处原本还漏记 visitedTabs，懒挂载内容可能不渲染）。init 的「同步定 Tab」块保留直赋值（deliberate：hash 已就位、有自己的 visitedTabs push、避免开屏闪动） |
| R1-3 成功卡诚实化（D1/M2/M3/U1） | 试听区改绑真实存在的 `voice_b64`，标题如实改为「🎧 回放参考音（已嵌 AI 水印的原录音）」——不再冒充"克隆效果"（真效果试听走「用它合成一句」/引擎对比）；引擎对比失败新增 `cloneEngineRecErr` 降级文案："引擎对比暂不可用（语音服务未就绪），可稍后在角色编辑里手动选引擎"——转圈之后不再静默消失 |

**实施中发现并追加的方案外优化（再次优化点）：**

1. **`?open=tune` 深链的 visitedTabs 漏记**——排查 D3 时发现该处不止 hash 不同步，还漏 push visitedTabs（懒挂载内容可能空白），goTab 一并治好；这是"统一走公共入口"比"逐处补字段"优越的实证。
2. **「回放参考音」按需加载**——`preload="none"`：成功卡自带 base64 音频，避免挂载即解码。
3. **警示条不反向隐藏**——勾选确认后警示条保持可见（只解锁按钮），用户在点「开始克隆」前始终能看到自己确认了什么；门禁断言固化这个语义。

**验证**：UR1 断言 4 条入 `test_ui_optimization.py`（319/324 全过，跳过 5 条为 HTTP 探活）；`gate.py` 三层全绿（Tier A 223 文件 / Tier B 151 PASS + 授权契约 70/70 / Tier U 319）；无头 Edge 截图 `#clone` 深链直达渲染正常。纯前端改动，刷新即生效。

## 三、R2 展望（下一阶段建议）

1. **克隆页方向观察**：R1 修掉实锤后，三步向导本体（质检→协议→结果）已闭环；剩余候选——克隆历史列表（近期克隆的音色一览+试听+删除，数据在 `voices/clone/` 目录有、无 UI）价值中等，建议随用户反馈触发。
2. **角色库页避让并行线**：该页由并行会话活跃维护，R 线不进入；若并行线收口后仍有模式缺口（按门禁清单索引表核对），再开补轮。
3. **决策①拍板**（全局持续项）：等重启窗口+一周数据。
4. **全站 `tab=` 直赋值回归护栏已立**：UR1 断言 `@click="tab=` 清零 + goTab 关键落点，后续新页面/新入口自然被门禁拦住，不需要再人工巡检这一项。
