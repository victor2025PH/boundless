## 〇、一句话结论

**这个页面的"功能力"已经是产品级（P0~P13 全部落地），短板不在缺功能，而在三件事：危险操作零确认、专家信息未分层（监控/守护/统计混在一行）、视觉语言收敛只走了一半（bd-\* 体系与散拼样式并存、同屏彩色过载）。** 本方案不新增后端能力，纯前端「收敛 + 分层 + 降噪」，全部改动可对照门禁清单逐项落地。

## 二、三视角分析

### 2.1 开发工程师视角

**痛点：**
1. **样式决策仍大量内联**：tone→颜色的三元映射在 ≥8 处重复手写（就绪度 949~965、主裁决 780~794、标定结果 1600、设备体检 1784~1791、试音结果 2042 起……）；`rounded-lg border border-hub-orange/45 bg-orange-900/15` 这类一次性色组合全页仍有约 30 处——bd-strip 四态已定义（brand.css 178~182），但只替换了英雄区。
2. **巨型内联表达式**：如画质体检结果行 `x-text`（ui.html 1817）单表达式 300+ 字符，逻辑该下沉 hub.js。
3. **轮询无页面可见性门控**：预览开着时两路快照每 160ms 刷一次（hub.js 4109，≈12.5 req/s × 2）、信号轮询 1.1s 一次（4117）——`grep visibilitychange` 全库为 0，切去别的浏览器标签页照烧不误。
4. **实时画面卡职责过载**：预览 + 健康灯 + 监控阈值 + 自动化开关 + 告警配置 + 性能统计 + 复盘时间线 + 摄像头，全在一张卡（1754~1975）。监控/守护是独立关注点，却是全页唯一没被面板化的专家区块。
5. **窄屏(<1280px) 专家模式 DOM 顺序倒置**：右列（效果调节/视觉效果，1485）在 DOM 里先于左列（实时画面，1751），单列时设置面板把预览挤到第二屏。
6. **计数类门禁脆弱**：`bd-card − bd-card-title == 35`、`bd-card-title == 27` 是精确等值断言，每加一张卡都要重基线——改造摩擦的主要来源在测试侧而非代码侧。

**主张：** 收敛而非重写。tone 映射进 hub.js 单源函数 + bd-strip 文本继承规则；「监控与守护」抽成 bd-panel 对齐既有体系；轮询挂 visibility 门控；窄屏调 DOM 顺序；计数断言改为分区/下限式（测试侧一次性优化）。

### 2.2 市场经理视角

**痛点：**
1. **硬核卖点无叙事**：脸区原生通道清晰度 4.5×（P2 实测）、自愈体系、稳定性账本，页面上呈现为 `frame-v2 85ms / avg 87ms`——这些词做不了 demo、写不进朋友圈。「按住看原图」是全页最强的演示交互，却藏在悬浮 title 里。
2. **「失败率 16%」红 chip 裸露**（统计行 1872）：口径是自进程起累计、含离席时段——成绩单已做离席拆账（有效失败率），实时行没跟上。用户截图传播"失败率 16%"就是一次口碑事故。
3. **成绩单只能复制纯文本**：没有可传播的品牌化产物（对比图/分享卡）。
4. **常驻琥珀噪音**：「未配置 webhook · 仅本地弹窗」（1858）对单机用户是永久性的"我没配好"暗示，不可关闭。
5. （远期）六档画质预设是商业分层的天然钩子，UI 尚无 license 感知——单机版暂不需要，代理版留意。

**主张：** 预设 hint 带实测数字（"高清脸 · 脸区清晰度实测 ~4.5×"）；实时失败率换有效口径；预览工具条加「📸 导出前后对比图」（canvas 拼图 + 品牌角标，纯前端）；守护能力具名化（"智能守护"徽章，悬停展示本场自愈次数）；webhook 提示降为 muted 且可关闭。

### 2.3 用户（主播）视角

**痛点：**
1. **两个最危险的按钮零确认**：「⏹ 停止」直接停播（hub.js 2724 `oneClickStop` 无任何拦截）；在播时点主 CTA「🔄 重新开播」直接重启画面（3775，`guardedStart` 的体检门禁只管未播分支）。直播中手滑 = 事故，这是全页风险最高的交互。
2. **用户模式没有"面子功能"入口**：虚拟背景/离席画面在专家「视觉效果」面板里（1610，随 1485 整列 `x-show="!streamSimple"` 隐藏）——主播想开个背景虚化，必须切专家模式面对全部参数。
3. **阈值行是全页认知负担之最**（1822~1864）：监控数字（高/中延迟、失败率 3 个裸输入框）+ 自动化开关（自动降档/服务端自愈/本端自愈 3 个 checkbox）+ 告警（开关+测试按钮+webhook 徽标）三类东西混排一行，没有分组、全是黑话。
4. **统计口径引发焦虑**：「成功 170 · 失败 33 · 失败率 16%」无时间口径说明（自进程累计），和成绩单的"有效失败率"对不上号——同一页面两套数字。
5. **同屏彩色过载**：绿/橙/红/蓝/紫/琥珀 chips 与提示条常驻 5~8 种，全是重点=没有重点；部分驻留提示（webhook、标定过期）不可关闭。
6. **播中看不到声音状态**：注意力在预览卡上，但"有没有声"只在英雄区音频胶囊——预览卡自身没有电平指示。

**主张：** 危险操作轻确认（不弹窗，内联二次点击）；用户模式给背景/离席一行轻入口；阈值行预设化+分组+人话；统计口径与成绩单对齐；预览卡集成迷你电平；色彩降噪三原则（见 §4.3）。

### 4.3 色彩降噪三原则

1. **语义色只表状态**：绿=健康、橙=需注意、红=事故、蓝=进行/信息；装饰一律 muted 或品牌蓝。紫色仅保留 RVC 标题（门禁 P9 要求保留 `text-hub-purple font-bold text-sm`），紫色填充按钮改描边。
2. **同屏彩色 chip ≤4**；任何常驻（>30s）提示必须可关闭或降为 muted。
3. **主 CTA 渐变绿是全页唯一大面积强调色**，其余按钮不用填充色，保证"开播"永远是视觉焦点。

## 五、护栏与验收

### 5.1 实施护栏（每一步动手前对照）
