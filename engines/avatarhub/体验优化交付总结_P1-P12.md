# 前端体验优化 · 交付总结（P1–P12）

> 一轮"深入思考 → 安全实施 → 严格验证"的前端体验优化。主线：**降噪、平实化（去术语）、克制、增加价值感/信任感**；后期（P11–P12）进一步**打通真机验证闭环**并把成果**用双向回归护栏钉死**。
> 本文件是可复盘交付物：每阶段"改了什么 / 为什么 / 怎么回滚"，加跨页一致性清单、验收脚本与自守护网。

## 二、逐阶段详情

### P1 · `ui.html`/`hub.js` — 降噪（告警单一真相）
- **改**：`hubAlerts()` 合并/去重/排序告警，健康卡去冗余，用户模式把步骤详情"人话化"（`humanizeStepDetail()`）；新增 `alertHubOpen` 状态与 `stepLabels`。
- **为什么**：同一问题多处重复报警 = 噪声；用户看不懂 ops 术语。
- **回滚**：移除 `hubAlerts()`/`humanizeStepDetail()` 及 `alertHubOpen`，告警区恢复直出。
- **附带**：修正 `test_ui_optimization.py`（合并 `ui.html`+`hub.js` 做存在性检查；两处硬编码计数重新基线）。

### P2 · `ui.html`/`hub.js` — 在播价值条
- **改**：在播横幅加"在播时长 + 🔒本地运行"信任徽标；用户模式把开播步骤日志折叠成摘要（失败步骤恒显）；在播时精简 preflight 药丸。新增 `streamStepsOpen`、`streamClock`、`streamDurStr()`。
- **为什么**：把"正在发生什么、安全吗"讲清楚，压掉过程噪声。
- **回滚**：删 `streamDurStr()`/`streamClock`，横幅恢复无时长；展开步骤日志。

### P3 · `ui.html`/`hub.js` — 场次小结卡
- **改**：停播后出现可关闭的"本场小结"卡（时长 / 峰值 FPS / 是否用过变声）。新增 `sessPeakFps`、`sessUsedRvc`、`lastSession`。
- **为什么**：给操作者一个"这场干得怎么样"的收尾正反馈。
- **回滚**：删小结卡模板与三个状态变量。

### P4 · `ui.html`/`hub.js` — 小结可复制
- **改**：小结卡加"复制小结"按钮 → `copyRecap()`（镜像 `copySelfCheck()`），带 `aria-label`。
- **为什么**：便于交接/汇报，低成本高感知价值。
- **回滚**：删按钮与 `copyRecap()`。

### P5 · `ui.html`/`hub.js` — 修 bug + 无障碍
- **改（bug）**：场次计时原在 `signalTick()`（只在开播 tab 跑）→ 换 tab 会算错。改为全局 `$watch('perf.streaming')` 侦测起止，新增 `sessStartTs` 作为唯一起点真相，`streamDurStr()` 改用它（tab 无关）。
- **改（a11y）**：告警关闭按钮补 `aria-label="关闭该告警"`。
- **回滚**：把起止逻辑移回 `signalTick`、删 `sessStartTs`；移除 aria-label。

### P7 · `phone.html` — 定向降噪 + 正反馈
- **改**：手动接通实时画面成功时 `showToast('实时画面已接通 · 数字人已上镜','ok')`；`rag`（知识库）阶段不再往顶栏推片段/未命中，改由 `#kbLive` 字幕承载；`session_compress` → `正在整理对话…`；调参面板英文名（`temperature/top_p/repetition_penalty`）移入 `title`，界面显示"情感强度/多样性/咬字清晰度"；导出会话 → `会话记录已开始下载`。
- **为什么**：补正反馈（接通成功）、砍重复顶栏噪声、藏工程参数名。
- **回滚**：还原上述字符串；把英文名从 `title` 移回可见标签。

### P8 · `converse.html` — 内部测试台，最小改动
- **审计更正**：此页是 `/converse` **内部对话测试台（研发/联调用）**，非公开访客页 → 刻意**保留**技术术语与毫秒指标（对研发有用）。
- **改（仅 2 处）**：① 信任徽标 `C2PA 可验真 ↗` → `可验真伪 ↗`（与其它页对齐）；② 删除 `addUser('🎤 (识别中…)')` 占位气泡——`stt_done` 会补带识别文字的用户气泡，避免同句出现两个气泡（"处理中"由 `micHint` 承担）。
- **回滚**：还原徽标文案；把占位 `addUser('🎤 (识别中…)')` 加回。

### P9 · `landing.html` — 失效链接兜底 + 暖化报错
- **改（发现性）**：分享的角色不存在时，若还有其它角色 → `showGallery()` 转橱窗让访客挑选，别让链接扑空；一个都没有才显示 `这个声音暂时找不到了`。
- **改（暖化）**：`加载失败`→`加载遇到点问题，刷新重试一下`（`loadProfile`/`showGallery`）；`点击播放被拦截，请再点一次`→`再点一次「听我说一句」就能播放`；`该片段暂不可用`→`这段暂时听不了，试试另一个`。
- **为什么**：访客页，冷硬报错=流失点；死链应转发现而非死胡同。
- **回滚**：`const p=list.find(...)||list[0]` 复原为旧兜底；逐条还原报错字符串。

### P10 · `home.html` — 前门，克制到几乎不改
- **审计结论**：首页**已系统性贯彻同一套哲学**（术语翻译 `SHORT={fish_tts:"语音合成"…}`、不以负面打头、永不白屏、人话归因引导）。**主动不 churn**。
- **改（仅 1 处，a11y）**：版位按钮点击弹出原生 `<dialog>`，补 `aria-haspopup="dialog"`（放在 `class="edition"` 之后以保住回归断言）。
- **主动否决**：① 不把 `内容可验真 · C2PA/水印` 里的 C2PA 去掉——首页是 B2B 前门，C2PA 是可信度资产（与消费者页 `可验真伪` 属**刻意按人群差异化**，非 bug）；② 不暖化极罕见的功能加载失败文案（已带兜底 CTA，再改是过度润色）。
- **回滚**：删 ` aria-haspopup="dialog"`。

### P11 · `hub.js` / `ui_visual_regress.py` — 打通真机验证 + 修两处真实缺陷
- **背景**：发现环境其实可验证（Hub 跑在 `:9000`；装有 Chrome/Edge + 项目自带 `ui_visual_regress.py` 截图回归、`_fe_smoke.py` 零 pageerror 冲烟）。据此做**看图审计**而非拍脑袋重构。
- **先证伪一个"疑似 bug"**：以为 390px 手机顶栏 `工作室`/`⋯` 被裁切；用 Playwright 实测 `overflow=0`、控件全 `inView`——**不是 bug**（按钮 ≤430px 正确收成纯图标）。避免了无谓"修复"。
- **改（bug 1·产品）**：`hub.js` 初始 Tab 恢复逻辑原在 `init()` 多个 `await` 之后 → 刷新时"先闪角色库再跳回上次 Tab"。**提到 await 之前同步敲定**（`this.tab` + `visitedTabs` 同步入列），消抖动，且 `/ui#clone` 等深链首帧即命中。
- **改（bug 2·工具）**：`ui_visual_regress.py` 靠替换 `ui.html` 里内联 `tab:'profiles'` 切 Tab，但该状态早已迁进 `hub.js`（P1 重构）→ 替换长期空转，`clone/sing/settings` 三图**一直在截默认 Tab**（安全网破洞）。改用 `/ui#hash` 直选（配合上面的同步定 Tab），覆盖恢复。
- **验证**：`node --check` OK；`_fe_smoke.py` `/ui /phone /ops` 3/3 零 pageerror；重截图后三 Tab 显示正确（settings 0.46% 匹配基线）；`test_ui_optimization.py` 全过。随后**刷新像素基线**（11 张，`windows-edge149`），确认性复跑 **11/11 PASS**。
- **回滚**：`hub.js` 删"同步定 Tab"块即恢复；`ui_visual_regress.py` 单函数还原；基线可 `--update-baseline` 重采。

### P12 · `test_ui_optimization.py` — 双向回归护栏（钉死 P6–P10）
- **改**：新增 7 条 `PL`（平实化/信任/无障碍）护栏并载入 `landing`，覆盖 P6（阶段人话 + 无引擎名外泄）、P8（converse 验真措辞）、P9（landing 死链转橱窗 + 暖化报错）、P10（home 弹窗无障碍）。
- **关键点·正反双向断言**：不止断言"新串存在"，更加**负向断言**（`C2PA 可验真`/`Whisper 语音识别`/`LLM 生成回复`/`Fish-Speech 克隆合成` **not in**）——术语一旦重新外泄立即红灯。落笔前逐条 grep 核实正/负串与当前文件一致。
- **验证**：`test_ui_optimization.py` **111/116 通过**（5 项离线 HTTP 跳过），无 FAIL；Lint 0。
- **回滚**：删 `PL` 整段（含 `landing = read(...)`）。

## 三、跨页一致性清单

**信任/验真措辞（刻意按受众分层，而非一刀切）**
- 消费者/操作者客户端（`phone.html`、`converse.html`）：`AI 生成 · 可验真伪` —— 去 C2PA 术语。
- B2B 前门（`home.html`）：`内容可验真 · C2PA/水印` —— 保留 C2PA 作可信度背书。
- 验真工具页（`verify.html`）：全程保留 C2PA/CAI/Ed25519 等 —— 专业工具页，术语即准确性。

**平实化模式（全页统一遵循）**
- 不回显引擎名/协议名（Whisper/LLM/Fish-Speech/WebRTC/RTMP/StreamOut）给非技术用户。
- 不以负面打头；能给"一个可点按钮"就不留死胡同。
- 报错=人话 + 出路（刷新/换一个/联系支持/转橱窗）。
- 工程参数（temperature 等）藏进 `title`，界面用意象化中文。

**刻意**不统一**的**：`converse.html` 保留研发术语与毫秒指标（内部工具）；各页错误文案措辞随场景微调（不强行同字）。

## 五、未决 / 需你参与的后续

1. **结构性精简（P11 已侦察，暂不建议动）**：用真机截图审计 `/ui`、`/phone` 多分辨率后，未见明显"杂乱"或布局 bug，页面信息层级本就干净。故**不做**投机式重构（会给跑着生产服务的系统平添风险）。若你在某具体页/流程有实际别扭感，指出来即可用现有闭环定点改。
2. **人话文案下沉到后端单一真相**（如 `/api/setup/status` 步骤标题、告警文案）——让所有页面自动一致，而非逐页改前端。**仍是最高杠杆的未决项**。
3. **可视化基线随服务态维护**：`settings`/`sing` 截图含实时服务在线态，服务常态若变，对这两张 `--update-baseline` 重采即可（非回归）。
4. 若换全新目标（性能 / 全站无障碍审计 / 新功能），另定方向。

> 全轮改动均"可单点回滚、不破坏既有回归护栏"。P1–P10 平实化/降噪 + P11 验证闭环 + P12 双向护栏后，客户端页面的安全优化已**闭环且自守护**，继续逐页改文案进入明显边际递减——下一步价值更高的是"文案下沉后端"或你指定的实际痛点。
