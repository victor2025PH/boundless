# 购买页 / 下载页 · 三视角分析与优化方案（2026-07-11）

> 对象：今天上线的 `/order`（购买与下单）、`/download`（下载客户端）、首页客户端引流区块。
> 本次已把两页从「独立静态 HTML」重构为**官网原生 Next.js 页面**，与首页同一套版式体系。

## 一、本次重构做了什么（对比早上的静态版）

| 维度 | 早上静态版 | 现在（官网原生版） |
| --- | --- | --- |
| 版式 | 独立 brand.css，与官网两套皮 | 与官网同一套：暗色霓虹 + glass 玻璃卡 + 渐变主按钮 + framer-motion 入场 |
| 白天模式 | 不支持 | 自动继承官网 `data-mode="day"` 晨光主题 |
| 双语 | 仅中文 | 跟随官网语言切换（中/英全量文案） |
| 导航 | 无 | 官网 Navbar/Footer，购买、下载入首页导航高亮 |
| 下单 | 写引擎端 JSONL（官网没跑引擎→只能降级 TG） | 官网原生 `/api/order`：订单入 CRM + 管理员 Telegram 实时通知 + orders.jsonl 流水 |
| SEO | 无 | metadata + canonical + schema.org（Product/Offer、SoftwareApplication）+ sitemap + IndexNow |
| 埋点 | 无 | 全链路 track：档位点击/月年切换/打开结算/提交订单/下载点击（漏斗可算转化率） |

## 二、三视角分析

### 开发工程师视角（可维护性 / 正确性）

- **定价单一真相**：五档会员 + 9 项按量 + 5 项授权全部收敛到 `lib/avatarhub-pricing.ts` 一个文件，双语文案同源，改价改一处。与官网原有服务类 SKU（`lib/pricing.ts`）物理分开，互不污染。
- **订单闭环接对了地方**：`/api/order` 复用官网留资 CRM（`lead-store`），订单即最高意向线索——管理员 TG 秒收通知、`/admin` 后台可跟进状态、`orders.jsonl` 独立流水可对账。不再依赖「官网没跑的引擎后端」。
- **下载清单仍单一真相**：页面读 `/releases/release_manifest.json`（Nginx 静态目录），发布新版只需跑 `gen_download_manifest.py` + `upload_release.py`，前端零改动。
- **遗留风险**：① 收款地址未配置（`NEXT_PUBLIC_USDT_ADDR` 空时页面引导找客服拿地址，防伪冒但多一步摩擦）；② 订单无状态查询接口（付款后用户只能等通知）；③ 引擎侧 `static/order.html` 与官网页并存，长期应废弃引擎侧一份。

### 市场经理视角（转化 / 定价叙事）

- **自助漏斗成型**：以前唯一路径是「加 TG 问价」；现在 首页 CTA → 下载试用（14 天免费）→ 到期转付费 → /order 自助下单，深夜流量不再流失。锚定提价（旧价划线 78→99、198→249）+ 年付送 2 个月 + 首年 8 折的紧迫感齐备。
- **Pro 档位聚焦**：默认选中 Pro（249）+「最受欢迎」徽标，利用默认效应抬客单价；免费试用档承接犹豫用户，不再用 18 USDT 收体验费挡在门口。
- **99 USDT 远程代部署**上了购买页（含预约按钮直达 TG），把「装不好」这个最大流失点变成一个高毛利 SKU。
- **可量化**：埋点覆盖 期次切换→选档→打开结算→提交 全漏斗，一周后即可看数据调价、调默认档。
- **待做**：① 下载页/购买页目前无成交案例与用户证言（信任要素弱于首页）；② 无 UTM 专属落地承接（TG 频道帖直链 /order 时看不到针对性文案）。

### 用户视角（体验 / 信任）

- **不再跳出**：从首页任意位置到付款指引全程不离站，语言、明暗模式、视觉风格零切换成本。
- **价格透明可算**：月/年一键切换实时重算、旧价划线、首年 8 折折后价直接标出，不用心算；按量表和授权表把「超出会员额度花多少」讲清楚。
- **下载省心**：平台卡片给足决策信息（版本/体积/SHA-256/硬件要求），Mac 用户明确知道「轻量控制台」的定位不会误装；三步上手指引管理预期。
- **安全感**：地址防伪提示、C2PA 水印说明、克隆合规声明都在付款动线上；订单提交成功给单号，有凭据可追。
- **仍有摩擦**：① 机器指纹要去客户端里找（试用用户第一次会懵）；② 付款后无订单状态页，只能被动等通知。

## 三、后续优化与美化路线（按 ROI 排序）

| 优先级 | 事项 | 说明 |
| --- | --- | --- |
| P0 | 配置真实 TRC20 收款地址 | `.env.local` 加 `NEXT_PUBLIC_USDT_ADDR` 重新部署，结算弹窗直接展示+一键复制 |
| P1 | 订单状态自助查询 | `GET /api/order?id=AH-…`：待付款/已到账/已开通三态，付款焦虑归零 |
| P1 | 下载页加动效演示 | 15 秒换脸/克隆对比 GIF/WebM 各一条，下载转化预计明显提升 |
| P1 | TRC20 到账半自动核销 | TronGrid 轮询比对金额+单号 → 管理员一键签发，开通从小时级到分钟级 |
| P2 | 购买页信任区块 | 3 条用户证言 + 已服务客户数 + 可验真产物示例（复用首页 RealProof 风格） |
| P2 | 结算弹窗生成收款二维码 | 已有 qrcode.react 依赖，手机钱包扫码付，少一次复制粘贴 |
| P2 | /order /download 英文独立路由 | 加 /en/order /en/download（SEO hreflang 成对），承接英文搜索流量 |
| P3 | 指纹自动带入 | 客户端内「购买」按钮打开 /order?fp=xxx 自动填指纹，绑机零输入 |

## 四、一句话总结

- **工程**：两页并入官网单一代码库、单一定价真相、订单进 CRM——技术债清掉，发布链路自动化。
- **市场**：自助「试用→订阅」漏斗第一次真正打通，埋点齐全，一周后用数据说话。
- **用户**：从「加 TG 等报价」到「三分钟自助下单」，摩擦点只剩收款地址与订单状态两处，P0/P1 解决。
