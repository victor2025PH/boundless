# 无界 BOUNDLESS · 合作伙伴部署与验收清单

> 面向**现场部署/运维人员**的一页式操作手册与验收单。
> 工程实现细节见 `打包与分发方案_v1.md`；本文只讲「怎么装、怎么验、出问题怎么办」。

---

## 0. 前置条件（装前确认）

| 项 | 要求 |
|---|---|
| 操作系统 | Windows 10/11 64 位 |
| 显卡 | NVIDIA 显卡 + 较新驱动（`nvidia-smi` 可用） |
| 显存 | Lite ≥6–8GB / 标准 ≥16GB / 旗舰 ≥24GB |
| 磁盘 | 安装盘可用空间 **≥20GB**（按档位 11–35GB，含模型） |
| 网络 | 可访问分发方下载站（首启向导按档下载环境/模型） |
| 权限 | 普通用户即可（每用户安装到可写目录，免管理员） |

> 无需预装 conda / Python / Ollama —— 首启向导自动下载自包含运行环境。

---

## 1. 交付物

- `AvatarHub-Setup-1.0.0.exe`（安装包，薄核心 ~40MB）**或** `AvatarHub-portable-1.0.0.zip`（便携版，解压即用）。
- 下载站（或 U 盘离线介质）：`manifest.json` + `packs/*`（环境/模型/基座分片）。
- 若用环境变量指向下载站：`AVATARHUB_MANIFEST_URL=<站点>/manifest.json`。

### 1.1 离线介质（U 盘/移动硬盘）装箱单 —— 内部伙伴交付按此核对

| # | 内容 | 来源 / 说明 |
|---|---|---|
| 1 | `AvatarHub-Setup-<版本>.exe` 或便携 zip | `dist\`（薄核心，不在 publish 子目录里） |
| 2 | `manifest.json` + `packs\*`（环境/基座/模型全量） | `dist\publish\<版本>\`；base_url 留空=离线模式，向导以介质目录为源 |
| 3 | `versions.json` / `channels.json` / `SHA256SUMS.txt` | 同上（增量/回滚/校验用） |
| 4 | **LLM 二选一**：`secrets.bat`（含该伙伴的云端 key）**或** `OllamaSetup.exe` + `ollama_models\`（约 9.5GB） | 见 `部署指南.md` §3.1；导出：`python tools\export_ollama_models.py --models qwen2.5:14b bge-m3 --out <介质>\ollama_models` |
| 5 | `license.key`（按伙伴机指纹签发） | `python license_admin.py issue --machine <指纹> --edition <档> --days <N>`；伙伴机设 `AVATARHUB_LICENSE_ENFORCE=1` |
| 6 | 本清单 + `部署指南.md` + `使用说明.md` | 验收签字用 |

> 红线：介质**不得包含** `secrets\license_vendor_sk.pem`（签发私钥）、`_pending_models\community\*`（名人脸 DFM，平台封号风险）、母机 `.ollama` 全量目录。

## 2. 部署步骤

1. **安装**：双击 `AvatarHub-Setup-1.0.0.exe` → 下一步到底（或便携版直接解压）。
   - 静默/批量：`AvatarHub-Setup-1.0.0.exe /VERYSILENT /SUPPRESSMSGBOXES /NORESTART /DIR=<目标目录>`
2. **首次启动**：双击桌面「无界 BOUNDLESS」→ 弹出**首启向导**：
   - ① **验机**：自动识别显卡 + 检查磁盘可用空间 / 安装目录可写。
   - ② **选档**：选版本（显存不足项自动置灰；空间不足会红字预警）。
   - ③ **安装**：下载 + 校验 + 解压（含实时速度 / 剩余时间 / 生效下载源）。
3. **一键开机**：主界面点 **`▶ 一键开机并进入控制台`** → 核心链路就绪后自动打开控制台网页。

## 3. 验收清单（逐项勾验）

> 提示：底栏 **「一键验收」** 一键自动核验下表第 1–10 项 + STT SLA，结果可「保存报告」存为 `logs\acceptance_*.json/.txt` 作签字归档件；Hub 未起时可在该页直接「一键开机并复检」。

| # | 验收项 | 通过标准 | 验证方式 |
|---|---|---|---|
| 1 | 安装完成 | 桌面/开始菜单有「无界 BOUNDLESS」图标 | 目视 |
| 2 | 程序自检 | `selftest_result.txt` 含 `deps: …=ok` | 设 `AVATARHUB_SELFTEST=1` 跑一次 exe，看其目录下结果文件 |
| 3 | 首启向导 | 验机页显示显卡 + 磁盘✓ + 可写✓ | 目视向导第①步 |
| 4 | 组件就位 | 所选档位组件全部安装成功（无失败项） | 向导第③步进度 100%、无「重试失败项」 |
| 5 | 一键开机 | 点 CTA 后自动打开控制台，徽章「● 系统就绪」 | 主界面目视 |
| 6 | 能力就绪 | 语音克隆/识别/口型/广播 4 卡均「就绪 · N/N 引擎在线」 | 点能力卡看引擎明细（在线/延迟/许可） |
| 7 | 资源正常 | 资源卡显示 GPU% / 显存 X/Y GB / 内存% | 主界面目视 |
| 8 | 托盘常驻 | 关主窗口后托盘图标仍在、状态灯随核心链路变色 | 关窗 → 看系统托盘 |
| 9 | 掉线告警 | 核心服务异常时弹托盘红色告警 | 见 §4 故障排查（可选，勿在生产中演练） |
| 10 | 控制台可用 | 浏览器打开 `http://127.0.0.1:9000/ui` 正常 | 主界面「打开控制台」 |
| 11 | STT 实时 SLA | 最近一次闭环实测「达标」 | 「维护」设达标线 → 跑 `interp_selfcheck --stt-bench`；「一键验收」读最近报告 |

> 集成商整机自检（可选，需走打包环境的 python）：
> `python doctor.py --preflight`（离线预检：GPU/磁盘/脚本/端口/默认音色就绪）
> `python doctor.py`（全量：含 Hub/服务/角色/显存）
> 说明：未绑定克隆音的角色会自动回退「默认音色」——CosyVoice 内置参考(`CosyVoice/asset/zero_shot_prompt.wav`)优先、XTTS(`alltalk_tts/voices/*.wav`)兜底；doctor 的「默认音色」项离线核对二者是否就位，均缺会告警（无需额外打包音频文件，CosyVoice 自带）。
> 对话兜底：若本机语音引擎全部不可达，对话会继续出文字并提示「语音引擎暂不可用，已切为纯文字回复」（不再静默无声）；引擎在线时的偶发内容跳过不会误报。可设 `CONV_TTS_UNAVAIL_EVENT=0` 关闭该提示。
> 回退择优（默认关）：回退引擎固定按质量序 CosyVoice→XTTS，已知不可达的会被直接跳过（免 120s 死等）。设 `CONV_TTS_SMART_ROUTE=1` 后，仅当「中性句」且 CosyVoice 繁忙、XTTS 空闲时临时把 XTTS 提前以抢 TTFA；带情感的句子绝不降级 CosyVoice（唯一情感引擎）。单机单副本下保持默认关即可；多引擎/高负载部署可灰度开。
> `python pack_acceptance.py --quick`（装/更新/回滚链路快检）
> 以上三项与启动器「一键验收」**无需额外依赖**。
>
> 完整回归自检（含浏览器 E2E，可选）：`deliver_check.bat`（编排 doctor/acceptance/loadtest）。
> 其浏览器项（前端冲烟 / 加载·对话·降级·WebRTC 出画）需在 **facefusion 环境**装一次性自检工具。
> **最省心**：直接双击 `deliver_check.bat`——若检测到 facefusion 缺 playwright，会询问「是否现在自动备好(playwright + chromium)? [y/N]」，按 `y` 即一键就绪。
> **手动一条命令**：`python provision.py --with-selfcheck`（幂等：装 `requirements/selfcheck.txt` + `playwright install chromium` 到 facefusion）。
> 体检（`python provision.py`）末尾会显示「完整回归自检工具(可选)」一行，标 `[OK]/[--]` 提示是否已备好。
> 不想装也行：未安装时浏览器项会自动标为 **SKIP（不阻断"可交付"结论）**。非交互/CI 可设 `AVATARHUB_NOPROVISION=1` 跳过上面的询问。

## 4. 日常运维

- **托盘**：右键托盘图标 → 状态行 / 显示主窗口 / 打开控制台 / 一键开机 / 停止全部 / 退出。
  关主窗口 = 最小化到托盘，**服务继续后台运行**；彻底退出请用托盘「退出」。
- **设置**（底栏「设置」）：开机自启（随登录进托盘）/ 启动即最小化到托盘。
- **组件更新**（底栏「组件」）：增量更新，仅下载变化的组件。
- **维护**（底栏「维护」）：切换发布通道 stable/beta · 一键回滚到上一稳定版（带确认）。
- **授权**（底栏「授权」）：复制机器指纹 / 输兑换码在线激活 / 导入 `license.key`。

## 5. 故障排查速查

| 现象 | 可能原因 | 处理 |
|---|---|---|
| 安装时报「未知发布者」 | 安装包未代码签名 | 点「更多信息 → 仍要运行」；正式发布前由分发方签名（见下） |
| 向导验机页磁盘标红 | 安装盘空间不足 | 清理空间或换盘安装（需 ≥20GB） |
| 向导验机页「目录不可写」 | 安装到了受保护目录 | 重装到用户可写目录（如 `%LOCALAPPDATA%`） |
| 下载卡住/失败 | 网络或下载站不可达 | 向导会自动切换镜像并续传；确认 `AVATARHUB_MANIFEST_URL` / 网络 |
| 某能力卡「未启动」 | 该引擎服务未起或缺模型权重 | 点能力卡 →「重启」；或「组件」补装 |
| 徽章长期「正在启动」 | 首次加载模型较慢 | 等待 1–2 分钟；仍不就绪看「运行日志」/「一键体检」 |
| 托盘弹掉线告警 | 核心服务异常退出 | 点托盘「显示主窗口」→ 能力卡「重启」；进程守护通常会自动拉起 |

## 6. 发布前（分发方）

- **一键发布（推荐）**：先 `copy release.config.example.json release.config.json`，按注释填 `base_url`／证书／通道，然后：
  1. `python publish_release.py --dry-run` —— 预演：校验配置 + 母机预检 + 打印计划，**不构建/不发布**；
  2. `python publish_release.py` —— 正式发布：构建 exe → 签 exe → 门禁流水线（预检/烟测，失败即停）→ 装配 → 签安装包 → **发布即自检**（组件齐全+体积）→ 写发布回执 `logs/release_receipt_*.json` + 发布说明 `RELEASE_NOTES.md` + **可审计签收单 `dist/publish/<版本>/SIGNOFF.md`**；
  3. 按 `dist/publish/<版本>/UPLOAD.md` 上传后，`python publish_release.py --verify-remote`（组件极多可加 `--sample N`）—— 拉 `manifest.json` 并 HEAD 校验各组件可达+体积，通过后回写「远端已验证」到回执并刷新 `SIGNOFF.md`。
- **可审计签收单 `SIGNOFF.md`**（随包放在 `dist/publish/<版本>/`）：汇总发布身份、**已签名产物的签发主体+证书指纹(SHA1)+SHA256**、组件清单引用、自检结论、凭据出处与人工签收勾选项——交付方一眼可审、可长期归档。
- **完整性校验清单 `SHA256SUMS.txt`**（同目录）：逐文件 `<sha256>  <相对路径>`，标准格式，**覆盖下载负载 + `RELEASE_NOTES.md` + `SIGNOFF.md`**。整包批量核验：`sha256sum -c SHA256SUMS.txt`（Git Bash/Linux）；单文件：`certutil -hashfile <文件> SHA256`（Windows 自带）。`SIGNOFF.md` 发布后不可变（远端已验证状态记入回执 JSON，不回改签收单），故其哈希稳定可纳入清单。`--verify-remote` 会额外拉取远端 `SHA256SUMS.txt` 与本地逐行比对，任一不符或远端多出文件即判 FAIL。
- **代码签名**（消除 SmartScreen 拦截）：在 `release.config.json` 的 `sign` 段配置证书（PFX 文件 / 证书库指纹 `sha1` / EV 令牌主体）即自动签名；也可单独运行 `sign_artifacts.bat`。
- **底层闸门（进阶）**：`python make_release.py --ci`（母机预检 → 构建 → 烟测 → 装配，逐级门禁）；`python make_release.py --release-diff <上一版 manifest>`（老用户各档位实际下载量）。
