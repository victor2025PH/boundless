# 发布与上传指南（订单面板 + 下载页 + Win/Mac 安装包）

> 本指南串起本次商业化改造的全部产物：定价 → 订单面板 → 安装包 → 上传服务器 → 下载页自动更新。
> 标注 **【需你提供】** 的步骤需要服务器凭证或 Mac 设备，其余全部已就绪。

## 一、本次新增/涉及的文件

| 文件 | 作用 |
| --- | --- |
| `static/order.html` | 订单与购买面板（路由 `/order`、`/pricing`），月付/年付切换，USDT 收款 |
| `static/download.html` | 下载页（路由 `/download`），自动读取发布清单渲染下载按钮 |
| `avatar_hub.py` | 新增 `/api/order/plans`、`/api/order/create`、`/api/release/manifest` 三个公开 API |
| `AvatarHub-mac.spec` | macOS PyInstaller 打包配置（产出 `.app`） |
| `installer/build_mac.sh` | macOS 一键构建脚本（`.app` → `.dmg` + SHA-256） |
| `gen_download_manifest.py` | 扫描 `dist/` 安装包，生成 `dist/release_manifest.json`（下载页的单一真相） |
| `upload_release.py` | scp 上传安装包 + 清单到服务器，并做远端可达性校验 |
| `secrets/pricing.json`（可选） | 存在时覆盖内置定价，改价无需改代码 |
| `secrets/orders_web.jsonl` | 网页订单流水（每行一单，人工/半自动核销） |

## 二、Windows 安装包（已有流水线，直接用）

```bat
:: 完整门禁发布（构建 exe → 签名 → 打包 → 烟测 → publish 树）
python publish_release.py

:: 或只要安装包：build_launcher.bat 后用 Inno Setup 编译 installer\AvatarHub.iss
:: 产物：dist\AvatarHub-Setup-<ver>.exe
```

当前 `dist/` 已有 `AvatarHub-Setup-1.0.1.exe`（41 MB，SHA-256 已登记进清单）。

## 三、macOS 安装包 【需 Mac 设备】

PyInstaller 不能跨平台交叉编译，必须在 macOS 12+ 上构建一次：

```bash
# 在 Mac 上，把本仓库同步过去后：
python3 -m venv .venv_launcher_mac
source .venv_launcher_mac/bin/activate
pip install pyside6-essentials pyinstaller pillow cryptography zstandard
brew install create-dmg          # 可选，没有会自动回退 hdiutil

chmod +x installer/build_mac.sh
installer/build_mac.sh 1.0.1     # 产物：dist/AvatarHub-1.0.1.dmg
```

注意：
- Mac 版定位为「轻量控制台 / 远程接入」（重推理仍需 N 卡机器），下载页已如此标注。
- 脚本默认 ad-hoc 签名，用户首次打开需右键 → 打开。正式分发建议 Apple Developer ID 签名 + `notarytool` 公证（99 USD/年）。
- 图标：把 `assets/app.ico` 转成 `assets/app.icns` 后自动使用（无则用默认图标）。

## 四、生成下载清单

安装包就位后（`.exe` 在 Windows 构建，`.dmg` 从 Mac 拷回 `dist/`）：

```bat
python gen_download_manifest.py --base-url https://bd2026.cc/releases
```

生成 `dist/release_manifest.json`：自动登记两个平台的版本、体积、SHA-256、下载 URL。
下载页 `/download` 通过 `/api/release/manifest` 读取它，**无需改任何前端代码**。

## 五、上传服务器 【需你提供：服务器地址 / SSH 用户 / 密钥或密码】

```bat
python upload_release.py --host <服务器IP或域名> --user <SSH用户> ^
    --key C:\Users\user\.ssh\id_ed25519 ^
    --dest /var/www/dl/releases ^
    --base-url https://bd2026.cc/releases
```

服务器侧一次性准备（Nginx 示例）：

```nginx
# 在 bd2026.cc 或独立下载域名的 server 块中：
location /releases/ {
    root /var/www/dl;                 # 实际目录 /var/www/dl/releases/
    add_header Content-Disposition "attachment";
    types { application/octet-stream exe dmg; }
}
```

上传脚本会自动 HEAD 校验每个文件的远端体积与本地一致，不一致会报错。

## 六、官网接入（bd2026.cc 是独立仓库 victor2025PH/web3）

两种方式，二选一：

1. **反向代理（推荐，零重复代码）**：在官网 Nginx 加
   ```nginx
   location /order    { proxy_pass http://<引擎服务器>:8000/order; }
   location /pricing  { proxy_pass http://<引擎服务器>:8000/pricing; }
   location /download { proxy_pass http://<引擎服务器>:8000/download; }
   location /api/order/   { proxy_pass http://<引擎服务器>:8000/api/order/; }
   location /api/release/ { proxy_pass http://<引擎服务器>:8000/api/release/; }
   ```
2. **静态拷贝**：把 `static/order.html`、`static/download.html` 复制进 web3 仓库的 `public/`，
   官网导航加「购买 / 下载」两个入口。此方式下单接口不可用时，页面会自动降级为
   「生成订单摘要 → Telegram 提交」，流程不断。

## 七、改价与订单处理

- **改价**：编辑 `secrets/pricing.json`（参考 `/api/order/plans` 返回结构），保存即生效，无需重启改代码。
- **订单**：网页每笔下单追加写入 `secrets/orders_web.jsonl`（含 order_id、套餐、金额、联系方式、机器指纹）。
  收到 USDT 转账后，用现有 `license.py` 流程按机器指纹签发对应版本的许可证密钥，发给用户即完成核销。
