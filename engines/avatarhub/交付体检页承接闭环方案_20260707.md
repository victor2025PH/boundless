# 交付体检页「去启动」承接闭环（UK1）

> 2026-07-07 立项并完成。起因：四页巡检（H2）后复盘发现，全站警示条的「去启动」全部跳到交付体检页，
> 但这个"去处"承接不住——闭环 **告警 → 去处 → 解决** 的最后一环是断的。

## 一、摸底：三视角问题

**开发工程师**：
- D1 体检页只有「交付结论 + doctor 报告行」，没有逐服务在线状态，更没有启动手段。doctor 对离线服务的修复建议是一句"在控制台/向导确认对应服务已启动"——等于没说。
- D2 顶栏悬浮的服务状态面板有在线/离线，但 hover 才出现、不可达（触屏/键盘）、也没有启动动线。
- D3 后端能力其实齐全但没串起来：`app_config.SERVICES` 有每个服务的 label/env/script/port（启动器同源单一真相）；`POST /api/engine/start` 复用 `supervisor.launch_any` 可拉起任意登记服务——缺的只是一个把"哪个离线+怎么启动"摆到用户面前的面板。
- D4 `#selfcheck` 深链有装载缺口：F5 直达时 doctor 不自动跑（与 UH1 修过的 history/settings 同款坑，当时漏了这页）。

**市场经理**：
- M1 "自愈/体检"是卖点，但从告警到恢复的最后一步靠用户自己知道"启动器在哪、点哪个按钮"——演示时最尴尬的问题就是"它说去启动，启动在哪？"
- M2 多机部署时服务在别的机器上，一键代启无效——必须给可复制的手动命令兜底，否则闭环只对单机成立。

**用户**：
- U1 被「去启动」送到体检页后，看到的是"交付结论"和一堆报告行话，找不到任何"启动"按钮。
- U2 就算知道哪个服务离线，也不知道它对应启动器里哪个名字、哪条命令。

## 二、UK1 实施结果（2026-07-07 完成，门禁全绿）

| 项 | 结果 |
|---|---|
| UK1-1 服务目录端点 | `GET /api/services/catalog`：从 `app_config.SERVICES`（启动器/doctor 同源）导出 label/port/core/gpu/env/script/delay，排除 hub 自身；纯静态元数据不做探测（活性由既有 /health 提供，不重复造探针） |
| UK1-2 服务就绪面板 | 体检页顶部新卡：核心链路/可选增强两组，每行=状态点+人话名+端口；**离线行置顶**（来这的人就是为了找哪个没起）；「↻ 重新探测」即时刷新；目录空白时如实提示"Hub 版本较旧，重启后生效" |
| UK1-3 一键启动+命令兜底 | 离线行给「▶ 一键启动」：走既有 `POST /api/engine/start`（supervisor.launch_any，含单实例去重/GPU 释放态拒绝），发起后**就绪轮询到变绿**（上限=启动宽限×2，≥40s），成功/超时都有 toast 收尾；旁边「📋 命令」一键复制 `conda activate {env} && python {script}`——一键启动失败或多机部署时的兜底动线（M2） |
| UK1-4 来源聚焦 | 新 `goFix(服务名)`：各页警示条「去启动」改带服务名跳转——落地即高亮该服务行（蓝色描边+「← 你要找的就是它」+滚动定位+摆出手动命令）。接线四处：语音页情感胶囊条→`emotion_tts`、批量页情感条→`emotion_tts`、批量页 TTS 条→`fish_tts`（原来只有文字没按钮，顺带补上）、语音页就绪行 `voiceFix('tts')`→`fish_tts`；同传页未运行提示补「去体检页一键启动」；离开体检页自动清聚焦，不残留 |
| UK1-5 深链装载补齐 | init 补 `if(tab==='selfcheck')`：F5/书签直达时自动装载目录+跑 doctor（UH1 同款修法，这页当时漏了） |

**实施中发现并追加的方案外优化（再次优化点）：**

1. **不新造启动通道**——摸底时发现 `/api/engine/start` 的 `_engine_launchable` 本就放行全部登记服务（`_ENGINE_ONDEMAND` 只限制 /ops 卡的显示），直接复用，后端只加了一个只读目录端点；启动链路的单实例去重/远端迁移/GPU 释放态守卫全部白拿。
2. **就绪轮询代替"点完自己刷"**——一键启动后每 3s 探一次直到变绿或超时（按各服务 delay 元数据自适应上限），用户不用反复点「重新探测」；超时文案给两条出路（看日志/手动命令）。
3. **唱歌页豁免记录**——翻唱/原创歌的"未就绪"是**权重未部署**（要跑部署脚本下载 3.5~8.3GB），不是"服务没启动"，一键启动帮不上忙；该页已有诚实的部署指引卡，不接 goFix（豁免入案，防后续"补齐执念"）。

**验证**：UK1 断言 5 条入门禁（324/329 全过）；`gate.py` 三层全绿（Tier B 一次假红为并行唱歌会话改 `hist_prefix` 的中途竞态，其测试补齐后自愈）；`avatar_hub.py` py_compile 通过；catalog 数据源离线验证（17 服务/4 核心）。**后端目录端点需 Hub 重启后生效**——前端面板对旧 Hub 显示诚实的"版本较旧"提示，不装死。

### UK2 跨页着陆（展望第 2 条即时落地）

对话页（converse.html）是独立页面，goFix 够不着。原方案 STT 离线只有按钮 title 提示（悬停才可见、不可点）：

- **控制台侧**：`/ui?fix=服务名` 查询参数深链——init 时 `goFix(_fix)` 直落体检页对应服务行（服务名过白名单正则，防注入）；聚焦滚动定位改**两拍重试**（400ms/1500ms）：深链场景下目录/行首拍可能未渲染完，第二拍兜底，id 定位幂等。
- **对话页侧**：STT 离线时 `mic-hint` 升级为可点链接「语音识别离线 · 去启动 ↗ · 文字输入仍可用」（`/ui?fix=stt` 新标签打开）；`_sttGateHintOn` 记住"提示是门控写的"，恢复在线只还原自己的文案——不抢录音中/免提中的实时提示（micHint 同时被免提 partial 复用，直接覆写会打架）。
- **真机截图验证**：`/ui?fix=tts`（Coqui XTTS 实际离线）着陆即高亮该行 +「← 你要找的就是它」+ 摆出 `conda activate rvc && python tts_api.py`（截图 `logs/_k2_fixlink.png`）。

UK2 断言 2 条入门禁（终态 339/344）；`gate.py` 三层全绿（Tier B 授权契约 78/78，含并行会话同窗新增；Tier U 新增「一页纸对账」项亦绿）。
