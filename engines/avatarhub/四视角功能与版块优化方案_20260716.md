## 1. 当日已修复：窗口图标与最大化（根因 + 变更清单）

**为什么所有窗口都是 Edge 图标？** 三个原因叠加：

1. 桌面端站内页面统一用 Edge/Chrome `--app` 应用窗口打开（`launcher_qt.py _open_app_window`）。这种窗口的任务栏/标题栏图标**取自页面 favicon 的位图**；而全站此前只挂了 SVG favicon（`/static/icon.svg`），Chromium 不用 SVG 光栅化窗口图标 → 回退 Edge 图标。
2. 浏览器兜底请求的 `GET /favicon.ico` 在 Hub(9000) 上是 **404**（实测复现），换脸面板(8000)与同传页(7900)则**完全没有任何 favicon**。
3. 窗口尺寸只传了 `--window-size` 预设，从未传最大化旗标。

**已落地的修复**（均过 `_tokens_lint` / `_frozen_assets_test` 门禁与语法编译）：

| 文件 | 变更 |
| --- | --- |
| `launcher_qt.py` | `_open_app_window()` 默认最大化：`--start-maximized` + 主屏工作区尺寸钉位兜底（部分 Edge 版本对 `--app` 忽略最大化旗标）；`AVATARHUB_APP_MAXIMIZED=0` 可退回原 `_WIN_SIZES` 预设 |
| `avatar_hub.py` | 新增 `GET /favicon.ico`（优先 `assets/app.ico`，回退 `static/app-icon-256.png`）；`/help` 模板补双格式 favicon |
| `faceswap_api.py` | 新增 `GET /favicon.ico`；`/ui` 面板 head 挂 favicon |
| `live_interpreter.py` | 新增 `GET /favicon.ico`；主页/字幕层/术语表/会话导出/转写复盘 5 个内嵌页 head 挂 favicon |
| `static/*.html` ×13 | ui/home/phone/dashboard/ops/ask/converse/delivery/verify/setup/order/download/landing 统一补 `<link rel="icon" type="image/png" sizes="256x256" href="/static/app-icon-256.png"/>`（SVG 继续管标签页，PNG 管窗口/任务栏） |
| `设计规范_图标与令牌.md` | 「入口与窗口公约」写入两条新铁律：窗口默认最大化、favicon 双格式公约 |

**生效方式**：重启 Hub / faceswap / 同传服务后重开窗口即可（页面与 favicon 路由均 no-cache；若 Edge 图标缓存顽固，关闭全部 Edge 窗口再开一次）。

## 3. 四视角深度分析

### 3.1 开发工程师（可维护性 / 一致性 / 防回归）

**亮点**：单一真相意识强（/api/features 注册表、design-tokens.json、brand-icons.svg、_WIN_SIZES）；门禁文化成熟（tokens lint、冻结态资源、像素回归）。

**痛点**：
- **D1 巨石文件**：`avatar_hub.py` ≈3.1 万行（全产品路由耦合）、`ui.html` ≈8.5 千行（12 个页签挤在一个 Alpine 单文件里）——任何页签改动都要动全站文件，回归面大。
- **D2 三套 UI 栈漂移**：控制台(Tailwind+Alpine) / 同传页(内联令牌，人工同步) / 换脸面板(**完全脱离令牌体系**，Arial + `#e94560` 红)。今天的 favicon 事故正是这种"每个服务自己造页面"的典型产物：公约只覆盖了 Hub，8000/7900 两个端口天然漏网。
- **D3 窗口公约无门禁**：favicon/最大化/尺寸公约写在文档里，但没有 lint 拦截，下次新增页面仍可能漏。

**建议**（对应 §4 P1/P2）：新增 `tools/_favicon_lint.py` 扫描所有被 `_open_app_window` 打开的页面 head（P1）；`ui.html` 按页签拆分模块（P2）；换脸面板并入控制台或重皮肤（P1，同时是设计诉求）。

### 3.2 市场经理（品牌一致性 / 转化 / 演示体验）

**亮点**：free/pro 版位已进注册表；order/download/sales_onepager 商业前门齐全；白标(brand.js)可整体换肤。

**痛点**：
- **M1 品牌时刻掉链子**：给客户演示/交付验收时，任务栏一排 Edge 图标 = "这是网页不是软件"（今日已修）。同理，窗口非最大化开屏显小气（已修）。
- **M2 三套叫法打架**：启动台说「直播换脸/克隆你的声音」、首页说「VoiceX/ChatX/LiveX/LingoX」、控制台说「角色/创作/运营」。买家从官网 → 安装 → 使用要换三次心智词表，销售话术无法贯通到界面。
- **M3 促销横幅抢 C 位**：顶部横幅（自动播放/水印任务提示）常驻黄金位置，演示模式下干扰主叙事；pro 功能在控制台里无升级引导（版位信息只在首页展示）。

**建议**：页签副标题挂产品线名（如「语音 · 幻声 VoiceX」，P1）；演示模式收纳横幅+隐藏日志/运维页签（P1）；pro 页签对试用户显示锁标+一键到 /order（P2）。

### 3.3 用户（任务动线 / 认知负担 / 上手）

**亮点**：命令面板 Ctrl+K 全局可达；data-tip 富提示统一「标题|作用|何时用」；首启向导/交付体检闭环。

**痛点**：
- **U1 四个门不知走哪个**：启动台、/home、/ui、独立面板各有入口逻辑，功能重叠但清单不一致（如「实时对话」在首页有卡、控制台侧栏却没有对应页签，藏在启动台按钮里）。
- **U2 分组不贴任务流**：核心动线「建角色 → 克隆声 → 试音 → 开播/对话」在侧栏里横跨「角色→角色→创作→运营」三个抽象组；「同传」列在「运营」组（它明明是核心能力）。用户按"我要干什么"找，界面按"东西是什么"摆。
- **U3 头部太厚**：标题栏+顶栏+横幅+页签四层固定头，1280×860 窗口里内容起点近 1/5 屏（最大化默认后有缓解，但小屏仍在）。
- **U4 术语墙**：试听胶囊「整声/开口/检索/声乐」等行话无首次解释（有 tip 但要 hover 才知道）。

**建议**：侧栏按任务流重分组：**我的分身**(角色库/克隆) · **内容创作**(语音/唱歌/批量) · **上线**(开播/对话/同传) · **数据与运维**(看板/历史/交付体检/日志/设置)——只动 `hub.js tabs[].group` 与 `tabGroups`，20 分钟工作量（P0）；「对话 /phone」以页签形式入侧栏（新窗口打开，P1）；顶栏胶囊按角色折叠（用户模式只留 授权+自检+演示，其余进「更多」，P1）。

### 3.4 高级美术设计师（视觉系统 / 细节品质）

**亮点**：令牌体系（色/圆角/字阶/暗色层级）与 43 枚 lucide 线性图标已成体系且有门禁；emoji 三分类公约明确。

**痛点**：
- **A1 换脸面板是"外星页面"**：`#e94560` 洋红 + Arial + 无暗色层级令牌，与全站无界蓝紫气质完全断裂——它还是用户高频页面。
- **A2 窗口标题栏配色不跟品牌**：Edge `--app` 窗口标题栏颜色取 `<meta name="theme-color">`，目前只有 phone/home/landing 挂了 `#080b10`，ui/dashboard/ops/delivery 等没挂 → 深色页面顶着一条浅色标题栏。
- **A3 功能位残留 emoji**：启动台托盘菜单「🌐 官网」「💬 联系客服」、部分按钮「🎭/📡」前缀，按公约应换线性图标（内容位/状态位保留）。
- **A4 图标身份刚统一到位**：icon.svg（声波纹）与 app.ico/app-icon-256 同母版——favicon 双格式公约落地后，需在像素回归基线里补"窗口图标"检查项，防再漂移。

**建议**：全站 `<meta name="theme-color" content="#080b10">` 补齐（P0，一行/页）；换脸面板令牌化重皮肤（P1）；托盘/按钮 emoji 清理进下一次图标批次（P2）。
