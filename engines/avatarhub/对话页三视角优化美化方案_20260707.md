# 对话页（converse.html）三视角优化美化方案
> 2026-07-07 · 语音页 P0~P7 模式复用（方法与组件沉淀见《语音页三视角优化美化方案_20260706.md》）
> 页面：`/static/converse.html`（697 行单文件：SSE 流式对话 + 录音/免提 + 验真/引用脚注）

## 一、三视角快评（浓缩）

**开发工程师**：
- D1 串流气泡用固定 id（`bot-text`/`bot-audio-area`），多轮后 `getElementById` 永远命中第一轮气泡——**第二轮起回复文字与播放器都串写到最早那条回复里**（实锤 bug，非体验问题）。
- D2 角色下拉硬编码「刘德华」两项，与 `/profiles` 实际角色库脱节；气泡署名、欢迎语同样写死人名，白标交付时是漏改点。
- D3 顶栏状态点纯装饰（恒绿常亮），`/health` 明明有全套服务探测没有接。

**市场经理**：
- M1 演示时挂着别的客户角色名=事故；角色跟随角色库后，同一页面可给任何客户演示。
- M2 「回复质量反馈」在手机端/工作室都有（👍👎 闭环+差评触发调音建议），唯独对话页没有——数据面板上 converse 来源永远是空白。
- M3 出错只会「❌ 出错了」，观众看着演示者重新打一遍字，观感差。

**用户**：
- U1 说了一长段话结果失败，只能重录——重发按钮是刚需。
- U2 状态点常绿但实际语音掉线，回复突然没声音且无解释（诚实能力声明缺失）。
- U3 排队提示「排队中#1」用完不消失，一直挂在顶栏。

## 二、C1 实施结果（2026-07-07 完成，门禁全绿）

| 项 | 结果 |
|---|---|
| C1-1 角色动态化 | 下拉由 `/profiles?fields=name,has_voice` 动态填充（🎤=有声音 👤=仅形象 + 「默认角色」兜底项）；选择记 `localStorage(converse_profile)`（getItem null 区分"从没选过"与"选过默认角色"），优先级=上次选择>服务端 active>第一项；气泡署名/欢迎语走 `currentProfileName()`，**全文件不再出现硬编码人名**（UC1 门禁禁令）；切角色即时气泡确认 |
| C1-2 状态点三态 | 接 `/health` 15s 轮询：绿=就绪 / 黄=部分能力降级（语音合成 4 引擎全离线→「只能纯文字回复」、STT 双引擎全离线→「麦克风/免提不可用」、压力红→「回复可能变慢」，title 逐条说明）/ 红=Hub 不可达。与语音页 UI-P2 能力声明同一诚实原则：探测未返回不假告警 |
| C1-3 失败可重发 | `addRetry()`：error / busy / 网络异常三类气泡尾部加「🔁 重发这条」——同 payload 原样重试，**语音轮次不用重录**（payload 里带原 audio_base64）；点击后按钮自锁防连点 |
| C1-4 👍👎 反馈闭环 | `attachFeedback()`：done 后回复气泡尾部挂 👍👎，走与手机端/工作室同一 `/api/metrics/feedback`（source=converse），差评连击自动触发的调音建议/好评率聚合零改造直接获益；profile 传真实选择值（默认角色=空串），不传显示兜底名「数字人」，不污染按角色聚合 |
| C1-5 串流 id bug 修复 | 气泡内 `bot-text`/`bot-audio-area` 固定 id 改 class + `botDiv.querySelector()` 局部查找——多轮对话文字/播放器不再串写到第一条回复；`bot-bubble` 无引用一并清除；UC1 门禁加「禁止固定 id」回归护栏 |

**实施中发现并追加的方案外优化（再次优化点）：**

1. **排队提示自还原**——`queue` phase 会把顶栏 session 标签改写成「排队中#1 · 预计~Ns」，原代码用完不还原；收进 `finally` 统一复位（U3 治理）。
2. **反馈的 profile 语义校正**——初版把显示用的兜底名「数字人」传进指标；纠正为真实选择值（默认角色=空串），避免数据面板出现幽灵角色。
3. **欢迎语时序**——欢迎语移到 `loadProfiles()` 之后发（async init），署名与下拉一致；Hub 不可达时兜底「数字人」+红点解释，不装死也不报错吓人。

**验证**：`test_ui_optimization.py` 新增 UC1 断言 6 条（含两条"禁令"：不得出现硬编码人名、不得用固定 id），274/274 全过；`gate.py` 三层全绿；无头 Edge 截图确认动态角色下拉（当前角色库首个=刘亦菲）+ 欢迎语署名跟随 + 绿点 title。

**注意**：本页全部为前端改动，刷新即生效，无需重启 Hub。

## 三、C2 展望（下一阶段建议）

1. **会话持久化**：刷新即丢全部聊天记录——按语音页 P4 经验用 sessionStorage 存文本轮次（音频不存，气泡标注「音频未保留」），关标签页即清，符合对话隐私预期。约 40 行。
2. **角色头像进气泡**：`/profiles` 已带 thumbnail 字段，气泡 🎤 圆标换成角色缩略图，演示观感直接上一档。约 15 行。
3. **免提模式状态可视化**：免提中「听/想/说」三态只有 micHint 一行小字；参照语音页 readiness 模式给个显眼的状态胶囊。
4. **打断体验**：barge-in 已亚秒级，但打断后旧回复气泡停在半截没有任何标记；给被打断的气泡补「（已打断）」小注。约 10 行。
5. **STT 引擎健康门控下拉**（低优先）：Nemotron 离线时下拉仍可选，后端已有回退；可给选项加「（离线）」后缀——纯诚实性改进。

## 四、C2 实施结果（2026-07-07 完成，门禁全绿）

> §三的 1/2/3/4 全量落地；5（STT 下拉门控）后端已有优雅回退，本轮维持不做。

| 项 | 结果 |
|---|---|
| C2-1 会话持久化 | `sessionStorage(converse_session_v1)` 存最近 30 轮文字（单条截 2000 字，**音频体积大不存**）；**连服务端 session id 一起存**——刷新后 LLM 上下文接得上，恢复的不只是气泡壳，是真的还在同一轮对话里。带语音的旧回复标注「🔇 语音未保留」；顶部居中「已恢复 N 条」注记；「新对话」一并清档。选 sessionStorage 不选 localStorage：对话有隐私预期，关标签页即清才是对的语义（与语音页 P6 批量选 localStorage 的论证互为镜像——按内容性质选储存语义） |
| C2-2 头像进气泡 | `/profiles` 拉取追加 thumbnail 字段建 `profileThumbs` 映射；机器人气泡圆标有缩略图用真人头像、无则保留 🎤 渐变兜底。恢复的历史轮次用 `who` 字段还原"当时是谁说的"，切过角色后署名/头像不漂移 |
| C2-3 打断标记 | `markInterrupted()` 双路挂接：服务端 `cancelled`（新一轮顶掉旧轮）+ 本地 AbortError（⏹ 按钮/免提抢话）——被打断气泡尾部补「⏹ 已打断（回复未说完）」，防重复标注；**残段文字也入持久化**（后缀「（已打断）」），刷新后上下文不缺一截 |
| C2-4 免提状态胶囊 | `hf-state` 胶囊 400ms 轮询四态：👂 在听 / 🎙 你在说… / 💭 思考中… / 🗣 回答中 · 开口即打断——把"现在轮到谁说话"从 micHint 一行小字升格为常驻可视状态，免提口播时瞟一眼就知道能不能开口 |

**实施中发现并追加的方案外优化（再次优化点）：**

1. **持久化恢复的是"上下文"而不是"画面"**——原方案只说"存文本轮次"；实施时把服务端 session id 一并存取，刷新后继续追问，LLM 记得前文。这是从"视觉恢复"升级到"会话恢复"。
2. **署名/头像随 `who` 固化**——恢复时若用当前角色渲染旧气泡，切过角色的会话会"集体换脸"；每条 bot 轮次记录当时的角色名，历史忠实于发生时刻。
3. **打断残段进存档**——被打断的半截话原本只在屏幕上，刷新即失；随打断事件入档（带标注），恢复后上下文完整、且不假装说完了。

**验证**：`test_ui_optimization.py` 新增 UC2 断言 4 条（286/286 全过，含并行会话同窗新增项）；`gate.py` 三层全绿；无头 Edge 种档截图确认恢复态：居中注记、缩略图头像、「🔇 语音未保留」、「（已打断）」全部就位。纯前端改动，刷新生效。

## 六、UC3 会话导出回灌（2026-07-07 完成——手机页↔桌面页双向对照）

等待期反向巡检：以前都是桌面页模式迁给手机页，这次反过来拿**手机页对照桌面页**找缺口。

**摸底结论**：手机页（phone.html，并行线维护）功能面已超桌面对话页——验真/存证/择优重试/服务降级横幅/会话导出俱全。其中大多与桌面页已有能力等价或属于该页专属场景（直播模式），**唯独「会话导出」是桌面页真实缺口**：

- C2-1 给桌面页做的 sessionStorage 持久化是"关标签页即清"的隐私设计——这是对的；
- 但"聊出一段好话术/好台词想留底"是真实诉求，隐私设计反而堵死了唯一出口；
- 手机页早有 `exportSession()` 且后端 `/api/converse/session/export`（含滚动摘要的 Markdown）现成。

**落点**：桌面对话页顶栏「🔄 新对话」旁加「⬇ 导出记录」，走与手机页**同一个后端端点**（口径一致，禁止分家）；空会话点击就地 sys-note 提示"聊几句再导出"，不下载空文件。title 明示"本页记录关标签页即清空，导出是唯一留档方式"——把隐私设计的代价变成用户可理解、可行动的信息。

**验证**：UC3 断言 3 条（导出动线/空会话防呆/两页同端点）；后端端点真机冒烟 200 + Markdown 正确；头浏览器截图确认顶栏按钮在位（新对话｜导出记录｜大字）。

**顺手核对**（无需改动）：手机页 STT 门控（`pollHealthDegrade` 降级横幅）与桌面页 `_applySttGate` 是两套形态但同一诚实语义，各自适配其页面风格，记入模式索引不强行统一。
