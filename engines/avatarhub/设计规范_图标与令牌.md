# 设计规范 · 图标与设计令牌（全站单一真相）

> 生效日期：2026-07-16　适用范围：网页控制台(/ui) · 统一首页(/) · 对话端(/phone) · 同传页(7900) · 桌面启动台(launcher_qt)
> 本规范由门禁守护：`tools/_tokens_lint.py`（令牌三方一致）与 `tools/_frozen_assets_test.py`（安装版资源定位），均在 `run_all_tests.py` 套件内。

## 一、设计令牌（颜色 / 圆角 / 字阶）

**单一真相 = [static/design-tokens.json](static/design-tokens.json)**，两端各自消费：

| 消费方 | 接入方式 | 说明 |
| --- | --- | --- |
| 网页全站 | [static/brand.css](static/brand.css) 中 `--bd-*` 变量 | 与 tokens 同值，改一边必须同步另一边（lint 会拦） |
| 桌面启动台 | [launcher_theme.py](launcher_theme.py) 启动时读取 | 随包分发离线可读；文件缺失回退内置值；冻结态从 exe 旁 `static/` 定位 |
| 同传页(7900) | 页面内联令牌（自包含单页） | 值与 brand.css 同源，改版时人工同步 |

**关键令牌口径**：

- 状态色语义唯一：绿 `#34d399`=就绪/在线 · 琥珀 `#fbbf24`=加载/待动作 · 红 `#f87171`=故障 · 石板灰=未启动。
- 品牌色：主 `#4f7aff` 无界蓝 + 辅 `#a855f7` 幻紫；蓝→紫渐变只用于「品牌 Hero 与全页唯一主 CTA」，其余按钮一律中性底。
- 暗色层级（预混实色，Qt/网页同观感）：底 `#080b10` < 卡片 `#12151a` < 抬起面 `#171a1e`，描边 `#26282d`。
- 圆角：12 / 16 / 20 / pill；间距 4/8/12/16 基准刻度；正文字号下限 12px。

**改色流程**：改 `design-tokens.json` → 同步 `brand.css` 对应变量 →（若动了同传页涉及色）同步 `live_interpreter.py` 内联 → 跑 `python tools/_tokens_lint.py` → 重启启动台与 Hub 验收。

## 二、图标体系

**单一真相 = [static/brand-icons.svg](static/brand-icons.svg)（43 枚，lucide 风格单色线性）**，由 [tools/_gen_brand_icons.py](tools/_gen_brand_icons.py) 生成——**改图标改脚本重跑，不要手编 svg 文件**（曾因手编编码事故立此规矩）。

| 消费方 | 用法 |
| --- | --- |
| 网页（ui/home/phone） | `<svg class="bd-ic" width="15" height="15" fill="none" stroke="currentColor" stroke-width="1.9" stroke-linecap="round" stroke-linejoin="round"><use href="/static/brand-icons.svg?v=版本#i-名称"/></svg>` |
| hub.js 动态渲染 | `this.icx('名称')`；首页用 `icSvg('名称', px)` |
| 桌面启动台 | `_sprite_icon('名称', 颜色, 像素, dot=状态点色)` 渲染为 QIcon（QtSvg） |
| 同传页(7900) | 内联持有所用子集拷贝（跨端口自包含），新增先入库再同步内联 |

**铁律：尺寸与描边必须写成 SVG 呈现属性**（如上示例），不能只依赖 CSS 类——CSS 被缓存/丢失时图标最坏退化为 15px 小图，绝不允许再出现"无样式 SVG 300×150 黑块爆版"（2026-07-16 实锤事故）。

**emoji 三分类公约**（判断"该不该换图标"就问这一条）：

| 类别 | 判据 | 处置 |
| --- | --- | --- |
| 功能位 | 按钮/导航/菜单/字段标签上的"装饰性图形" | 一律换线性图标 |
| 状态位 | 表达运行状态的图形（🔴 录音中、⚡ 流式、🔇 无音色） | 保留（红点=录音是通用隐喻）；新增时优先"彩色圆点+文字" |
| 内容位 | 角色名旁 🔥、情绪选项 😄、字幕文本、空态引导插画 | 保留，emoji 就是正确形态 |

**缓存与版本**：`/static` 已带 `Cache-Control: no-cache`（改文件即生效）；图标库引用带 `?v=YYYYMMDD字母` 版本参数，视觉大改时 bump 一次（ui.html / hub.js / home.html / phone.html 同步）。

**新增图标步骤**：`tools/_gen_brand_icons.py` 的 SYMBOLS 加一条（24 视口 lucide 路径）→ 重跑脚本（自动 XML 校验）→ 消费处按上表引用 →（若桌面/同传用到）同步映射或内联。

## 三、入口与窗口公约（桌面门户）

- 功能入口单一真相 = Hub `GET /api/features` 注册表（含 `ic` 图标字段、`service` 就绪键、`edition` 版位）：网页首页、命令面板、桌面工作台三端同源，加功能只改注册表。
- 桌面端所有**站内页面**一律 `_open_app_window()`（Edge `--app` 无边框应用窗口）打开；官网/客服等**外部链接**走默认浏览器；本地文档走系统关联程序。
- **窗口默认最大化**（2026-07-16 定版）：`_open_app_window()` 统一带 `--start-maximized` + 工作区尺寸钉位（部分 Edge 版本对 `--app` 忽略最大化旗标，钉位兜底）；`AVATARHUB_APP_MAXIMIZED=0` 可退回 `_WIN_SIZES` 预设尺寸。
- **页面身份 = favicon 双格式公约**（2026-07-16 定版）：Edge/Chrome `--app` 窗口的任务栏/标题栏图标取自页面 favicon 的**位图**（SVG 不参与光栅化）——只有 SVG favicon 时窗口退化成 Edge 图标（实锤事故）。故每页必须两条：`icon.svg`（标签页）+ `app-icon-256.png`（窗口/任务栏）；三个带 UI 的服务（Hub 9000 / faceswap 8000 / 同传 7900）均须提供 `GET /favicon.ico` 兜底路由（未挂链接的页面浏览器自动回退请求它）。Web Manifest（`manifest*.json`）继续用于 PWA 安装身份。
