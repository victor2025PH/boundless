# 安装教程（图文版）

> 面向**第一次装机**的人：从拿到程序到“装完 10 分钟出声”。技术细节/多机/授权见 `部署指南.md` 与 `README.md`。
> 全程只在电脑上操作；手机相关的看《手机同传扫码授权全流程》。

---

## 0. 先看你的电脑够不够

| 项目 | 最低 | 推荐 |
|---|---|---|
| 系统 | Windows 10/11 | Windows 11 |
| 显卡 | NVIDIA ≥ 8GB（RTX 3060 起） | RTX 4090 / 5090（实时高清/同传） |
| 运行时 | 装了 Miniconda / Anaconda | 同左 |
| 大脑(LLM) | 本机 Ollama 或云端 key | 二选一即可 |

> 只想做“图片/视频换脸 + 声音克隆”入门卡也能跑；“实时数字人 / 同传全家桶”建议 24GB+ 显存。装完后控制台「设置 → 🧮 硬件档位」会按你这台机器实测告诉你每个功能能不能开。

---

## 1. 装程序本体（三选一）

**方式 A · 安装包（给非技术同学，最省事）**
1. 双击 `AvatarHub-Setup-1.0.0.exe`（每用户安装，免管理员）。
2. 安装完成后，桌面/开始菜单出现 **AvatarHub** 图标。
3. 双击打开——这是“前门控制台”，各 AI 服务还需第 2–3 步就位。

**方式 B · 直接用源码目录**
- 把整个项目目录放到本地（例如 `C:\模仿音色`），进入第 2 步。

**方式 C · 已有旧版**
- 覆盖代码即可；**模型和 conda 环境不用重装**。

---

## 2. 备齐运行环境（conda 环境 + 依赖）

打开命令行，进入项目目录，跑体检：

```bat
provision.bat
```

- 只读，不改动，列出 6 个 conda 环境与关键模型目录**是否就位**。
- 有缺失就自动补：

```bat
provision.bat --create
```

> 会按 `requirements\` 基线创建缺失环境并装依赖（可反复跑，幂等）。
> 换了机器/盘符/用户名**不用改代码**，路径全自动探测。
> 图形启动器里也有「环境体检」按钮，等价于上面命令。

---

## 3. 放模型 + 配“大脑”（LLM）

**模型**：体积大，**不随安装包分发**，要单独就位（参见 `MANIFEST.json`，从源机复制 MuseTalk / LivePortrait / CosyVoice / facefusion 等目录）。

**大脑 LLM（对话/同传要用）二选一**：
- **云端 key（有外网最简单）**：把 `secrets.example.bat` 复制成 `secrets.bat`，填入给你的 key。
- **本地 Ollama（离线/内网）**：装 `OllamaSetup.exe`，导入 `qwen2.5:14b` + `bge-m3`（约 9.5GB）。

---

## 4. 一键启动

```bat
:: 方式 A（推荐）：图形启动器，点“启动全部”，实时看每个服务就绪灯
launcher.bat

:: 方式 B：命令行一键起核心链路
start_all_services.bat

:: 需要换脸/唱歌/同传等扩展：
set START_EXTRAS=1 & start_all_services.bat
```

启动后打开控制台：**http://127.0.0.1:9000/ui**

> 第一次打开建议先走**首次运行向导** `http://127.0.0.1:9000/setup`：它检测硬件、配好并发/互动，写入 `deploy.env.bat`，重启 hub 生效。

---

## 5. 验证“真的装好了”

```bat
python ready.py       :: 核心链路是否就绪（退出码 0=全就绪）
python doctor.py      :: 一键体检（全绿即可用）
```

启动脚本会自动跑一次体检。控制台右侧「📊 服务状态」全绿即可用。

**装完 10 分钟出声（冒烟测试）**：
1. 控制台 `/ui` → 角色/声音 → 上传一段 10 秒清晰人声 → 克隆 → 试听。
2. 进对话页 `/converse`（或手机 `/phone`）→ 打字或说话 → 数字人用克隆音回应。

出声了 = 安装成功。接着看《使用教程（图文版）》。

---

## 6. 装不上/打不开——按现象查

| 现象 | 处理 |
|---|---|
| 控制台打不开 | `python service_manager.py status` 看 hub 是否运行；重启 hub |
| 某服务红/未就绪 | 看对应 `logs\<服务>.log`；多为显存不足或模型没加载完，等待或重启 |
| 端口被占用 | `start_all_services.bat` 会先杀旧窗口；或手动结束占用进程 |
| 显存爆(OOM) | 只跑核心（关扩展）；高清口型最吃显存 |
| 找不到 conda | 按第 2 步跑 `provision.bat --create`；或装 Miniconda 后重试 |
